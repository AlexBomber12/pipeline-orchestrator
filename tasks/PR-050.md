# PR-050: Verify head SHA change after FIX push

Branch: pr-050-fix-head-verification

- Type: bugfix
- Complexity: low

## Problem

After `fix_review_async` returns exit code 0, handle_fix increments push_count, updates _last_push_at, and logs "Fix pushed". But exit code 0 doesn't prove that Claude actually committed and pushed anything. Claude may have decided there was nothing to fix, or committed but failed to push (network error silently swallowed), or written local edits without committing.

Downstream consequences:
- push_count tracks "fix attempts" rather than actual pushes
- _last_push_at is set to now() even when no push happened, breaking _has_new_codex_feedback_since_last_push logic
- Daemon posts @codex review even though nothing changed, leading to duplicate reviews on the same HEAD
- Operational logs misrepresent what the daemon did

This directly explains the open bug "daemon posts @codex review without push".

## Root cause

`src/daemon/runner.py:1677-1688` unconditionally treats exit 0 as a successful push:

```python
push_time = datetime.now(timezone.utc)
self._last_push_at = push_time
if self.state.current_pr is not None:
    self._last_push_at_pr_number = self.state.current_pr.number
    self.state.current_pr.push_count += 1
    ...
```

No check whether HEAD actually moved.

## Scope

After fix_review_async returns 0, compare HEAD SHA before and after. Only treat as a push if HEAD changed. If HEAD unchanged, log that Claude made no changes and return to WATCH without posting a new @codex review.

## Changes

### src/daemon/runner.py

In `handle_fix`, capture HEAD before running Claude:

```python
# After the branch refresh block (PR-049), before fix_review_async:
head_before = ""
try:
    head_before = _git(
        self.repo_path, "rev-parse", "HEAD"
    ).stdout.strip()
except (subprocess.CalledProcessError, subprocess.TimeoutExpired, OSError):
    # Empty string falls through: the post-check will then treat any
    # resolvable HEAD as "changed" (safer bias than treating as no-op).
    pass
```

After fix_review_async returns 0, check HEAD again:

```python
head_after = ""
try:
    head_after = _git(
        self.repo_path, "rev-parse", "HEAD"
    ).stdout.strip()
except (subprocess.CalledProcessError, subprocess.TimeoutExpired, OSError) as exc:
    self.state.state = PipelineState.ERROR
    self.state.error_message = f"rev-parse after fix failed: {exc}"
    self.log_event(self.state.error_message)
    return

if head_before and head_before == head_after:
    self.state.state = PipelineState.WATCH
    self.log_event(
        "FIX REVIEW exited 0 but HEAD unchanged; "
        "no push, skipping @codex review"
    )
    return

# HEAD moved: proceed with push accounting and post @codex review
push_time = datetime.now(timezone.utc)
self._last_push_at = push_time
...
```

Note: local HEAD change proves a commit happened, but not that push succeeded. For stronger verification, also check `git rev-parse origin/{branch}` after and compare. However, Claude's AGENTS.md flow pushes as part of the fix loop, and if push failed Claude would typically re-try or surface the error in stderr. Local HEAD comparison is the minimum viable check; remote comparison is a possible future improvement.

### tests/test_runner.py

Add tests:
- test_handle_fix_skips_review_post_when_head_unchanged: mock _git rev-parse to return same SHA before and after, expect no @codex review posted, state = WATCH, push_count not incremented
- test_handle_fix_counts_push_when_head_changes: mock rev-parse to return different SHAs, expect push_count +1, _last_push_at updated, @codex review posted
- test_handle_fix_error_on_rev_parse_after_failure: mock second rev-parse to raise, expect ERROR with message containing "rev-parse after fix"
