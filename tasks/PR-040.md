# PR-040: Rate limit auto-pause

Branch: pr-040-rate-limit

## Scope

Daemon should auto-pause when approaching Claude rate limit threshold. Configurable via config.yml AND Settings UI. Currently the daemon burns tokens on diagnose_error loops and does not know when to stop.

## Changes

### src/config.py

Add to DaemonConfig:
```python
rate_limit_pause_percent: int = 90
```

Add `"rate_limit_pause_percent"` to `_DAEMON_FIELDS` set.

### config.yml

Add under daemon:
```yaml
daemon:
  rate_limit_pause_percent: 90
```

### src/daemon/runner.py

1. Add a new PipelineState:
No, don't add new state. Use existing `active` field on RepoState. When rate limited, daemon sets a flag and skips Claude CLI calls until reset time.

Better approach: add rate limit tracking to PipelineRunner:

```python
self._rate_limited_until: datetime | None = None
```

2. Before every Claude CLI call (run_planned_pr, fix_review, diagnose_error), check:
```python
def _check_rate_limit(self) -> bool:
    if self._rate_limited_until is not None:
        if datetime.now(timezone.utc) < self._rate_limited_until:
            remaining = (self._rate_limited_until - datetime.now(timezone.utc)).total_seconds()
            self.log_event(f"Rate limited, resuming in {int(remaining)}s")
            return False
        self._rate_limited_until = None
        self.log_event("Rate limit window expired, resuming")
    return True
```

3. After Claude CLI calls, parse stderr for rate limit signals. Claude Code returns rate limit info. If detected:
```python
if "rate limit" in stderr.lower() or "429" in stderr:
    self._rate_limited_until = datetime.now(timezone.utc) + timedelta(minutes=30)
    self.log_event("Rate limit detected, pausing for 30 min")
```

4. In handle_coding, before claude_cli.run_planned_pr:
```python
if not self._check_rate_limit():
    return
```

5. Same guard in handle_fix before fix_review, and in handle_error before diagnose_error.

6. In handle_error, do NOT call diagnose_error if the error_message itself contains "rate limit" or "Timeout":
```python
if any(kw in context.lower() for kw in ["rate limit", "timeout", "429"]):
    self.log_event("Skipping AI diagnosis for rate-limit/timeout error")
    return
```
