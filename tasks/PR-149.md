# PR-149: Coder-neutral log_event text in Python handlers

## Problem

Dashboard event log displays messages hardcoded to "Claude" regardless of which coder is actually active. Observed live on PR-138 screenshot: event log shows `FIX: Claude pushed, resetting idle timer` while the "Coder: Codex CLI" chip is displayed at the top of the same page.

PR-129 already fixed hardcoded coder labels in templates. This PR cleans up the remaining occurrences in Python handler code.

Authoritative grep reveals only two user-facing log_event / error_message strings with hardcoded "Claude":

1. `src/daemon/handlers/fix.py:65`: `self.log_event("FIX: Claude pushed, resetting idle timer")` inside `_monitor_fix_idle`. This fires regardless of active coder - the idle monitor polls GitHub for pushes without regard to who pushed.
2. `src/daemon/handlers/coding.py:249`: `f"Claude CLI succeeded but no PR found for branch {target_branch!r}"` set as `error_message` and then logged. Fires for whichever coder just completed.

Out of scope intentionally:

- `src/daemon/handlers/breach.py:56` ("killing Claude CLI") - the breach monitor is Claude-specific (Codex has no usage API to poll for breaches), so the text is accurate in context.
- Legitimate Claude-specific references in docstrings, `src/coders/claude.py` display name, `src/usage.py` module docstring, auth settings UI labels - all technically correct.
- `_CLAUDE_CLI_COAUTHOR` co-author trailer constant - Claude-specific feature, correct.

## Scope

Change the two user-facing strings to use the active coder name with the project's existing `[{coder_name}]` style (matches `fix.py:94` pattern `f"[{coder_name}] entering FIX"`).

### fix.py:65

Inside `_monitor_fix_idle`, `coder_name` is NOT in local scope. Two clean options:

Option A (preferred): read from `self.state.coder or "coder"` at log emission time. Always up-to-date, no signature change.

Option B: add `coder_name: str` parameter to `_monitor_fix_idle`, pass from caller at spawn time.

Choose Option A for minimal diff. Resulting line:

`self.log_event(f"FIX: [{self.state.coder or 'coder'}] pushed, resetting idle timer")`

### coding.py:249

`coder_name` IS in local scope at line 249. Resulting line:

`self.state.error_message = (f"[{coder_name}] coder succeeded but no PR found for branch {target_branch!r}")`

(Note the move from "Claude CLI succeeded" to "coder succeeded" - the `[{coder_name}]` prefix makes the identity explicit.)
