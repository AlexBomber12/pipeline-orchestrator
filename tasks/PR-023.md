# PR-023: Index page redesign - zero scroll repo cards

Branch: pr-023-index-redesign

## Scope

Redesign the dashboard index page to show all information in zero-scroll view. Each repo is a card with status, sprint progress, current task, PR status, and controls. Remove separate stats cards, activity feed, and alerts panel from index. Stats become a compact bottom bar. Alerts are shown inline on repo cards.

## Design spec

### Layout
- Nav bar (existing, no changes)
- Repo cards grid (2 columns on desktop, 1 on mobile) - main content
- Bottom status bar (compact, single row)

### Repo card anatomy

Each card is a rounded container (bg-surface-2, border-white/5, rounded-lg, p-4).

Header row:
- Left: colored dot (state-dependent, pulsing for active states) + repo name (font-mono, white) + state badge (colored pill)
- Right: action buttons ("Upload tasks", "Start" when IDLE with TODO tasks)

Body (2-column grid):
- Left column "Sprint progress": "N / M done" text + progress bar (4px height, green fill for done ratio)
- Right column "Current task": PR ID + short title, or "No tasks" italic gray

PR row (only if current_pr exists):
- PR number + CI status dot (green/red/yellow pulsing) + CI label + review emoji (thumbs up/eyes/pencil) + iteration count
- If state is CODING: "Claude is coding..." italic

ERROR state:
- Card gets left border 2px solid fail color
- Body shows error message instead of normal content

### Bottom status bar

Single row container (bg-surface-2, rounded-lg, p-3):
- Left: stat clusters (repos count, done today, active count, alert count with red color if > 0)
- Right: latest alert summary (if any) - single line with repo name + error text + time ago

### Sprint progress calculation

In the web app, when building repo state for the index:
- Read QUEUE.md content from Redis (it should be included in RepoState) OR parse from the activity history
- Actually simpler: add queue_summary to RepoState model

### src/web/templates/components/repo_cards.html (new, replaces repo_list.html)

Grid of repo cards. Each card:
- Header: dot + name + badge + buttons
- Body: sprint progress + current task
- PR row: number + CI dot + review emoji + iteration
- Error variant: red left border + error message

Color mapping:
- IDLE: gray dot, gray badge
- CODING: cyan dot (pulsing), cyan badge
- WATCH: yellow dot (pulsing), yellow badge
- FIX: orange dot (pulsing), orange badge
- MERGE: green dot, green badge
- HUNG: purple dot, purple badge
- ERROR: red dot, red badge, red left border on card

Progress bar: 4px height, bg-surface-3 track, bg-ok fill, width = (queue_done/queue_total*100)%

Buttons:
- "Upload tasks": always visible, purple border, triggers file upload modal or drag area
- "Start": only when state is IDLE and queue_total > queue_done, cyan border

### src/web/templates/components/status_bar.html (new, replaces stats_cards.html)

Single row:
- Left: 4 stat clusters inline (number + label)
  - Repos: count (white)
  - Done today: count (white)
  - Active: count of repos in CODING/WATCH/FIX (white)
  - Alerts: count of repos in ERROR/HUNG (red if > 0, white if 0)
- Right: latest alert one-liner (if any error/hung repos exist)
  - Format: "{repo_name} {error_message_truncated} - {time_ago}"
