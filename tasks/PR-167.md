# PR-167: CI logs in FIX prompt and rename FIX REVIEW → FIX FEEDBACK

Branch: pr-167-fix-feedback-with-ci-logs
- Type: feature
- Complexity: low
- Depends on: none
- Priority: 1
- Coder: any

## Problem

When daemon enters FIX state, the coder is invoked with the bare prompt string `"FIX REVIEW"` (`src/claude_cli.py:92,265` and `src/codex_cli.py:117`). The coder receives no information about WHY FIX was triggered — neither the CI failure logs nor the review comments. AGENTS.md instructs the coder to fetch GHA logs via `gh run view --log-failed`, but reliance on coder discipline is non-deterministic. Sometimes the coder reads logs, sometimes it does not, sometimes it gets distracted by review comments and ignores CI failures entirely.

Observed today on PR #220 and PR #217: claude in FIX cycle correctly identified the bug because human (Aleksei) had previously left a diagnostic comment. Without that hint, claude might have looped without finding root cause.

The fix is to fetch the CI failed logs (and the latest review comment) in the daemon and inject them into the FIX prompt directly. Coder no longer needs to remember the discipline — the daemon hands the relevant context on a plate.

Additionally, the trigger phrase "FIX REVIEW" is misleading. It reads grammatically as "fix the review" (which is nonsensical). The intended meaning is "apply fix based on feedback" — which covers both review comments AND CI failures. Renaming to "FIX FEEDBACK" matches the actual semantics.

## Scope

Two combined changes:

1. **Inject CI failed logs and latest review feedback into the FIX prompt.**
2. **Rename trigger phrase from "FIX REVIEW" to "FIX FEEDBACK"** across daemon code, AGENTS.md, and tests.

Both changes are small and tightly coupled (touching the same code paths and same documentation). Combining keeps the PR coherent.

This PR does NOT change FIX cycle semantics, transitions, retry logic, escalation, or merge logic. It does NOT remove the existing fallback where coder can still read logs via gh CLI itself — the new prompt enriches but does not constrain.

## Files to touch

`src/daemon/handlers/fix.py`: in `handle_fix`, before invoking the coder, fetch:

- Latest CI run logs for the failing job (if `current_pr.ci_status == FAILURE`). Use `gh run view <run_id> --log-failed` or equivalent. Truncate to last ~5000 chars to keep prompt size sane.
- Latest review feedback (if `current_pr.review_status == CHANGES_REQUESTED`). Use `gh pr view <number> --json reviews` and pick most recent CHANGES_REQUESTED body.

Compose the prompt:

```
FIX FEEDBACK

CI failure logs (last 5000 chars):
<truncated logs>

Latest review feedback:
<review body>
```

If only CI is failing (no review feedback), include only CI section. Vice versa for review-only. If both are present, both sections in the prompt.

Pass this enriched prompt to the coder plugin. The coder plugin's `run_claude` / `run_codex_async` already accepts arbitrary prompt text — no plugin changes needed.

`src/claude_cli.py` and `src/codex_cli.py`: rename hardcoded `"FIX REVIEW"` strings to `"FIX FEEDBACK"`. Three sites in claude_cli (line 92 sync, 265 async; one helper signature mention), one in codex_cli (line 117). Functions named `run_fix_review_*` rename to `run_fix_feedback_*` for symmetry.

`src/daemon/handlers/fix.py`: the comment / docstring / log lines that reference "FIX REVIEW" rename to "FIX FEEDBACK". Find via `grep -n "FIX REVIEW" src/daemon/handlers/fix.py`. About 5-8 occurrences.

`AGENTS.md`: section `Fix loop` runbook (or wherever "FIX REVIEW" is documented) — rename to "FIX FEEDBACK." Update prose to describe new prompt structure (coder receives CI logs + review feedback in prompt directly, no longer needs to fetch them itself, but CAN still fetch additional context via gh CLI if needed).

`tests/test_fix_handler.py` (or wherever fix handler is tested): update test expectations for the new prompt content (CI logs section, review feedback section). Update string assertions from "FIX REVIEW" to "FIX FEEDBACK."

`tests/test_claude_cli.py`, `tests/test_codex_cli.py`: update string assertions for the renamed functions and prompt text.

`tests/e2e/lib/coder_shim.sh` and `tests/e2e/lib/coder_shim.py`: any mention of "FIX REVIEW" updated to "FIX FEEDBACK."

If any other test fixture or string literal references "FIX REVIEW" (search via `grep -rn "FIX REVIEW" .` from repo root), update consistently.
