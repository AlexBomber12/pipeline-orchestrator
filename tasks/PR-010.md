# PR-010: Daemon recovery and error handling

Branch: pr-010-recovery

## Scope

Add recovery logic to PipelineRunner so that on restart the daemon reconstructs its state from QUEUE.md + GitHub (open PRs). Also handle the edge case where daemon was in DOING state when it crashed.

## Files to modify

### src/daemon/runner.py

Add method:

async recover_state(self)
- Called at the beginning of run_cycle, before preflight, but only on first cycle or when state is IDLE/ERROR.
- Step 1: Parse QUEUE.md, look for tasks with status DOING.
  - If found: check GitHub for open PR with matching branch name.
    - PR exists: set state to WATCH, set current_task and current_pr, log "Recovered: DOING task {pr_id} -> WATCH PR #{number}"
    - PR does not exist: this means coding was interrupted. Set state to CODING, set current_task, log "Recovered: DOING task {pr_id}, no PR -> re-running CODING". Then call handle_coding().
- Step 2: If no DOING tasks, check GitHub for any open PRs on this repo.
  - If found: set state to WATCH, set current_pr. Try to match PR branch to a DONE task in QUEUE (the task was marked DONE but PR is still open = not yet merged). Log "Recovered: orphan PR #{number} -> WATCH"
  - If not found: state stays IDLE.

Modify run_cycle:
- Add a flag self._recovered (bool, default False in __init__)
- At start of run_cycle, if not self._recovered: call recover_state(), set _recovered = True
- Continue with normal cycle logic

Add git pull in handle_idle (before parsing QUEUE.md) to ensure daemon has latest task files. Use `git checkout {branch} -- tasks/` to avoid conflicts with working tree changes from Claude CLI. Actually safer: `git fetch origin {branch}` then `git reset --hard origin/{branch}` only on the main branch when in IDLE state (no active work in progress).

### tests/test_recovery.py

Create separate test file for recovery scenarios:

- Test: DOING task with existing PR -> recovers to WATCH
- Test: DOING task without PR -> recovers to CODING (mock handle_coding)
- Test: No DOING tasks, open PR exists -> recovers to WATCH
- Test: No DOING tasks, no open PRs -> stays IDLE
- Test: Recovery runs only once (_recovered flag)

Mock subprocess.run (git), github_client.get_open_prs, queue_parser.parse_queue.

## Important: git working tree management

When daemon enters IDLE and is about to pick next task, it must sync the repo to latest main:
- `git fetch origin {branch}`
- `git checkout {branch}`
- `git reset --hard origin/{branch}`

This ensures QUEUE.md and task files are current. This must NOT happen during WATCH/FIX (would destroy Claude's working branch).

Add a helper method sync_to_main(self) that does this, and call it at the start of handle_idle.

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- Recovery correctly reconstructs state from QUEUE.md + GitHub
- sync_to_main only runs in IDLE state
