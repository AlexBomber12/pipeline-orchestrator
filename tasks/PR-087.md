# PR-087: Auto-queue integration in handle_idle

Branch: pr-087-auto-queue-idle
- Type: feature
- Complexity: medium
- Depends on: PR-086
- Priority: 2
- Coder: any

## Problem

handle_idle reads QUEUE.md via parse_queue() to find the next task. This requires operator to maintain QUEUE.md manually and creates race conditions when daemon updates statuses.

## Scope

Replace parse_queue(QUEUE.md) in handle_idle with: scan tasks/ directory, parse headers, derive statuses from git, build DAG, select eligible. Fall back to QUEUE.md if task files lack structured headers (backward compatibility).

## Changes

### src/daemon/handlers/idle.py

New method `_select_next_task_from_dag`:
```python
async def _select_next_task_from_dag(self) -> QueueTask | None:
    task_dir = Path(self.repo_path) / "tasks"
    if not task_dir.is_dir():
        return None
    headers = []
    for f in sorted(task_dir.glob("PR-*.md")):
        try:
            headers.append(parse_task_header(f))
        except QueueValidationError:
            continue  # skip files without structured headers
    if not headers:
        return None
    merged = get_merged_branches(self.repo_path, self.repo_config.branch)
    open_prs = {pr.branch for pr in github_client.get_open_prs(self.owner_repo)}
    statuses = {h.pr_id: derive_task_status(h, merged, open_prs) for h in headers}
    eligible = get_eligible_tasks(headers, statuses)
    if not eligible:
        return None
    picked = eligible[0]
    return QueueTask(
        pr_id=picked.pr_id,
        title=picked.title,
        status=TaskStatus.TODO,
        task_file=f"tasks/{picked.pr_id}.md",
        branch=picked.branch,
    )
```

In handle_idle: try `_select_next_task_from_dag()` first. If returns None (no eligible tasks or no structured headers), fall back to existing `get_next_task(parse_queue(...))`.

Update queue_done/queue_total from DAG counts when DAG is active.
