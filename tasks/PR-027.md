# PR-027: Subprocess helper and squash merge

Branch: pr-027-subprocess-helper

## Scope

Foundation PR: extract repeated subprocess.run patterns into a helper, switch merge strategy to squash.

## Changes

### src/daemon/runner.py

Add a module-level helper:
```python
def _git(repo_path: str, *args: str, timeout: int = 30) -> subprocess.CompletedProcess:
    return subprocess.run(
        ["git", *args],
        capture_output=True,
        text=True,
        timeout=timeout,
        check=True,
        cwd=repo_path,
    )
```

Replace all 23 bare subprocess.run(["git", ...]) calls in runner.py with _git(). Keep non-git subprocess calls (scripts/ci.sh, claude CLI) as-is.

### src/github_client.py

Change merge_pr to use --squash instead of --merge:
```python
run_gh(["pr", "merge", str(pr_number), "--squash", "--delete-branch"], repo=repo)
```

### tests/test_runner.py

Update test mocks if they assert on subprocess.run call args directly - the _git helper changes the call pattern.

### tests/test_github_client.py

Update test_merge_pr to expect --squash instead of --merge.

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- No bare subprocess.run(["git", ...]) left in runner.py
- merge_pr uses --squash
