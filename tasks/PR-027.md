# PR-027: Revert branch pre-creation and retry PR detection

Branch: pr-027-revert-and-retry

## Scope

Two fixes that eliminate the primary causes of "no PR found" errors.

1. Revert the branch pre-creation MICRO PR. The daemon's git checkout -B before Claude conflicts with AGENTS.md step 4 ("create branch from origin/main"). Claude owns the full git workflow: branch creation, commit, push, PR creation. Daemon should not interfere.

2. Add retry with delay on get_open_prs after handle_coding. GitHub API has eventual consistency - a PR created by Claude may not appear in the list for several seconds. Current code calls get_open_prs once and immediately errors if PR is missing. Add retry loop: 3 attempts, 5 seconds apart.

## Changes

### src/daemon/runner.py

In handle_coding:

1. Remove the entire block that does git checkout -B / git checkout before claude_cli.run_planned_pr (lines ~1145-1204). The method should go straight from target_branch validation to claude_cli.run_planned_pr. Keep the reset_branch parameter removal or make handle_coding a simple method again without branch logic.

2. Remove _commit_and_push_dirty call after run_planned_pr (line ~1216). Claude owns commit/push per AGENTS.md. If Claude didn't push, daemon should not second-guess.

3. Replace the single get_open_prs call (lines ~1220-1238) with a retry loop:
```python
candidate = None
for attempt in range(3):
    try:
        prs = github_client.get_open_prs(self.owner_repo)
    except Exception as exc:
        self.state.state = PipelineState.ERROR
        self.state.error_message = f"get_open_prs failed: {exc}"
        self.log_event(str(exc))
        return
    candidate = next(
        (pr for pr in prs if pr.branch == target_branch), None
    )
    if candidate is not None:
        break
    if attempt < 2:
        self.log_event(
            f"PR not found for {target_branch!r}, "
            f"retrying in 5s ({attempt + 1}/3)"
        )
        await asyncio.sleep(5)
```

4. In handle_fix: keep the git checkout of PR branch (Claude needs to be on the right branch for FIX REVIEW per AGENTS.md "stay on existing PR branch"). Remove _commit_and_push_dirty call after fix_review. Claude owns commit/push for fix too.

5. In recover_state where handle_coding is called with reset_branch=False: simplify to just call handle_coding() without branch logic since branch pre-creation is removed.

6. Add `import asyncio` at top if not already imported.
