# PR-029: Deduplicate codex review posts

Branch: pr-029-dedup-codex-review

## Scope

Claude posts @codex review as part of the AGENTS.md runbook. Daemon also posts @codex review via _post_codex_review after handle_coding and handle_fix. This creates duplicate review request comments on every PR.

Fix: before posting, check if a recent @codex review comment from PR author already exists (posted within last 5 minutes). If so, skip.

## Changes

### src/github_client.py

Add function:
```python
def has_recent_codex_review_request(
    repo: str, pr_number: int, pr_author: str, within_minutes: int = 5
) -> bool:
```
Fetch issue comments, find @codex review from pr_author, check if created_at is within within_minutes of now. Return True if found.

### src/daemon/runner.py

In _post_codex_review, before posting, call has_recent_codex_review_request. If True, log "skipping duplicate @codex review" and return True (success, no action needed).

### tests/test_github_client.py

- test_has_recent_codex_review_request_true: recent comment exists
- test_has_recent_codex_review_request_false: no recent comment

### tests/test_runner.py

- test_post_codex_review_skips_duplicate: mock has_recent returns True, verify no post_comment call

## Success criteria
- scripts/ci.sh exits 0
- No duplicate @codex review comments on PRs
- Daemon still posts if Claude did not (no recent comment)
