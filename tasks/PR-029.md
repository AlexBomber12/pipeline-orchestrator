# PR-029: Fix stale review detection

Branch: pr-029-stale-review

## Scope

Fix the Codex review gate to prevent merging with stale approvals. Currently, Codex puts +1 on PR body after first review. When new commits are pushed, the old +1 remains. Daemon sees APPROVED and merges without waiting for a new review of the latest code.

## Changes

### src/github_client.py

In get_pr_review_status, modify the PR body reactions check (lines ~119-136):

1. Fetch the PR's latest push timestamp. Add to get_open_prs JSON fields: `"updatedAt,commits"`. The last commit's `committedDate` or the PR's `updatedAt` gives us the latest push time.

2. When checking PR body reactions, compare the reaction's `created_at` against the latest push timestamp. If the +1 reaction is OLDER than the latest push, ignore it (stale approval).

3. Pass the last_push timestamp into get_pr_review_status as an optional parameter:
```python
def get_pr_review_status(
    repo: str,
    pr_number: int,
    pr_author: str = "",
    last_push_at: datetime | None = None,
) -> ReviewStatus:
```

4. In the body reactions block:
```python
if "+1" in codex_contents:
    if last_push_at is not None:
        plus_one_reaction = next(
            (r for r in issue_reactions
             if isinstance(r, dict)
             and r.get("content") == "+1"
             and "codex" in ((r.get("user") or {}).get("login", "")).lower()),
            None,
        )
        if plus_one_reaction:
            reaction_time = _parse_iso(plus_one_reaction.get("created_at"))
            if reaction_time and reaction_time < last_push_at:
                pass  # stale, fall through to comment-based checks
            else:
                return ReviewStatus.APPROVED
    else:
        return ReviewStatus.APPROVED
```

5. In get_open_prs, extract the latest commit date from the commits array and pass it to get_pr_review_status:
```python
commits = entry.get("commits") or []
last_push_at = None
if commits:
    last_commit = commits[-1]
    committed = (last_commit.get("committedDate")
                 or last_commit.get("authoredDate"))
    if committed:
        last_push_at = _parse_iso(committed)
```

### tests/test_github_client.py

- test_fresh_approval_accepted: +1 reaction created_at after last commit date, verify returns APPROVED
- test_approval_without_push_date: no last_push_at provided, verify +1 returns APPROVED (backward compatible)
