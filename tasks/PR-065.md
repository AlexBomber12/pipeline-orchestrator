# PR-065: Codex CLI as alternative coder with manual selection

Branch: pr-065-codex-coder

- Type: feature
- Complexity: medium

## Problem

The daemon has a single coder: Claude Code (Opus). When Claude Max rate limits hit, the daemon pauses and waits. Meanwhile an OpenAI ChatGPT subscription sits idle. Codex CLI (OpenAI's terminal coding agent, built in Rust) can perform the same PLANNED PR and FIX REVIEW workflows by reading the same AGENTS.md, but the daemon has no way to invoke it.

Secondary benefit: Codex CLI runs in Rust, not Node.js, so the upstream Claude Code memory leak (anthropics/claude-code#4953) does not apply. Running sprint PRs through Codex CLI exercises a different runtime and validates that the AGENTS.md instructions are truly coder-agnostic.

## Terminology

The project uses "Codex" in 2 contexts that must be clearly separated:

- **Codex CLI**: OpenAI's terminal coding agent (`codex exec`). Used as an alternative CODER.
- **Codex Review**: The GitHub bot invoked via `@codex review` comments. Used as the REVIEWER.

All code, logs, UI labels, and documentation must use these exact terms. The daemon log must prefix coder identity: `[claude]` or `[codex]` in event messages related to CODING and FIX.

## Scope

Add Codex CLI as a second coder option. Selection is manual: a config field (global default + per-repo override) and a UI dropdown on the settings page. No automatic fallback or routing. The operator switches between coders explicitly.

### src/codex_cli.py (new)

Mirror the structure of `claude_cli.py`. Key differences:

```python
async def run_codex_async(
    prompt: str,
    cwd: str,
    timeout: int | None = 600,
    model: str | None = None,
) -> tuple[int, str, str]:
    cmd = [
        "codex", "exec",
        "--full-auto",
    ]
    if model:
        cmd.extend(["--model", model])
    cmd.append(prompt)
    logger.info("[codex] running codex exec with prompt: %s", prompt[:80])
    proc = await asyncio.create_subprocess_exec(
        *cmd,
        stdout=asyncio.subprocess.PIPE,
        stderr=asyncio.subprocess.PIPE,
        cwd=cwd,
        stdin=asyncio.subprocess.DEVNULL,
    )
    stdout_bytes, stderr_bytes = await asyncio.wait_for(
        proc.communicate(), timeout=timeout
    )
    stdout = stdout_bytes.decode("utf-8", errors="replace")
    stderr = stderr_bytes.decode("utf-8", errors="replace")
    code = proc.returncode or 0
    logger.info("[codex] codex exec exited with code %s", code)
    return (code, stdout, stderr)


async def run_planned_pr_async(
    repo_path: str, model: str | None = None, timeout: int = 900
) -> tuple[int, str, str]:
    return await run_codex_async(
        "PLANNED PR", repo_path, timeout=timeout, model=model
    )


async def fix_review_async(
    repo_path: str, model: str | None = None, timeout: int | None = None
) -> tuple[int, str, str]:
    return await run_codex_async(
        "FIX REVIEW", repo_path, timeout=timeout, model=model
    )
```

Notes:
- No `NODE_OPTIONS` (Rust binary, no V8 heap)
- No `--print` flag (Codex exec already writes final message to stdout, progress to stderr)
- No `--dangerously-skip-permissions` (Codex uses `--full-auto` instead)
- No `--append-system-prompt-file` (Codex reads AGENTS.md from repo root natively)
- `diagnose_error_async` not ported for now (AI diagnosis uses Claude; Codex is only for PLANNED PR and FIX REVIEW)

### src/daemon/runner.py

Add a method to resolve the active coder and return the correct CLI module:

```python
def _get_coder(self) -> tuple[str, ModuleType]:
    coder = self.repo_config.coder or self.app_config.daemon.coder
    if coder == CoderType.CODEX:
        return "codex", codex_cli
    return "claude", claude_cli
```

Update `handle_coding`:

```python
async def handle_coding(self) -> None:
    if not self._check_rate_limit():
        return
    coder_name, coder_module = self._get_coder()
    ...
    heartbeat = asyncio.create_task(self._publish_while_waiting("CODING"))
    try:
        model = (
            self.app_config.daemon.codex_model
            if coder_name == "codex"
            else self.app_config.daemon.claude_model
        )
        code, stdout, stderr = await coder_module.run_planned_pr_async(
            self.repo_path,
            model=model,
            timeout=self.app_config.daemon.planned_pr_timeout_sec,
        )
    finally:
        heartbeat.cancel()
    await self._save_cli_log(stdout, stderr, f"PLANNED PR output [{coder_name}]")
    self._detect_rate_limit(stderr)
    ...
```

Same change in `handle_fix`: resolve coder, call the matching module, log with coder identity.

Event log messages must include coder identity:

```python
self.log_event(f"[{coder_name}] Picked task {task.pr_id}: {task.title}")
self.log_event(f"[{coder_name}] claude/codex CLI failed: ...")
```

Rate limit handling per coder:
- `coder == "claude"`: existing `_detect_rate_limit` + proactive OAuth check (PR-063) apply
- `coder == "codex"`: `_detect_rate_limit` still runs (parse stderr for generic "rate limit" / "429"), but proactive OAuth check is skipped (different provider, different endpoint). Codex rate limit format in stderr is unknown; rely on exit code + generic string match for now

## Risks

- Codex CLI may interpret AGENTS.md differently from Claude Code. Mitigation: manual test run before enabling in daemon. If Codex misinterprets trigger phrases, add a Codex-specific section to AGENTS.md (low effort).
- Codex rate limit format in stderr is unknown. Mitigation: generic "rate limit" / "429" string match works as fallback. Once observed, add Codex-specific regex to `_detect_rate_limit`.
- `codex exec --full-auto` grants Codex full file and command access in the repo. Same trust level as `claude --dangerously-skip-permissions`. Acceptable for a self-hosted daemon on a private server.
- Codex CLI updates may change exec flags. `--full-auto` is documented and stable. Pin Codex version in Dockerfile install if stability is a concern.
- Auth persistence: `~/.codex/auth.json` contains OAuth tokens. Docker volume `codex-auth` must be treated as sensitive. Same policy as Claude auth volume.
- If Codex CLI is not installed or not authenticated, selecting "codex" as coder will fail every CODING/FIX attempt with FileNotFoundError. The auth status check on settings page prevents silent misconfiguration.
