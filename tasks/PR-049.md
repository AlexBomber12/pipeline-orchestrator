# PR-049: Refresh PR branch from origin before FIX

Branch: pr-049-fix-branch-refresh

- Type: bugfix
- Complexity: low

## Problem

handle_fix does `git checkout <pr_branch>` before running Claude CLI, but does not sync the branch from origin first. If the remote branch has moved forward (Codex made a commit, another operator pushed, or a previous daemon instance pushed and the local state is stale), Claude runs against outdated code.

Consequences:
- Rejected pushes when Claude tries to push because remote is ahead
- Claude fixes issues on stale code that are already resolved upstream
- Merge conflicts on push that could have been avoided
- Inaccurate review-response behavior (fixing feedback that references newer lines)

## Root cause

`src/daemon/runner.py:1613-1636` performs `git checkout self.state.current_pr.branch` but no `git fetch` or `git reset --hard origin/<branch>`. The local branch may be behind origin.

## Scope

Before invoking fix_review_async, ensure the local branch matches origin. Use fetch + hard reset to force the local state to exactly origin/<branch>, discarding any local divergence.

## Changes

### src/daemon/runner.py

In `handle_fix` around lines 1609-1636, replace the single checkout with a refresh sequence:

```python
if (
    self.state.current_pr is not None
    and not self.state.current_pr.is_cross_repository
):
    branch = self.state.current_pr.branch
    try:
        _git(self.repo_path, "fetch", "origin", branch)
        _git(self.repo_path, "checkout", branch)
        _git(self.repo_path, "reset", "--hard", f"origin/{branch}")
    except (
        subprocess.CalledProcessError,
        subprocess.TimeoutExpired,
        OSError,
    ) as exc:
        stderr = getattr(exc, "stderr", "") or ""
        self.state.state = PipelineState.ERROR
        self.state.error_message = (
            f"git refresh {branch} failed: {stderr.strip() or exc}"
        )
        self.log_event(self.state.error_message)
        return
```

The fetch + reset sequence guarantees:
1. Local branch ref exists (creates if missing)
2. Local branch matches remote exactly
3. Working tree matches the latest remote state

Cross-repo (fork) PRs still skip this block for the same reason as before: the head branch lives on the contributor's fork, not origin.

### tests/test_runner.py

Add tests:
- test_handle_fix_fetches_and_resets_branch_before_run: mock _git to record calls, verify fetch -> checkout -> reset sequence
- test_handle_fix_error_on_fetch_failure: mock fetch to raise CalledProcessError, expect state = ERROR with message containing "git refresh"
- test_handle_fix_error_on_reset_failure: mock reset to raise, expect ERROR
- test_handle_fix_skips_refresh_for_fork_pr: cross-repo PR, expect no fetch/checkout/reset calls (existing behavior preserved)

Existing handle_fix tests may need mock updates since checkout is now one of three git calls instead of standalone.

- New tests pass

- Before fix_review_async runs, local PR branch is synchronized with origin
- If any git operation fails, runner enters ERROR with a clear message identifying the failed step
