# PR-011: Repo scaffolding on connect

Branch: pr-011-scaffolding

## Scope

When a new repository is added to config.yml and the daemon clones it for the first time, check if the required pipeline structure exists. If not, create it from templates bundled with the orchestrator.

## Files to create

### templates/AGENTS.md

Copy the current AGENTS.md from the root of this project. This serves as the template for new repos. If AGENTS.md already exists in a target repo, do not overwrite.

### templates/QUEUE.md

Minimal template:
```
# Task Queue
```

Just the header, no example tasks.

### templates/ci.sh

Minimal CI script template:
```
#!/usr/bin/env bash
set -euo pipefail
cd "$(dirname "$0")/.."
echo "==> No CI checks configured yet"
```

### templates/make-review-artifacts.sh

Copy of the current scripts/make-review-artifacts.sh from this project.

### src/daemon/scaffolder.py

Import: pathlib, shutil, logging, subprocess

TEMPLATES_DIR = Path(__file__).resolve().parent.parent.parent / "templates"

scaffold_repo(repo_path: str) -> list[str]
- repo_path is the local clone path
- Returns list of actions taken (for logging)
- Check and create if missing:
  - AGENTS.md (or CLAUDE.md - check if either exists, if neither, copy templates/AGENTS.md as AGENTS.md)
  - tasks/ directory
  - tasks/QUEUE.md (only if tasks/ was just created or QUEUE.md missing)
  - scripts/ directory
  - scripts/ci.sh (only if missing, make executable)
  - scripts/make-review-artifacts.sh (only if missing, make executable)
  - artifacts/ directory
  - .gitignore entry for artifacts/ (append if not present, do not overwrite)
- If any files were created:
  - git add the new files
  - git commit -m "chore: initialize pipeline orchestrator structure"
  - git push origin {branch}
- Return list of created file paths

### Integrate into runner

Modify PipelineRunner.ensure_repo_cloned:
- After cloning (not after fetch), call scaffolder.scaffold_repo(self.repo_path)
- Log the actions taken

### tests/test_scaffolder.py

- Test scaffold_repo creates all files when repo is empty (use tmp_path as repo, init git repo in it)
- Test scaffold_repo does not overwrite existing AGENTS.md
- Test scaffold_repo does not overwrite existing QUEUE.md
- Test scaffold_repo does not overwrite existing scripts/ci.sh
- Test scaffold_repo skips commit when nothing was created
- Test scaffold_repo appends artifacts/ to .gitignore without duplicating

Mock git add/commit/push via monkeypatch on subprocess.run.

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- templates/ directory contains all 4 template files
- scaffold_repo is idempotent (running twice produces no changes on second run)
