# PR-126: Remove hardcoded claude_cli from error and merge handlers

Branch: pr-126-eligible-coder-selection
- Type: bugfix
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Two critical paths in the daemon hardcode `claude_cli` as the coder, regardless of which coder is eligible:

1. `src/daemon/handlers/error.py:143` - `diagnose_error_async` is always called on `claude_cli`. If the user has no Claude subscription or Claude is rate-limited, self-heal from ERROR state breaks entirely. The daemon cannot diagnose its own failures, and ERROR accumulates until manual intervention.

2. `src/daemon/handlers/merge.py:78` - `run_claude_async` is always called for merge-conflict resolution. If the user is Codex-only or Claude is rate-limited, merge conflicts never get resolved and the PR is stuck indefinitely.

Both call sites bypass the eligible-coder selection logic used during normal CODING, so the whole routing architecture is broken for users who do not have Claude.

## Scope

Replace the hardcoded `claude_cli` calls at both sites with dynamic coder selection that respects the existing eligibility filters (auth status, rate-limit state, user disabled_coders config).

Use the existing coder selector infrastructure. The selection should:
- Prefer Claude if eligible (for continuity with current behavior when Claude is available)
- Fall back to Codex if Claude is not eligible
- If no coder is eligible, surface a clear log message and keep the daemon in ERROR without trying to self-heal (do not silently swallow the problem)

Do not add new config flags for this - the existing selector fallback policy (introduced in PR-091) is sufficient. Just route `diagnose_error` and `merge-conflict-resolution` through it instead of hardcoding claude.

## Files to modify

- `src/daemon/handlers/error.py`: replace the `claude_cli.diagnose_error_async` call with a call through the selector. Preserve all existing error-handling logic (rate-limit detection, retry counter, verdict parsing) - only the coder identity changes.
- `src/daemon/handlers/merge.py`: replace the `claude_cli.run_claude_async` conflict-resolution call similarly.
- `src/daemon/selector.py` or equivalent: if needed, expose a method like `select_for_auxiliary_task()` that returns the best eligible coder for diagnose/merge-conflict work (distinct from `select_for_coding()` which drives the bandit).
- Tests for both handlers: add cases covering (a) Claude eligible => routes through Claude, (b) Claude unavailable + Codex eligible => routes through Codex, (c) no coder eligible => logged and daemon stays in ERROR.

- New tests cover all three branches (claude eligible, codex fallback, no eligible coder).
