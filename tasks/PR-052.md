# PR-052: Empty CI rollup returns PENDING, not SUCCESS

Branch: pr-052-empty-rollup-pending

- Type: bugfix
- Complexity: low

## Problem

`_ci_status_from_rollup` in github_client.py returns `CIStatus.SUCCESS` when the statusCheckRollup list is empty. But an empty rollup can mean:
- no checks configured on the repo (misconfiguration)
- checks haven't started yet (race with PR creation)
- API returned partial response
- required checks are missing

Combined with auto_merge=true, this default can auto-merge PRs that have no CI protection. Unsafe.

## Root cause

`src/github_client.py:649-654`:
```python
def _ci_status_from_rollup(rollup: object) -> CIStatus:
    if not isinstance(rollup, list):
        return CIStatus.PENDING
    if not rollup:
        return CIStatus.SUCCESS  # <- unsafe default
    ...
```

## Scope

Change the default for empty rollup from SUCCESS to PENDING. Add a per-repo config flag `allow_merge_without_checks` (default False) that restores the old behavior for repos intentionally running without CI.

## Changes

### src/github_client.py

Accept a parameter for the safe default:

```python
def _ci_status_from_rollup(
    rollup: object,
    empty_is_success: bool = False,
) -> CIStatus:
    if not isinstance(rollup, list):
        return CIStatus.PENDING
    if not rollup:
        return CIStatus.SUCCESS if empty_is_success else CIStatus.PENDING
    ...
```

Update the caller (around line where _ci_status_from_rollup is used in get_open_prs) to read the repo config and pass the flag through.

### src/config.py and src/models.py

Add `allow_merge_without_checks: bool = False` to RepoConfig. Default False preserves the safe behavior.

### src/web/app.py and templates

Add a checkbox to the Settings page for each repo labeled "Allow merge without CI checks (unsafe)". Small helper text: "Only enable for repos intentionally running without CI. Default off."

### tests/test_github_client.py

Add tests:
- test_ci_status_empty_rollup_defaults_to_pending: empty list, default flag, expect PENDING
- test_ci_status_empty_rollup_with_flag_returns_success: empty list, empty_is_success=True, expect SUCCESS
- test_ci_status_non_list_returns_pending: None or dict, expect PENDING (unchanged)

### tests/test_config.py

Add tests for the new flag loading and default value.

### tests/test_settings.py

Add a test verifying the checkbox renders and saves the config correctly.

## Success criteria
- scripts/ci.sh exits 0
- New tests pass
- All existing tests pass (may need one update for the CI status default change)
- PR with empty statusCheckRollup returns PENDING by default, preventing auto-merge
- Config flag `allow_merge_without_checks: true` restores the old behavior per repo
- Settings page exposes the flag with clear labeling
- Default is OFF (safe)
