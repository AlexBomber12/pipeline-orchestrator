# PR-041: Smart error handling and stderr logging

Branch: pr-041-smart-errors

## Scope

3 improvements to error handling and diagnostics:

1. Save stderr alongside stdout in CLI logs. Currently only stdout is saved, but stderr often contains the most valuable diagnostic info (auth errors, rate limits, crash traces).

2. Guard handle_fix against stale reviews. Don't trigger 30-minute FIX REVIEW when there's no new Codex feedback after the last push.

3. Improve handle_error semantics. Current FIX verdict just clears error and returns to IDLE without actually fixing anything. Add backoff and cap max attempts.

Add helper method `_has_new_codex_feedback_since_last_push`:
```python
def _has_new_codex_feedback_since_last_push(self) -> bool:
    if self.state.current_pr is None:
        return False
    last_activity = self.state.current_pr.last_activity
    if last_activity is None:
        return True  # no push recorded, treat as new
    try:
        comments = github_client._gh_api_paginated(
            f"repos/{self.owner_repo}/issues/{self.state.current_pr.number}/comments"
        ) or []
    except Exception:
        return True  # can't check, err on side of fixing
    for c in reversed(comments):
        user = (c.get("user") or {}).get("login", "")
        if "codex" not in user.lower():
            continue
        body = c.get("body") or ""
        if "P1" not in body and "P2" not in body:
            continue
        created = github_client._parse_iso(c.get("created_at"))
        if created and last_activity.tzinfo is None:
            last_activity = last_activity.replace(tzinfo=timezone.utc)
        if created and created > last_activity:
            return True
    return False
```

4. In handle_error, add backoff and max attempts:

Add to PipelineRunner.__init__:
```python
self._error_diagnose_count = 0
self._error_diagnose_backoff = 1  # cycles to skip
```

In handle_error:
```python
async def handle_error(self, error_context: str | None = None) -> None:
    self._error_diagnose_count += 1
    if self._error_diagnose_count > 3:
        self.log_event("diagnose_error: max attempts reached, staying ERROR")
        return
    if self._error_diagnose_count > 1:
        self.log_event(
            f"diagnose_error: attempt {self._error_diagnose_count}/3, "
            f"backoff {self._error_diagnose_backoff} cycles"
        )
        self._error_diagnose_backoff *= 2
        return  # skip this cycle, try next
    # ... rest of existing logic
```

Reset counters when leaving ERROR:
```python
# In any transition from ERROR to IDLE/WATCH:
self._error_diagnose_count = 0
self._error_diagnose_backoff = 1
```

5. Skip fork PRs in handle_fix entirely (not just checkout/push):
