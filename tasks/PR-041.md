# PR-041: Smart error handling, stderr logging, FIX timeout, and stale review fix

Branch: pr-041-smart-errors

## Scope

6 improvements:

1. Fix stale review detection false negative. Body +1 is ignored when latest Codex review SHA != head SHA, even though Codex put +1 AFTER seeing the latest push. Compare reaction time vs push time, not review SHA vs head SHA.

2. Save stderr alongside stdout in CLI logs.

3. Guard handle_fix against stale reviews. Don't trigger FIX when no new Codex feedback after last push.

4. Improve handle_error: backoff, max 3 attempts, skip diagnose for timeout/rate-limit.

5. Configurable fix_review and planned_pr timeouts via config.yml AND Settings UI.

6. Skip fork PRs in handle_fix.

## Changes

### src/github_client.py

This is the critical fix. In `get_pr_review_status`, the body reactions block (around lines 130-145) currently does:

```python
if "+1" in codex_contents:
    if head_sha:
        reviewed_sha = _get_latest_codex_reviewed_sha(repo, pr_number)
        if reviewed_sha and not head_sha.startswith(reviewed_sha) and not reviewed_sha.startswith(head_sha):
            pass  # stale: reviewed a different commit
        else:
            body_approved = True
    else:
        body_approved = True
```

Problem: Codex puts +1 on PR body when it approves the CURRENT state. But `_get_latest_codex_reviewed_sha` returns the commit_id from the last formal review object, which may be from an earlier push. After Claude pushes fix #13, Codex may react with +1 on body (approving current state) without creating a new formal review record. The SHA comparison then sees "old review SHA != new head SHA" and rejects a valid approval.

Replace with time-based comparison. Find the +1 reaction's `created_at` and compare against the head commit's committer date:

```python
if "+1" in codex_contents:
    if head_sha:
        plus_one = _find_codex_plus_one_reaction(issue_reactions)
        if plus_one:
            reaction_time = _parse_iso(plus_one.get("created_at"))
            head_commit_time = _get_commit_time(repo, head_sha)
            if reaction_time and head_commit_time and reaction_time > head_commit_time:
                body_approved = True
            elif not head_commit_time:
                body_approved = True  # can't verify, trust +1
            # else: reaction is older than head commit, stale
        else:
            body_approved = True  # +1 exists in set but no individual match, trust it
    else:
        body_approved = True
```

Add helpers:

```python
def _find_codex_plus_one_reaction(reactions: list[dict]) -> dict | None:
    """Return the most recent +1 reaction from a Codex user, or None."""
    best = None
    for r in reactions:
        if not isinstance(r, dict):
            continue
        if "codex" not in ((r.get("user") or {}).get("login", "")).lower():
            continue
        if r.get("content") != "+1":
            continue
        if best is None or (r.get("created_at") or "") > (best.get("created_at") or ""):
            best = r
    return best

def _get_commit_time(repo: str, sha: str) -> datetime | None:
    """Return the committer date of a commit, or None on failure."""
    try:
        raw = run_gh(
            ["api", f"repos/{repo}/commits/{sha}", "--jq", ".commit.committer.date"]
        )
    except RuntimeError:
        return None
    return _parse_iso(raw.strip() if isinstance(raw, str) else "")
```

Remove `_get_latest_codex_reviewed_sha` function entirely. It is no longer used after this change.

Add helper:
```python
def _has_new_codex_feedback_since_last_push(self) -> bool:
    if self.state.current_pr is None:
        return False
    last_activity = self.state.current_pr.last_activity
    if last_activity is None:
        return True
    if last_activity.tzinfo is None:
        last_activity = last_activity.replace(tzinfo=timezone.utc)
    try:
        comments = github_client._gh_api_paginated(
            f"repos/{self.owner_repo}/issues/"
            f"{self.state.current_pr.number}/comments"
        ) or []
        review_comments = github_client._gh_api_paginated(
            f"repos/{self.owner_repo}/pulls/"
            f"{self.state.current_pr.number}/comments"
        ) or []
    except Exception:
        return True
    for c in reversed(comments + review_comments):
        user = (c.get("user") or {}).get("login", "")
        if "codex" not in user.lower():
            continue
        body = c.get("body") or ""
        if "P1" not in body and "P2" not in body:
            continue
        created = github_client._parse_iso(c.get("created_at"))
        if created is None:
            continue
        if created.tzinfo is None:
            created = created.replace(tzinfo=timezone.utc)
        if created > last_activity:
            return True
    return False
```
