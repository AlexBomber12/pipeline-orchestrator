# PR-078: Unified coder settings table + repo card coder selector

Branch: pr-078-coder-settings-table
- Type: feature
- Complexity: medium
- Depends on: PR-077
- Priority: 2
- Coder: any

## Problem

Three separate issues:
1. Coder settings scattered: Coder dropdown in Daemon Settings, Codex model as text field, Auth Status as separate section with 3 cards. Should be one unified table.
2. Switching coder requires navigating to Settings page. Should be accessible from repo card or repo detail page with one click.
3. Changing coder via Settings UI requires stop/start because runner.repo_config is not hot-reloaded when config changes. runner.app_config has hot-reload via setter, but repo_config does not.

## Scope

1. Unified coder settings table in Settings page (replaces separate coder/model/auth sections).
2. Coder dropdown on repo detail page and optionally on repo card.
3. Hot-reload repo_config in daemon main loop so coder switch applies on next poll cycle without stop/start.

## Changes

### src/daemon/main.py

In the hot-reload path (where app_config is reloaded and runner.app_config is updated), also update runner.repo_config:

```python
for runner in runners:
    new_repo_config = find_repo_config(new_app_config, runner.repo_config.url)
    if new_repo_config is not None:
        runner.repo_config = new_repo_config
```

This makes all repo-level config changes (coder, review_timeout_min, auto_merge, poll_interval_sec) take effect on the next poll cycle without restart.

### src/web/templates/components/settings_coders.html (new)

Unified table replacing separate CODER section and Auth Status section:

```
CODERS
+------------------+-------------------+--------------+----------+
| Coder            | Model             | Auth         | Default  |
+------------------+-------------------+--------------+----------+
| Claude Code      | opus         [v]  | AUTHORIZED   | (*)      |
| Codex CLI        | (default)    [v]  | AUTHORIZED   | ( )      |
+------------------+-------------------+--------------+----------+
```

Each row populated from plugin registry. Model column is dropdown from plugin.models. Auth column shows status badge. Default column is radio group for daemon-level default coder.

GitHub CLI stays as separate infra item below the table (it is not a coder).

### src/web/templates/components/settings_daemon.html

Remove CODER section (moved to settings_coders.html). Remove standalone Codex model field.

### src/web/templates/settings.html

Replace Auth Status section with `{% include "components/settings_coders.html" %}`.

### src/web/templates/repo.html (or components/repo_summary.html)

Add coder dropdown on repo detail page header, next to the state badge:

```html
<select name="coder"
        hx-put="/settings/repo/{{ repo.name }}"
        hx-trigger="change"
        hx-target="..."
        hx-swap="...">
    <option value="">Inherit ({{ daemon_coder }})</option>
    <option value="claude" {% if repo_config.coder == "claude" %}selected{% endif %}>Claude Code</option>
    <option value="codex" {% if repo_config.coder == "codex" %}selected{% endif %}>Codex CLI</option>
</select>
```

Change takes effect on next daemon poll cycle (via hot-reload). No stop/start needed.
