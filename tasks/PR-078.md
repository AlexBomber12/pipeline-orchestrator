# PR-078: Unified coder settings table

Branch: pr-078-coder-settings-table
- Type: feature
- Complexity: medium
- Depends on: PR-077
- Priority: 3
- Coder: any

## Problem

Coder settings scattered: Coder dropdown in Daemon Settings, Codex model as text field, Auth Status as separate section with 3 cards. Should be unified table where each coder is one row.

## Scope

Replace separate coder/model/auth UI sections with one "Coders" table populated from plugin registry.

## Changes

### src/web/app.py

New endpoint `GET /api/coders` returning JSON list of registered coders with auth status and current config. Used by settings template.

### src/web/templates/components/settings_coders.html (new)

Table with columns: Coder (display_name), Model (dropdown), Auth (status badge), Active (radio).
- Model dropdown populated from plugin.models
- Auth badge: green AUTHORIZED / red ERROR with detail
- Active: radio group, only one active at daemon level
- Per-repo override shown in repo list (already exists)

### src/web/templates/settings.html

Replace CODER section in settings_daemon.html and Auth Status section with single `{% include "components/settings_coders.html" %}`. Keep GitHub CLI as separate infra item below.

### tests/test_settings.py

- test_coders_table_renders_all_plugins
- test_coders_table_shows_auth_status

## Success criteria
- scripts/ci.sh exits 0
- All registered coders visible in one table
- Model, auth, active selection in same row
- GitHub CLI shown separately
- All tests pass
