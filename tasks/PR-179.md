# PR-179: Pulse badge replaces pulse dot for active states

Branch: pr-179-pulse-badge-not-dot
- Type: ux
- Complexity: low
- Depends on: PR-178
- Priority: 3
- Coder: any

## Problem

Active states (CODING, WATCH, FIX, MERGE) currently show a tiny 10px pulsing dot next to the state badge text. The dot is small, easy to miss, and visually disconnected from the badge it's adjacent to. Users glancing at the dashboard sometimes don't notice that a repo is actively working — the dot is too subtle.

The fix is to apply the pulse animation to the **state badge itself** (the rounded rectangle showing "CODING" / "WATCH" / etc.), not to a separate dot. The whole badge gently pulses when the repo is in an active state. This is significantly more visible and intuitively communicates "this thing is alive and working."

## Scope

Replace the pulse-dot pattern with pulse-badge pattern. State badge gets a subtle pulse animation (e.g., border opacity oscillation or background subtle brighten/dim) when in active state. Pulse dot removed.

This PR DEPENDS on PR-178 (smooth transition-based pulse) so the badge pulse uses the same transition mechanism, not keyframes. Without PR-178 first, badge pulse would have the same jump issue.

## Files to touch

`src/web/templates/base.html`: add new CSS class `.pulse-badge` similar in spirit to `.pulse-dot` but applied to a badge container (rounded rect with text). The animation parameters:

```css
.pulse-badge {
    transition: box-shadow 1s ease-in-out, opacity 1s ease-in-out;
}

.pulse-badge.dim {
    box-shadow: 0 0 0 0 rgba(255, 165, 0, 0); /* shadow at 0 */
    opacity: 0.85;
}

/* When NOT dim: */
.pulse-badge {
    box-shadow: 0 0 8px 2px rgba(255, 165, 0, 0.4); /* warm glow */
    opacity: 1;
}
```

Reuse the same global timer from PR-178 (toggle `.dim` class every 1000ms).

`src/web/templates/components/repo_cards.html`, `repo_summary.html`, `repo_list.html`: where `pulse-dot` is currently applied to a separate `<span>`, change to apply `pulse-badge` class to the **state badge container** instead. Remove the now-redundant pulse dot span.

Sketch for repo_cards.html line 70:


After:

Apply consistently in all three templates (cards, summary, list).
