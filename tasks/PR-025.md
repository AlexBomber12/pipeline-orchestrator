# PR-025: Start stop controls per repo

Branch: pr-025-start-stop

## Scope

Add per-repo Start/Stop controls. When stopped (paused), daemon skips the repo in its poll loop. When started, daemon resumes normal operation. This prevents conflicts when user is working manually on the same repo.

## Design

Add an `active` boolean to RepoConfig. Default: true. When false, daemon skips run_cycle for this repo.

Dashboard shows:
- "Start" button when repo is inactive (paused)
- "Stop" button when repo is active (running)

Stopping a repo mid-cycle: daemon finishes current cycle, then stops polling. It does NOT interrupt a running claude CLI process.

## Changes

### src/config.py

Add to RepoConfig:
```python

```

### src/daemon/main.py

In the poll loop, check runner.repo_config.active before calling run_cycle:
```python
for name, runner in runners.items():
    if not runner.repo_config.active:
        continue
    try:
        await runner.run_cycle()
    ...
```

On config reload (_sync_runners): update active flag on existing runners when config changes.

### src/daemon/runner.py

```python
active: bool = True
```

In publish_state, include self.repo_config.active in the state.

### src/models.py

### src/web/app.py
