# PR-081: Capture metrics in coding handler

Branch: pr-081-coding-metrics
- Type: feature
- Complexity: low
- Depends on: PR-080
- Priority: 2
- Coder: any

## Problem

No metrics captured during CODING. Cannot measure coding duration, which coder was used, or exit reason.

## Scope

Create RunRecord at CODING start, update on completion, save to MetricsStore.

## Changes

### src/daemon/handlers/coding.py

At start of handle_coding (after rate limit check):
```python
run_record = RunRecord(
    run_id=str(uuid.uuid4()),
    task_id=self.state.current_task.pr_id,
    profile_id=f"{coder_name}:{model}:container",
    task_type=...,  # parse from task file if available
    complexity=...,
    started_at=datetime.now(timezone.utc).isoformat(),
    ...defaults...
)
```

On success (PR found): `run_record.exit_reason = "coding_complete"`, save.
On error: `run_record.exit_reason = "error"`, save.
On rate limit: `run_record.exit_reason = "rate_limit"`, save.

Store `self._current_run_record` on runner for fix/merge handlers to update later.

### src/daemon/runner.py

Add `self._current_run_record: RunRecord | None = None` and `self._metrics_store: MetricsStore` to __init__.

### tests/

- test_coding_creates_run_record
- test_coding_saves_record_on_success
- test_coding_saves_record_on_error
