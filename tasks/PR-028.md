# PR-028: QUEUE.md auto-update on merge

Branch: pr-028-queue-auto-done

## Scope

After successful PR merge in handle_merge, daemon updates QUEUE.md Status from TODO/DOING to DONE, commits and pushes to main. Claude sometimes skips this step, leaving the daemon to re-pick already-merged tasks.

## Changes

### src/daemon/runner.py

In handle_merge, after successful github_client.merge_pr:

1. git fetch origin {branch} and git checkout {branch} and git reset --hard origin/{branch}
2. Read tasks/QUEUE.md
3. Find the current task by pr_id
4. Replace "- Status: TODO" or "- Status: DOING" with "- Status: DONE" for that task only
5. git add tasks/QUEUE.md, git commit -m "{pr_id}: mark DONE", git push origin {branch}
6. Log the update
7. If any step fails, log warning but do NOT set ERROR state - the PR is already merged, this is a best-effort cleanup

Use _git helper from PR-027.

### tests/test_runner.py

Add tests:
- test_handle_merge_marks_queue_done: mock merge_pr success, verify QUEUE.md is updated
- test_handle_merge_tolerates_queue_update_failure: mock git push failure after merge, verify state is IDLE not ERROR

## Success criteria
- scripts/ci.sh exits 0
- After merge, QUEUE.md shows DONE for the merged task
- Queue update failure does not block IDLE transition
