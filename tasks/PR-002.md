# PR-002: Config loader and data models

Branch: pr-002-models

## Scope

Create the config loader (reads config.yml) and all data models used by the daemon and dashboard.

## Files to create

### src/config.py
Pydantic models for configuration:
- RepoConfig: url (str), branch (str, default "main"), auto_merge (bool, default True), review_timeout_min (int, default 60), poll_interval_sec (int, default 60)
- DaemonConfig: poll_interval_sec (int, default 60), review_timeout_min (int, default 60), hung_fallback_codex_review (bool, default True), error_handler_use_ai (bool, default True)
- WebConfig: host (str, default "0.0.0.0"), port (int, default 8000)
- AuthConfig: claude_config_dir (str, default "/data/auth/claude"), gh_config_dir (str, default "/data/auth/gh")
- AppConfig: repositories (list[RepoConfig], default []), daemon (DaemonConfig), web (WebConfig), auth (AuthConfig)
- load_config(path: str = "config.yml") -> AppConfig: reads YAML file, returns AppConfig. If file missing, return defaults.

### src/models.py
Enums (all str, Enum):
- PipelineState: PREFLIGHT, IDLE, CODING, WATCH, FIX, MERGE, HUNG, ERROR
- TaskStatus: TODO, DOING, DONE
- ReviewStatus: PENDING, EYES, APPROVED, CHANGES_REQUESTED
- CIStatus: PENDING, SUCCESS, FAILURE

Pydantic models:
- QueueTask: pr_id (str), title (str), status (TaskStatus), task_file (str | None), depends_on (list[str], default []), branch (str | None)
- PRInfo: number (int), branch (str), ci_status (CIStatus, default PENDING), review_status (ReviewStatus, default PENDING), push_count (int, default 0), url (str, default ""), last_activity (datetime | None)
- RepoState: url (str), name (str), state (PipelineState, default IDLE), current_task (QueueTask | None), current_pr (PRInfo | None), error_message (str | None), last_updated (datetime), history (list[dict], default [])

### tests/test_config.py
- Test load_config with missing file returns defaults
- Test load_config with valid YAML returns correct values
- Test RepoConfig defaults

### tests/test_models.py
- Test PipelineState enum values
- Test TaskStatus enum values
- Test QueueTask creation with defaults
- Test RepoState serialization/deserialization (model_dump_json / model_validate_json)

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- load_config("nonexistent.yml") returns valid AppConfig with empty repositories
- RepoState round-trips through JSON serialization
