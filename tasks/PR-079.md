# PR-079: Model dropdown validated from plugin

Branch: pr-079-model-dropdown
- Type: feature
- Complexity: low
- Depends on: PR-078
- Priority: 3
- Coder: any

## Problem

Codex model was a free-text input. User typed "o3" which does not exist. Need validated dropdown from plugin.models list.

## Scope

Model dropdown in coder settings table populated from `plugin.models`. First entry "(default)" sends empty string. Web endpoint validates model on save.

## Changes

### src/web/app.py

In PUT /settings/daemon handler: validate codex_model against CodexPlugin.models. Reject unknown models with error message. Same for claude_model against ClaudePlugin.models.

### src/web/templates/components/settings_coders.html

Model column renders `<select>` with options from plugin.models. First option: `<option value="">(default)</option>`.

### tests/test_settings.py

- test_model_save_rejects_unknown_model
- test_model_save_accepts_valid_model
- test_model_dropdown_includes_default_option

## Success criteria
- scripts/ci.sh exits 0
- Cannot save an invalid model name
- Dropdown shows all available models per coder
