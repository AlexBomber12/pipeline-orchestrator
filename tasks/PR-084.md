# PR-084: Task file header parser

Branch: pr-084-task-header-parser
- Type: feature
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Task files have structured metadata in headers (Type, Complexity, Branch) but no parser extracts them. Auto-queue needs structured access to Depends on, Priority, Coder fields.

## Scope

New function `parse_task_header(path)` returning structured TaskHeader object. Validates required fields.

## Changes

### src/queue_parser.py

```python
@dataclass
class TaskHeader:
    pr_id: str
    title: str
    branch: str
    task_type: str          # bugfix, feature, architecture, refactor, docs
    complexity: str         # low, medium, high
    depends_on: list[str]   # ["PR-065", "PR-066"] or [] for "none"
    priority: int           # 1-5, default 3
    coder: str              # "claude", "codex", "any"

def parse_task_header(path: str | Path) -> TaskHeader:
    """Parse structured header from a task file.

    Raises QueueValidationError if required fields are missing.
    'Depends on: none' = empty list (explicit no deps).
    'Depends on:' missing = QueueValidationError.
    """
```

Parse rules:
- Title from `# PR-NNN: Title` line
- Branch from `Branch: xxx` line
- Type from `- Type: xxx` line
- Complexity from `- Complexity: xxx` line
- Depends on from `- Depends on: xxx` line. "none" = []. Missing = error.
- Priority from `- Priority: N` line. Missing = default 3.
- Coder from `- Coder: xxx` line. Missing = default "any".
