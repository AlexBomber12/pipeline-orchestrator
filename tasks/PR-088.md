## Problem

With auto-queue, QUEUE.md is no longer the source of truth for task status. But it is useful as a readable report. It should be auto-generated from DAG state, not manually edited.

## Scope

After each IDLE cycle, daemon generates QUEUE.md from current DAG state and commits to main. Format matches existing QUEUE.md for visual compatibility.

## Changes

### src/daemon/handlers/idle.py (or repo_ops.py)

New method `_generate_queue_md`:
```python
def _generate_queue_md(self, headers: list[TaskHeader], statuses: dict[str, TaskStatus]) -> str:
    lines = ["# Task Queue\n"]
    for h in headers:
        lines.append(f"## {h.pr_id}: {h.title}")
        lines.append(f"- Status: {statuses[h.pr_id].value}")
        lines.append(f"- Tasks file: tasks/{h.pr_id}.md")
        lines.append(f"- Branch: {h.branch}")
        if h.depends_on:
            lines.append(f"- Depends on: {', '.join(h.depends_on)}")
        lines.append("")
    return "\n".join(lines)
```

Call after task selection in handle_idle. Write to `tasks/QUEUE.md`. Commit only if content changed (avoid empty commits every cycle).
