# PR-022: IDLE open PR visibility

Branch: pr-022-idle-pr

## Scope

When daemon is IDLE (no TODO tasks), it should still check GitHub for open PRs and publish them to RepoState. This makes manual work (coding through VS Code/Cursor) visible in the dashboard.

## Changes

### src/daemon/runner.py

Modify handle_idle:

After checking QUEUE.md and finding no tasks (get_next_task returns None):
- Before returning, call github_client.get_open_prs(self.owner_repo)
- If open PRs exist: set self.state.current_pr to the first one (or the one matching a DONE task branch if possible)
- Do NOT change state from IDLE (this is observation, not action)
- Log "IDLE: {N} open PR(s) detected (manual work)"
- If no open PRs: clear current_pr (might have been set from previous cycle)

This means dashboard will show: state=IDLE + current_pr with CI/review status. Visually: gray IDLE badge but PR row shows live CI/review data.

### src/models.py

No changes needed. RepoState already supports current_pr without current_task.

### tests/test_runner.py (modify)

Add tests:
- handle_idle with no tasks but open PR sets current_pr
- handle_idle with no tasks and no open PRs clears current_pr
- handle_idle with no tasks does not change state from IDLE
- handle_idle open PR detection does not crash on GitHub API failure

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- Dashboard shows open PRs even when daemon is IDLE
- No state change when observing manual PRs
