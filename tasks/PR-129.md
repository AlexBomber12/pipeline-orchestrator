# PR-129: Replace Claude-specific labels with coder-neutral wording

Branch: pr-129-coder-neutral-labels
- Type: refactor
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Many user-facing strings in templates still say "Claude" or "Claude CLI" even when the active coder is Codex. This is a leftover from the pre-multi-coder era and breaks the mental model: the user sees "Claude CLI log" collapsed at the bottom of the repo detail page while "Coder: Codex CLI" is shown in the header.

Settings page is worse - several hint strings describe what "Claude" will or will not do, regardless of which coder the user actually selected.

## Scope

Sweep all templates and Python files for hardcoded "Claude" references that appear in user-facing output. Replace with coder-neutral wording.

Keep references to Claude where they are genuinely Claude-specific:
- `settings_auth.html`: label `"Claude CLI"` in the auth status table for the Claude-specific auth check is legitimate (it is checking Claude's auth specifically).
- Dropdown option `<option value="claude">Claude</option>` in `settings_repo_list.html` is legitimate (naming the concrete coder choice).
- Model selector `claude_model` field name is legitimate (it is the Claude-only setting).

Replace these (non-exhaustive; run full grep):
- `src/web/templates/repo.html:25` - `"Claude CLI log"` becomes `"Coder log"`
- `src/web/templates/components/settings_daemon.html` - all hint strings that say "Claude" generically (e.g. `"Max time Claude gets before being killed"`, `"Kills Claude during FIX if no push"`, `"Let Claude analyze failures before falling back to default recovery"`) become `"... coder ..."` or `"... the coder ..."`.
- `src/web/templates/components/repo_cards.html:133` - `_coder_name` derivation is fine as-is, it already handles both cases.

Run `grep -rn "Claude" src/web/templates/ src/web/app.py` and evaluate each hit. Document which ones were kept and why in the PR description.

## Files to modify

- `src/web/templates/components/settings_daemon.html`
- Any other template with generic "Claude" references found via grep.
