# PR-066: Codex CLI rate limit detection

Branch: pr-066-codex-rate-limit

- Type: feature
- Complexity: medium

## Problem

PR-065 introduces Codex CLI as a second coder with per-coder rate limit tracking (`rate_limited_coder`). But the actual detection logic is Claude-specific:
- `_detect_rate_limit` regex expects Anthropic stderr format ("95% of your weekly rate limit")
- Proactive check (PR-063) calls the Anthropic OAuth endpoint
- Codex CLI uses a different stderr format and a different auth provider (OpenAI/ChatGPT)

Without Codex-specific detection, the daemon will only catch Codex rate limits when the CLI exits non-zero with a generic "rate limit" string in stderr. This is fragile and late - the same problem PR-063 solved for Claude.

## Root cause

Codex CLI rate limit signals observed in the wild (from GitHub issues and community reports):

Stderr patterns:
- `"You've hit your usage limit. Upgrade to Pro or try again in 3 days 13 hours 6 minutes."`
- `"You've hit your usage limit. Try again in 4 hours 32 minutes."`
- `"Rate limit reached for organization <org> on tokens per min (TPM): Limit 250000, Used 250000, Requested 13472."`
- `"rate limit"` / `"429"` (generic, already caught by existing regex)

Codex subscription structure (ChatGPT Plus/Pro):
- 5-hour session limit (resets every 5 hours)
- Weekly limit (resets weekly)
- Same conceptual model as Claude Max, different API and auth

Codex CLI statusline JSON (available during runs) includes:

```json
{
  "rate_limits": {
    "five_hour": { "used_percentage": 87, "resets_at": 1744824000 },
    "seven_day": { "used_percentage": 42, "resets_at": 1745430000 }
  }
}
```

This is the same structure Claude Code uses. But for proactive checks BEFORE a run, a usage endpoint is needed.

Reverse-engineering from Codex CLI source code (open source, github.com/openai/codex):
- `backend-client/src/client.rs::get_rate_limits` calls `/wham/usage`
- `tui/src/chatwidget.rs::prefetch_rate_limits` polls it every 60s
- Full endpoint: `GET https://chatgpt.com/backend-api/wham/usage`
- Auth: ChatGPT OAuth token from `~/.codex/auth.json`
- Confirmed by issue #10869: the poller runs on ChatGPT base URL

Additional signal from API response headers (parsed in `codex-rs/codex-api/src/rate_limits.rs`):
- `x-codex-primary-used-percent` (session usage %)
- `x-codex-secondary-used-percent` (weekly usage %)
- These arrive on every API response but are NOT propagated to JSONL output in exec mode (issue #14728, unresolved)

## Scope

1. Add Codex-specific stderr parsing to `_detect_rate_limit`
2. Extract `resets_at` from Codex "try again in X" messages to set accurate `rate_limited_until`
3. Implement `OpenAIUsageProvider` in `src/usage.py` with proactive `GET https://chatgpt.com/backend-api/wham/usage` (same pattern as `OAuthUsageProvider` for Claude)
4. Share threshold config between coders (same `rate_limit_session_pause_percent` / `rate_limit_weekly_pause_percent` apply to both)

Depends on PR-065 (coder abstraction, `rate_limited_coder` field).

## Changes

### src/daemon/runner.py

Update `_detect_rate_limit` to handle Codex-specific patterns:

```python
def _detect_rate_limit(self, stderr: str) -> None:
    coder_name, _ = self._get_coder()
    session_threshold = self.app_config.daemon.rate_limit_session_pause_percent
    weekly_threshold = self.app_config.daemon.rate_limit_weekly_pause_percent
    lower = stderr.lower()
    triggered = False
    limit_type = "session"
    pause_min = 30

    if re.search(r"\b429\b", stderr):
        triggered = True
        limit_type = "session"

    m_anthropic = re.search(
        r"(\d{1,3})%\s*(?:of\s+)?(?:your\s+)?(?:(weekly|week|session|5-hour)\s+)?rate\s*limit"
        r"|(?:(weekly|week|session|5-hour)\s+)?rate\s*limit\s+(?:at\s+)?(\d{1,3})%",
        lower,
    )
    m_codex_retry = re.search(
        r"try again in\s+(?:(\d+)\s*days?)?\s*(?:(\d+)\s*hours?)?\s*(?:(\d+)\s*minutes?)?",
        lower,
    )

    if not triggered and m_anthropic and coder_name == "claude":
        pct = int(m_anthropic.group(1) or m_anthropic.group(4))
        qualifier = m_anthropic.group(2) or m_anthropic.group(3) or ""
        if qualifier in ("weekly", "week"):
            limit_type = "weekly"
            triggered = pct >= weekly_threshold
        else:
            limit_type = "session"
            triggered = pct >= session_threshold

    if not triggered and m_codex_retry and coder_name == "codex":
        days = int(m_codex_retry.group(1) or 0)
        hours = int(m_codex_retry.group(2) or 0)
        minutes = int(m_codex_retry.group(3) or 0)
        total_min = days * 1440 + hours * 60 + minutes
        if total_min > 0:
            triggered = True
            pause_min = total_min
            limit_type = "weekly" if days > 0 or hours > 12 else "session"

    if not triggered and "you've hit your usage limit" in lower:
        triggered = True
        limit_type = "session"

    if not triggered and not m_anthropic and not m_codex_retry and "rate limit" in lower:
        if "weekly" in lower or "week" in lower:
            limit_type = "weekly"
        triggered = True

    if triggered:
        self.state.rate_limited_until = datetime.now(timezone.utc) + timedelta(minutes=pause_min)
        self.state.rate_limited_coder = coder_name
        self.log_event(
            f"[{coder_name}] Rate limit detected ({limit_type}), "
            f"pausing for {pause_min} min"
        )
```

Key improvements:
- Codex "try again in X days Y hours Z minutes" parsed into exact pause_min instead of hardcoded 30
- "You've hit your usage limit" caught explicitly
- Anthropic percentage regex only applies when coder is "claude"
- Generic "rate limit" fallback still works for both coders

### src/usage.py

Add `OpenAIUsageProvider` alongside existing `OAuthUsageProvider`. Same pattern: HTTP GET, cache, fail-open.

```python
class OpenAIUsageProvider:
    """Proactive usage provider for Codex CLI (ChatGPT subscription).

    Calls the same endpoint that Codex CLI polls internally every 60s:
    GET https://chatgpt.com/backend-api/wham/usage
    Source: backend-client/src/client.rs::get_rate_limits (openai/codex)
    """

    ENDPOINT = "https://chatgpt.com/backend-api/wham/usage"

    def __init__(
        self,
        credentials_path: str,
        cache_ttl_sec: int = 60,
        request_timeout_sec: float = 5.0,
    ) -> None:
        self._credentials_path = Path(credentials_path)
        self._cache_ttl = cache_ttl_sec
        self._timeout = request_timeout_sec
        self._cached: UsageSnapshot | None = None
        self._consecutive_failures = 0

    @property
    def consecutive_failures(self) -> int:
        return self._consecutive_failures

    def fetch(self) -> UsageSnapshot | None:
        if self._cached is not None:
            age = time.time() - self._cached.fetched_at
            if age < self._cache_ttl:
                return self._cached
        token = self._read_token()
        if token is None:
            self._consecutive_failures += 1
            return None
        try:
            response = httpx.get(
                self.ENDPOINT,
                headers={
                    "Authorization": f"Bearer {token}",
                    "Accept": "application/json",
                },
                timeout=self._timeout,
            )
        except (httpx.HTTPError, OSError) as exc:
            logger.warning("OpenAI usage endpoint request failed: %s", exc)
            self._consecutive_failures += 1
            return None
        if response.status_code != 200:
            logger.warning(
                "OpenAI usage endpoint returned %s",
                response.status_code,
            )
            self._consecutive_failures += 1
            return None
        try:
            data = response.json()
            rl = data.get("rate_limit") or {}
            primary = rl.get("primary_window") or {}
            secondary = rl.get("secondary_window") or {}
            snap = UsageSnapshot(
                session_percent=int(primary.get("used_percent", 0)),
                session_resets_at=int(primary.get("reset_at") or 0),
                weekly_percent=int(secondary.get("used_percent", 0)),
                weekly_resets_at=int(secondary.get("reset_at") or 0),
                fetched_at=time.time(),
            )
        except (KeyError, TypeError, ValueError) as exc:
            logger.warning("OpenAI usage endpoint returned unexpected shape: %s", exc)
            logger.debug("OpenAI usage response body: %s", response.text[:500])
            self._consecutive_failures += 1
            return None
        self._cached = snap
        self._consecutive_failures = 0
        return snap

    def invalidate_cache(self) -> None:
        self._cached = None

    def _read_token(self) -> str | None:
        if not self._credentials_path.is_file():
            return None
        try:
            raw = self._credentials_path.read_text(encoding="utf-8")
            data = json.loads(raw)
        except (OSError, json.JSONDecodeError):
            return None
        tokens = data.get("tokens")
        if isinstance(tokens, dict):
            token = tokens.get("access_token")
            if isinstance(token, str) and token:
                return token
        return None
```

Notes on response shape (verified 2026-04-16 against live endpoint, plan_type: prolite):

```json
{
    "rate_limit": {
        "allowed": true,
        "limit_reached": false,
        "primary_window": {
            "used_percent": 2,
            "limit_window_seconds": 18000,
            "reset_after_seconds": 15418,
            "reset_at": 1776377151
        },
        "secondary_window": {
            "used_percent": 0,
            "limit_window_seconds": 604800,
            "reset_after_seconds": 602218,
            "reset_at": 1776963951
        }
    },
    "credits": { "has_credits": false, "balance": "0" },
    "rate_limit_reached_type": null
}
```

Field mapping:
- `rate_limit.primary_window` = 5-hour session (18000s)
- `rate_limit.secondary_window` = 7-day weekly (604800s)
- `used_percent` = usage percentage (same semantics as Anthropic's `used_percentage`)
- `reset_at` = Unix timestamp when window resets
- `limit_reached` = hard boolean, true when limit hit
- `allowed` = false when requests are blocked

Additional fields available but not used in v1:
- `additional_rate_limits[]` = per-model limits (e.g. GPT-5.3-Codex-Spark has separate buckets)
- `credits` = purchased credits info
- `code_review_rate_limit` = separate Codex Review limits (null when unused)
- `spend_control.reached` = spending cap hit

Token path in `~/.codex/auth.json`: `tokens.access_token`

### src/daemon/runner.py

Update `_proactive_usage_check` to use the correct provider per coder:

```python
def _proactive_usage_check(self) -> bool:
    coder_name, _ = self._get_coder()
    provider = (
        self._claude_usage_provider
        if coder_name == "claude"
        else self._codex_usage_provider
    )
    snapshot = provider.fetch()
    if snapshot is None:
        if provider.consecutive_failures == 10:
            self.log_event(
                f"[{coder_name}] Usage API degraded (10 consecutive failures), "
                f"falling back to reactive rate-limit detection"
            )
        return True
    session_threshold = self.app_config.daemon.rate_limit_session_pause_percent
    weekly_threshold = self.app_config.daemon.rate_limit_weekly_pause_percent
    breached = None
    resets_at = 0
    if snapshot.session_percent >= session_threshold:
        breached = "session"
        resets_at = snapshot.session_resets_at
    elif snapshot.weekly_percent >= weekly_threshold:
        breached = "weekly"
        resets_at = snapshot.weekly_resets_at
    if breached is None:
        return True
    self.state.rate_limited_until = datetime.fromtimestamp(resets_at, tz=timezone.utc)
    self.state.rate_limited_coder = coder_name
    self.state.state = PipelineState.PAUSED
    self.state.error_message = None
    self.log_event(
        f"[{coder_name}] Proactive pause: {breached} usage at "
        f"{snapshot.session_percent if breached == 'session' else snapshot.weekly_percent}%, "
        f"resumes at {self.state.rate_limited_until.isoformat()}"
    )
    return False
```

## Risks

- `/wham/usage` endpoint is undocumented (same as Anthropic's `/api/oauth/usage`). Found in Codex CLI source code, not in OpenAI API docs. May change without notice. Mitigation: fail-open, reactive fallback, endpoint URL in config for quick fix without rebuild.
- Response JSON shape verified against live endpoint (2026-04-16, plan_type: prolite). OpenAI may change field names or structure in the future. The `try/except` block and debug logging will catch regressions.
- `~/.codex/auth.json` token path verified: `tokens.access_token`. Codex CLI may change this in future versions. Reader logs warning on failure, daemon degrades to reactive detection.
- Codex stderr format may change across versions. The "try again in X" regex is deliberately loose to handle minor rewording.
- "You've hit your usage limit" is a UI-facing string that OpenAI may localize. If the user's ChatGPT account is in a different language, this string will not match. Mitigation: generic "rate limit" fallback catches most variants.
- `exec` mode does not propagate `x-codex-primary-used-percent` headers to JSONL output (issue #14728). This means in-flight monitoring via JSONL events is not possible for Codex until OpenAI fixes this. Proactive check via `/wham/usage` is the only reliable pre-call signal.
- Shared thresholds (session + weekly) for both coders. If Claude and Codex usage patterns differ enough to need separate thresholds, this is a simple config split (not an architecture change).
