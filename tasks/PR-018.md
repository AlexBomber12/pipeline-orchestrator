# PR-018: Dashboard alerts panel

Branch: pr-018-alerts

## Scope

Add a persistent alerts panel that highlights repos requiring attention: HUNG reviews, ERROR states, and escalations. Alerts stay visible until the underlying condition resolves.

## Files to modify

### src/web/app.py

Add routes:

GET /partials/alerts -> alerts panel component
- Collect all repo states from Redis
- Filter for states: HUNG, ERROR
- For HUNG: show repo name, PR number, how long it's been hung (from last_activity)
- For ERROR: show repo name, error message
- Sort: ERROR first, then HUNG, then by duration descending

### src/web/templates/index.html

Add alerts panel above stats cards:
- Only renders if there are active alerts
- Collapsible (default expanded)
- HTMX refresh: hx-get="/partials/alerts" hx-trigger="every 10s"

### src/web/templates/components/alerts_panel.html

Layout:
- Header: "Attention Required" with count badge (red)
- If no alerts: div is empty (HTMX swap replaces with nothing)
- Each alert card:
  - ERROR alerts:
    - Red left border
    - Repo name (link to /repo/{name})
    - Error message (truncated, expandable)
    - "Since {time}" timestamp
  - HUNG alerts:
    - Purple left border
    - Repo name (link to /repo/{name})
    - PR number (link to GitHub)
    - "Hung for {duration}" (human readable: "15 min", "2h 30min")
    - Last review status (eyes/pending)

## Design notes
- Alerts panel: bg-surface-2, border-l-4 for severity color
- ERROR: border-fail, subtle red background
- HUNG: border-hung, subtle purple background
- Count badge: rounded-full bg-fail text-white text-xs px-2
- Duration: human readable, updates on each HTMX refresh
- When no alerts: the div disappears cleanly (no empty box)
- Transitions: alerts appearing/disappearing should not cause layout jumps (use min-height or transition)

## Tests

### tests/test_alerts.py
- GET /partials/alerts with no alerts returns empty/minimal HTML
- GET /partials/alerts with ERROR repo returns alert card with error message
- GET /partials/alerts with HUNG repo returns alert card with PR number and duration
- GET /partials/alerts with both ERROR and HUNG returns ERROR first
- Alert links point to correct repo detail page
