# PR-044: Heartbeat during Claude CLI calls

Branch: pr-044-heartbeat

## Scope

Dashboard freezes during Claude CLI calls because subprocess blocks the async loop (pre PR-042) or because no state updates happen during long runs. Add periodic heartbeat that publishes state every 30 seconds while Claude is working, keeping dashboard alive and showing that daemon is not stuck.

## Changes

### src/daemon/runner.py

Add heartbeat coroutine:
```python
async def _publish_while_waiting(self, label: str):
    while True:
        await asyncio.sleep(30)
        self.log_event(f"{label}...")
        await self.publish_state()
```

In handle_coding, wrap Claude CLI call:
```python
heartbeat = asyncio.create_task(self._publish_while_waiting("CODING"))
try:
    code, stdout, stderr = await claude_cli.run_planned_pr_async(...)
finally:
    heartbeat.cancel()
```

Same pattern in handle_fix for FIX REVIEW.

### tests/test_runner.py

- test_handle_coding_publishes_heartbeat: verify publish_state called during Claude run
- test_handle_fix_publishes_heartbeat: same for fix

## Success criteria
- scripts/ci.sh exits 0
- Dashboard updates every 30s during Claude CLI runs
- Heartbeat cancelled cleanly after Claude exits
