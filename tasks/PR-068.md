# PR-068: Remove heartbeat spam from event log

Branch: pr-068-heartbeat-cleanup
- Type: bugfix
- Complexity: low
- Depends on: none
- Priority: 3
- Coder: any

## Problem

`_publish_while_waiting` writes "FIX..." / "CODING..." to event log every 30 seconds. Fills history with noise, hides real events. Daemon liveness already shown via `last_updated` timestamp in UI header.

## Scope

Remove `self.log_event(f"{label}...")` from `_publish_while_waiting` in runner.py. Keep `await self.publish_state()` so Redis state and `last_updated` still refresh.

## Changes

### src/daemon/runner.py

In `_publish_while_waiting` (around line 308), remove the `self.log_event(f"{label}...")` call. Keep the rest of the method intact.

### tests/

Update any tests asserting on "FIX..." or "CODING..." heartbeat events in history. These events will no longer appear.

## Success criteria
- scripts/ci.sh exits 0
- Event log no longer contains repeated "FIX..." or "CODING..." entries
- Dashboard still shows `updated just now` during long CLI calls (via publish_state)
- All tests pass with updated assertions
