# PR-090: Expanded error classification

- Type: feature
- Complexity: low
- Depends on: none
- Priority: 3
- Coder: any

## Problem

ErrorCategory enum has only 3 values: RATE_LIMIT, TIMEOUT, OTHER. Cannot distinguish OOM from auth failure from CI failure. RunRecord.exit_reason needs richer vocabulary.

## Scope

Expand ErrorCategory with new types. Update _classify_error patterns.

## Changes

### src/daemon/handlers/error.py

Expand enum:
```python
class ErrorCategory(Enum):
    RATE_LIMIT = "rate_limit"
    TIMEOUT = "timeout"
    OOM = "oom"
    AUTH_FAILURE = "auth_failure"
    CI_FAILURE = "ci_failure"
    GHOST_PUSH = "ghost_push"
    STALE_BRANCH = "stale_branch"
    CLI_NOT_FOUND = "cli_not_found"
    GIT_ERROR = "git_error"
    OTHER = "other"
```

Update `_classify_error`:
```python
def _classify_error(context: str) -> ErrorCategory:
    lowered = context.lower()
    if "rate limit" in lowered or "429" in lowered:
        return ErrorCategory.RATE_LIMIT
    if "timeout" in lowered:
        return ErrorCategory.TIMEOUT
    if "oom" in lowered or "out of memory" in lowered or "killed" in lowered:
        return ErrorCategory.OOM
    if "auth" in lowered or "unauthorized" in lowered or "401" in lowered:
        return ErrorCategory.AUTH_FAILURE
    if "ci" in lowered and "fail" in lowered:
        return ErrorCategory.CI_FAILURE
    if "ghost push" in lowered or "head sha" in lowered:
        return ErrorCategory.GHOST_PUSH
    if "not found" in lowered and "cli" in lowered:
        return ErrorCategory.CLI_NOT_FOUND
    if "git" in lowered and ("error" in lowered or "fail" in lowered):
        return ErrorCategory.GIT_ERROR
    return ErrorCategory.OTHER
```
