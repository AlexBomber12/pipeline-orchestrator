# PR-034: Model config, pytest in container, auth check

Branch: pr-034-config-hardening

## Scope

Three small hardening fixes bundled together:

1. Explicit model selection via config.yml instead of relying on container default.
2. Add pytest to container so self-hosted CI works.
3. Proper auth validation on startup.

## Changes

### src/config.py

Add to DaemonConfig:
```python
claude_model: str = "opus"
```

### src/claude_cli.py

Add model parameter to run_claude:
```python
def run_claude(
    prompt: str,
    cwd: str,
    timeout: int = 600,
    model: str | None = None,
) -> tuple[int, str, str]:
```

If model is not None, insert ["--model", model] into cmd after "--dangerously-skip-permissions":
```python
cmd = ["claude", "--print", "--dangerously-skip-permissions"]
if model:
    cmd.extend(["--model", model])
cmd.append(prompt)
```

Update run_planned_pr, fix_review, diagnose_error to accept and forward model parameter.

### src/daemon/runner.py

Pass model to all claude_cli calls:
```python
claude_cli.run_planned_pr(self.repo_path, model=self.app_config.daemon.claude_model)
claude_cli.fix_review(self.repo_path, model=self.app_config.daemon.claude_model)
claude_cli.diagnose_error(self.repo_path, context, model=self.app_config.daemon.claude_model)
```

### Dockerfile

Add pytest to pip install:
```dockerfile
RUN pip install --no-cache-dir -r requirements.txt pytest
```

Or better, add pytest to requirements.txt under a [dev] group. Simplest: just add `pytest` to requirements.txt.

### requirements.txt

Add:
```
pytest
```

### src/daemon/main.py

Fix _validate_auth to actually check Claude auth, not just binary presence:
```python
for name, cmd in [
    ("claude", ["claude", "auth", "status"]),
    ("gh", ["gh", "auth", "status"]),
]:
```

`claude auth status` exits 0 if logged in, non-zero if not. This replaces `claude --version` which only checks binary existence.

### config.yml

Add under daemon section:
```yaml
daemon:
  claude_model: opus
```

### tests/test_claude_cli.py

- test_run_claude_with_model: verify --model opus in cmd
- test_run_claude_without_model: verify no --model when None
- test_run_planned_pr_passes_model: verify model forwarded

### tests/test_config.py

- test_daemon_config_claude_model_default: verify default is "opus"

## Success criteria
- scripts/ci.sh exits 0
- Model explicitly set in config.yml
- pytest available in Docker container
- Auth check verifies actual login, not just binary
