# PR-034: Event log UX overhaul

Branch: pr-034-event-log-ux

## Scope

Fix 4 UX issues with the event log on the repo detail page:
1. Timestamps show full ISO with milliseconds in UTC - show "HH:MM" or "N min ago" in local time
2. HTMX polling (every 5s outerHTML swap) resets scroll position
3. Filter buttons may not work correctly
4. "updated" timestamp on page header is UTC, should be local

## Changes

### src/web/templates/components/event_log.html

1. Replace raw ISO timestamp with a relative time display. Add data-ts attribute with ISO value for JS to convert:
```html
<span class="text-gray-500 shrink-0" data-ts="{{ entry.get('time', '') }}"></span>
```

2. Change hx-swap from "outerHTML" to "innerHTML" on the event list (ul) only, not the whole section. Wrap the list in a div with id="event-list-inner" that is the HTMX target. Filter buttons and header stay outside the swap target.

3. Add scroll position preservation: use hx-swap="innerHTML show:none" or save/restore scrollTop via htmx:beforeSwap event.

### src/web/templates/base.html

Add a small JS snippet that converts all data-ts elements to local "HH:MM" or "N min ago" format:
```javascript
function formatTimestamps() {
    document.querySelectorAll('[data-ts]').forEach(el => {
        const dt = new Date(el.dataset.ts);
        const diff = (Date.now() - dt.getTime()) / 60000;
        if (diff < 60) el.textContent = Math.round(diff) + ' min ago';
        else el.textContent = dt.toLocaleTimeString([], {hour: '2-digit', minute: '2-digit'});
    });
}
```
Call on load and after htmx:afterSwap.

### src/web/templates/repo.html

Convert the "updated" timestamp in the page header to local time using the same data-ts approach.

### tests/test_web_app.py

- Verify "updated" header has data-ts attribute
