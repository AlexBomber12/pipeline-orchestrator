# PR-063: Proactive usage check via OAuth usage endpoint

Branch: pr-063-proactive-usage-check

- Type: feature
- Complexity: medium

## Problem

Rate-limit detection is reactive: `_detect_rate_limit(stderr)` parses CLI output and only triggers a pause after a CLI call that already burned quota. When Claude CLI succeeds silently at 95-99% usage, the daemon happily queues the next call. The first signal the daemon gets is a 429 from the API on the following run, which either wastes a large CODING attempt or flips the repo to ERROR mid-task.

Concrete failure observed 2026-04-16:
- Session usage at 100%, weekly at 92%, threshold weekly=99%
- Daemon completed CODING for PR-062 at the edge of the window (last successful call), moved to WATCH
- Next CODING would have hit 429 and wasted a planned PR attempt

Also: with Max subscription plus API billing fallback, hitting the 429 boundary silently spends API credits instead of pausing. Operator wants to choose pause vs spend deliberately, not discover it after the fact.

## Root cause

The daemon has no visibility into session/weekly usage except via stderr parsing after a CLI invocation. Claude CLI versions >= 1.2.80 expose usage in the `/usage` slash command and in the statusline JSON, both backed by a single OAuth endpoint the CLI itself calls:

```
GET https://api.anthropic.com/api/oauth/usage
Authorization: Bearer <oauth_access_token>
anthropic-beta: oauth-2025-04-20
User-Agent: claude-code/<version>
```

Response shape:
```json
{
  "five_hour": { "used_percentage": 100, "resets_at": 1744824000 },
  "seven_day": { "used_percentage": 92, "resets_at": 1745430000 }
}
```

This is the same source the dashboard at claude.ai uses. Calling it proactively before each Claude CLI invocation gives the daemon the data it needs to pause deliberately instead of reactively.

The endpoint is undocumented and beta. Strategy: use it as the primary signal, fall back to the existing reactive `_detect_rate_limit` when the endpoint is unavailable. Never block the daemon because of a usage probe failure (fail-open).

## Scope

Add a `UsageProvider` abstraction with a primary OAuth implementation. Call it from `_check_rate_limit` before every Claude CLI invocation. On breach, set `rate_limited_until = resets_at` and transition to PAUSED (from PR-062). On probe failure (network, 401, 5xx, malformed response) degrade silently to the existing reactive path and emit a WARNING after N consecutive failures so the operator sees the degradation.

Depends on PR-062 (PAUSED state enum and handler). This PR assumes `PipelineState.PAUSED` exists and `handle_paused` already recovers correctly when `rate_limited_until` expires.

## Changes

### src/usage.py (new)

New module. Abstract `UsageProvider` with one concrete implementation.

```python
from __future__ import annotations

import json
import logging
import time
from dataclasses import dataclass
from pathlib import Path
from typing import Protocol

import httpx

logger = logging.getLogger(__name__)


@dataclass(frozen=True)
class UsageSnapshot:
    session_percent: int
    session_resets_at: int
    weekly_percent: int
    weekly_resets_at: int
    fetched_at: float


class UsageProvider(Protocol):
    def fetch(self) -> UsageSnapshot | None: ...


class OAuthUsageProvider:
    """Primary usage provider. Calls the undocumented OAuth endpoint that
    powers Claude Code /usage. Fail-open: returns None on any error."""

    ENDPOINT = "https://api.anthropic.com/api/oauth/usage"

    def __init__(
        self,
        credentials_path: str,
        user_agent: str,
        beta_header: str,
        cache_ttl_sec: int = 60,
        request_timeout_sec: float = 5.0,
    ) -> None:
        self._credentials_path = Path(credentials_path)
        self._user_agent = user_agent
        self._beta_header = beta_header
        self._cache_ttl = cache_ttl_sec
        self._timeout = request_timeout_sec
        self._cached: UsageSnapshot | None = None
        self._consecutive_failures = 0

    @property
    def consecutive_failures(self) -> int:
        return self._consecutive_failures

    def fetch(self) -> UsageSnapshot | None:
        if self._cached is not None:
            age = time.time() - self._cached.fetched_at
            if age < self._cache_ttl:
                return self._cached
        token = self._read_token()
        if token is None:
            self._consecutive_failures += 1
            return None
        try:
            response = httpx.get(
                self.ENDPOINT,
                headers={
                    "Authorization": f"Bearer {token}",
                    "anthropic-beta": self._beta_header,
                    "User-Agent": self._user_agent,
                    "Accept": "application/json",
                },
                timeout=self._timeout,
            )
        except (httpx.HTTPError, OSError) as exc:
            logger.warning("usage endpoint request failed: %s", exc)
            self._consecutive_failures += 1
            return None
        if response.status_code != 200:
            logger.warning(
                "usage endpoint returned %s (check anthropic-beta header)",
                response.status_code,
            )
            self._consecutive_failures += 1
            return None
        try:
            data = response.json()
            snap = UsageSnapshot(
                session_percent=int(data["five_hour"]["used_percentage"]),
                session_resets_at=int(data["five_hour"]["resets_at"]),
                weekly_percent=int(data["seven_day"]["used_percentage"]),
                weekly_resets_at=int(data["seven_day"]["resets_at"]),
                fetched_at=time.time(),
            )
        except (KeyError, TypeError, ValueError) as exc:
            logger.warning("usage endpoint returned unexpected shape: %s", exc)
            self._consecutive_failures += 1
            return None
        self._cached = snap
        self._consecutive_failures = 0
        return snap

    def invalidate_cache(self) -> None:
        self._cached = None

    def _read_token(self) -> str | None:
        if not self._credentials_path.is_file():
            return None
        try:
            raw = self._credentials_path.read_text(encoding="utf-8")
            data = json.loads(raw)
        except (OSError, json.JSONDecodeError):
            return None
        for key in ("accessToken", "access_token"):
            token = data.get(key)
            if isinstance(token, str) and token:
                return token
        claude_ai = data.get("claudeAiOauth") or data.get("claude_ai_oauth")
        if isinstance(claude_ai, dict):
            for key in ("accessToken", "access_token"):
                token = claude_ai.get(key)
                if isinstance(token, str) and token:
                    return token
        return None
```

Notes:
- The token JSON shape varies across Claude CLI versions. The reader tries flat keys first, then a nested `claudeAiOauth` object. Extend the list if a new shape appears.
- `httpx` is already in the dependency tree via the web layer. Confirm in requirements.txt and add if missing.

Add a new helper `_proactive_usage_check()`:

```python
def _proactive_usage_check(self) -> bool:
    """Return True if CLI calls are allowed, False if usage threshold breached.

    Fail-open: returns True when the provider cannot reach the endpoint,
    deferring to the reactive _detect_rate_limit on stderr after the CLI run.
    """
    snapshot = self._usage_provider.fetch()
    if snapshot is None:
        if self._usage_provider.consecutive_failures == 10:
            self.log_event(
                "Usage API degraded (10 consecutive failures), "
                "falling back to reactive rate-limit detection"
            )
        return True
    session_threshold = self.app_config.daemon.rate_limit_session_pause_percent
    weekly_threshold = self.app_config.daemon.rate_limit_weekly_pause_percent
    breached = None
    resets_at = 0
    if snapshot.session_percent >= session_threshold:
        breached = "session"
        resets_at = snapshot.session_resets_at
    elif snapshot.weekly_percent >= weekly_threshold:
        breached = "weekly"
        resets_at = snapshot.weekly_resets_at
    if breached is None:
        return True
    self.state.rate_limited_until = datetime.fromtimestamp(resets_at, tz=timezone.utc)
    self.state.state = PipelineState.PAUSED
    self.state.error_message = None
    self.log_event(
        f"Proactive pause: {breached} usage at "
        f"{snapshot.session_percent if breached == 'session' else snapshot.weekly_percent}%, "
        f"resumes at {self.state.rate_limited_until.isoformat()}"
    )
    return False
```

Modify `_check_rate_limit` to call the proactive check first when not already paused:

```python
def _check_rate_limit(self) -> bool:
    if self.state.rate_limited_until is not None:
        if datetime.now(timezone.utc) < self.state.rate_limited_until:
            remaining = (self.state.rate_limited_until - datetime.now(timezone.utc)).total_seconds()
            self.log_event(f"Rate limited, resuming in {int(remaining)}s")
            if self.state.state != PipelineState.PAUSED:
                self.state.state = PipelineState.PAUSED
            return False
        self.state.rate_limited_until = None
        self._usage_provider.invalidate_cache()
        self.log_event("Rate limit window expired, resuming")
    return self._proactive_usage_check()
```

Key invariants:
- Proactive check runs only when not already paused (avoids double work per cycle).
- When a window expires, invalidate the cache so the next check fetches fresh data before making a decision about the next call.
- Reactive `_detect_rate_limit(stderr)` is untouched: it remains the safety net when the OAuth endpoint is degraded or when the CLI hits a 429 between proactive checks.

## Risks

- OAuth endpoint is undocumented. Anthropic may change the path, shape, or beta header. Mitigation: fail-open, reactive fallback, degraded indicator in UI, beta header in config for quick fix without rebuild.
- Token rotation: Claude CLI refreshes tokens periodically. The provider re-reads the file on every fetch (no in-memory caching of the token itself), so refreshes are picked up automatically.
- Clock skew between daemon and Anthropic: `resets_at` is a Unix timestamp. If daemon clock is skewed, recovery timing drifts. Acceptable: timestamps are in seconds, skew is typically under a minute.
- Rate-limit on the usage endpoint itself: unknown. Mitigation: 60s cache default, configurable up to 3600s.
- If proactive check runs during an already-active PAUSED window (after a re-check) the cache returns stale data until invalidated. Handled by `invalidate_cache()` when the pause expires.
