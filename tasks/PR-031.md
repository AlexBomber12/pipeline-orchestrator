# PR-031: Pre-merge sync with main

Branch: pr-031-pre-merge-sync

## Scope

When MICRO PRs merge to main while a PLANNED PR is in FIX cycle, the PR branch diverges from main. gh pr merge fails with "not mergeable" conflict. Daemon enters ERROR -> ESCALATE loop requiring manual intervention.

Fix: before merge, sync the PR branch with main. If conflict, use Claude CLI to resolve.

## Changes

### src/daemon/runner.py

In handle_merge, before calling github_client.merge_pr:

1. git fetch origin {base_branch}
2. git checkout {pr_branch}
3. git merge origin/{base_branch} --no-edit
4. If merge conflict (exit code 1 with "CONFLICT" in stderr):
   a. Run claude_cli.run_claude("Resolve all merge conflicts in the working tree. Keep both sides of changes. Run scripts/ci.sh to verify.", repo_path, timeout=300)
   b. If Claude succeeds: git add -A, git commit --no-edit, git push
   c. If Claude fails: set ERROR "merge conflict resolution failed"
5. If merge clean: git push origin {pr_branch}
6. Proceed to merge_pr

### tests/test_runner.py

- test_handle_merge_syncs_with_main: mock clean merge, verify git merge origin/main called before merge_pr
- test_handle_merge_resolves_conflict: mock merge conflict, mock Claude resolution success, verify merge proceeds
- test_handle_merge_errors_on_unresolvable_conflict: mock merge conflict, mock Claude failure, verify ERROR state

## Success criteria
- scripts/ci.sh exits 0
- PRs with merge conflicts get auto-resolved before merge
- Unresolvable conflicts surface as ERROR, not infinite ESCALATE loop
