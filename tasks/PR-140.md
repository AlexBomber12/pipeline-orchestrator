# PR-140: Event log readability on narrow screens

Branch: pr-140-event-log-mobile
- Type: feature
- Complexity: low
- Depends on: PR-139
- Priority: 2
- Coder: any

## Problem

The event log on the repo detail page is a horizontal-packed format: `{time_ago} [STATE_BADGE] {message}` on one line. On mobile the message text gets cut off or forces horizontal scroll if the message is long (common for `PLANNED PR output` or truncated stdout).

Readable mobile format should:
- Wrap long messages to multiple lines
- Keep timestamp and state badge on one line
- Prevent horizontal overflow

## Scope

Change the event log row layout on mobile to a stacked format:

```
[STATE]  2 min ago
Full message text wraps naturally across lines
```

And on desktop keep the current inline format:
```
2 min ago  [STATE]  Full message text
```

Use Tailwind: `flex flex-col sm:flex-row sm:items-start gap-1 sm:gap-2` on the event row container. Ensure message uses `break-words` to avoid long-URL overflow.

Also:
- Reduce font-size on mobile by one step (text-sm -> text-xs)
- Add more vertical padding between rows on mobile so taps do not accidentally select adjacent text

Respect the PR-130 dedup format (`(x9)` counter, last-seen timestamp) on mobile too - stack cleanly, do not overflow.

Do NOT introduce new text truncation. Long messages should wrap, not be cut.
