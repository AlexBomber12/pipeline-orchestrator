# PR-159: derive_task_status reflects current_task in QUEUE.md (B-1 fix)

Branch: pr-159-queue-doing-current-task
- Type: bugfix
- Complexity: low
- Depends on: none
- Priority: 1
- Coder: any

## Problem

The shim that runs in e2e tests (`tests/e2e/lib/coder_shim.sh`) parses `tasks/QUEUE.md` to find the DOING task. When daemon picks a task, it stores the task in Redis as `RepoState.current_task` and `RepoState.current_pr_id`. The auto-generated QUEUE.md reflects this state via `derive_task_status` in `src/task_status.py:29-53`, which currently determines status purely from GitHub state:

- DONE if PR ID in `merged_pr_ids` or matches a merged PR by branch
- DOING if matches an open PR by branch
- TODO otherwise

This produces a race condition observed in CI run 24941976007 (2026-04-25):

1. Test uploads task file PR-1777155574, web pushes to testbed/main.
2. Daemon git pull, sees new task file.
3. Daemon picks the task → sets `RepoState.current_pr_id = "PR-1777155574"` in Redis.
4. Daemon regenerates QUEUE.md (in `_write_generated_queue_md` IDLE handler). `derive_task_status` returns TODO because the PR has not been created yet (no open PR exists, no merged PR).
5. QUEUE.md committed and pushed with Status: TODO for the active task.
6. Daemon spawns shim (`claude --print "PLANNED PR"`).
7. Shim reads QUEUE.md, finds no DOING entry, prints `shim: no DOING task in QUEUE.md, exiting 0`.
8. Daemon sees no PR was created, retries 3 times, then `coder succeeded but no PR found for branch` → state ERROR.

This affects 3 of 6 e2e tests consistently: test_upload_and_merge, test_stop_and_resume, test_sigkill_recovery. Logs in `evidence/stack-logs.txt` artifact from run 24941976007.

The fix is to extend `derive_task_status` to accept the active task's PR ID. When the task being evaluated matches the active task in Redis state and has not been merged, it returns DOING even before an open PR exists.

This matches the design specified in `docs/roadmap.md` line 427-433 (PR-159 in old numbering, "QUEUE.md DOING reflects current_task"). The single-writer principle is preserved: only `_write_generated_queue_md` writes QUEUE.md, only the helper changes.

## Scope

Modify `src/task_status.py` to add `current_task_pr_id` parameter. Update both call sites that invoke it. Add unit tests for the new behavior. No other changes.

This PR does NOT change QUEUE.md format. It does NOT change shim behavior. It does NOT change `RepoState` schema. It does NOT remove QUEUE.md (that is a future architectural refactor — Variant D / Sprint F1.2). This PR is the minimum change that makes the existing QUEUE.md inter-process protocol race-free.

## Files to touch

`src/task_status.py`: extend `derive_task_status` signature to accept an optional keyword argument `current_task_pr_id: str | None = None`. The default None preserves existing behavior for any caller that does not pass it.

The new logic order:
1. If `task_header.pr_id in merged_pr_ids` → DONE (existing).
2. If matched merged PR by branch → DONE (existing).
3. If matched open PR by branch → DOING (existing).
4. **NEW:** If `current_task_pr_id == task_header.pr_id` (and we did not match DONE above) → DOING.
5. Else TODO (existing).

The new check goes AFTER the existing DOING check (open PR match) so that an open PR is the strongest signal. It goes BEFORE the TODO fallback.

Sketch (illustrative):

```python
def derive_task_status(
    task_header: TaskHeader,
    merged_pr_ids: set[str],
    open_prs: Iterable[PRInfo],
    merged_prs: Iterable[PRInfo] = (),
    *,
    current_task_pr_id: str | None = None,
) -> TaskStatus:
    if task_header.pr_id in merged_pr_ids:
        return TaskStatus.DONE
    if find_matching_merged_pr(task_header.pr_id, task_header.branch, merged_prs) is not None:
        return TaskStatus.DONE
    if find_matching_open_pr(task_header.pr_id, task_header.branch, open_prs) is not None:
        return TaskStatus.DOING
    if current_task_pr_id == task_header.pr_id:
        return TaskStatus.DOING
    return TaskStatus.TODO
```

Use keyword-only argument (`*,`) to prevent accidental positional misuse at call sites. Existing callers that do not pass `current_task_pr_id` continue to work unchanged.

`src/daemon/handlers/idle.py`: at the call site (line 343 in current code), pass the active task's PR ID:

```python
current_pr_id = self.state.current_pr.pr_id if self.state.current_pr else None
# OR more directly:
current_task_id = self.state.current_task.pr_id if self.state.current_task else None

statuses = {
    header.pr_id: derive_task_status(
        header,
        merged_pr_ids,
        open_prs,
        merged_prs,
        current_task_pr_id=current_task_id,
    )
    for header in headers
}
```

The implementer should pick whichever of `current_pr` or `current_task` gives the active PR ID. Both are RepoState fields. `current_task` is `QueueTask | None` per models.py:91 and has `pr_id` attribute. `current_pr` is `PRInfo | None` per models.py:92 — verify whether it has `pr_id` or different field name. Use `current_task.pr_id` if available; if not, extract from `current_pr` per the existing code conventions in idle.py.

`src/task_status.py:242`: the second call site is in `derive_queue_tasks_from_state` or similar function. Pass `current_task_pr_id` through here as well — read the active task ID from whatever context this function runs in. If the function does not have access to RepoState, ADD a parameter to its signature too, and update its caller. Trace through cleanly.

`tests/test_task_status.py`: add tests covering:
- `current_task_pr_id="PR-X"` and PR-X not merged not open → returns DOING
- `current_task_pr_id="PR-X"` and PR-X merged → returns DONE (DONE precedence over current task)
- `current_task_pr_id="PR-X"` and PR-X has open PR → returns DOING (same result, different reason)
- `current_task_pr_id="PR-X"` and evaluating PR-Y (different ID) → PR-Y returns TODO
- `current_task_pr_id=None` (default) → existing behavior preserved (regression guard)

`tests/test_idle_handler.py` or wherever IDLE handler tests live: add an integration-style test where:
1. RepoState has `current_task.pr_id = "PR-X"`
2. PR-X has no open PR yet (not yet created)
3. PR-X header in tasks/PR-X.md
4. IDLE handler runs, generates QUEUE.md
5. QUEUE.md contains `## PR-X` followed by `- Status: DOING`

This is the regression guard for the e2e race condition.
