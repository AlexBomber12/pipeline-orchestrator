# PR-176: Tasks viewer in repo detail page

Branch: pr-176-tasks-viewer
- Type: ux
- Complexity: medium
- Depends on: none
- Priority: 1
- Coder: any

## Problem

The dashboard shows aggregate counts for queue: `repo.queue_done / repo.queue_total` (e.g., "108 / 111 done"). But there is no way to see WHICH tasks are in the queue, what their status is, or what's coming up next, without SSHing to the AI-Server and reading files in `/data/repos/{slug}/tasks/`.

For a single architect (Aleksei) operating the orchestrator, this is a productivity gap. Every time a new task arrives or daemon picks something up, the user has to either trust the count or open a terminal and inspect files. There is no "what's in my queue right now" UI.

The fix: add a Tasks viewer in the repo detail page (or as expandable panel on the repo card). Show task file list grouped by status, with clickable rows that expand to show task file contents.

## Scope

Add a new endpoint `GET /repos/{name}/tasks` that returns rendered HTML of the task list. Add an expandable / linkable panel in the repo detail UI that fetches via HTMX and shows tasks grouped by status (TODO / DONE / DOING-equivalent inferred from QUEUE.md or daemon state). Each task row expandable to view full task file content.

This PR does NOT add task editing, task deletion, task creation, or task reordering. Pure read-only viewer. Future PR can add edit capability if needed.

This PR does NOT change daemon behavior, task file format, or QUEUE.md format. UI-only addition.
