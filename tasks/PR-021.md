# PR-021: Daemon startup hardening

Branch: pr-021-daemon-hardening

## Scope

Run `gh auth setup-git` automatically at daemon startup so git clone/push works without manual intervention. Also add startup validation for auth status.

## Changes

### src/daemon/main.py

Add function:

def _setup_git_auth() -> None:
- Run `gh auth setup-git` via subprocess (timeout 30s)
- Log result at INFO level
- On failure: log WARNING but do not crash (daemon can still work with repos that are public)

def _validate_auth() -> dict[str, bool]:
- Check `claude --version` (timeout 5s): success = True
- Check `gh auth status` (timeout 5s): success = True
- Return {"claude": bool, "gh": bool}
- Log results at INFO

Modify async def main():
- Call _setup_git_auth() before creating runners
- Call _validate_auth() and log summary
- If both False: log ERROR "No auth configured. Run: docker compose run --rm daemon bash"
- Continue running regardless (auth might be configured later)

### Environment variable for GH_CONFIG_DIR

Ensure `GH_CONFIG_DIR` is set before `gh auth setup-git` runs. In main(), at the very start:
```python
import os
gh_dir = os.environ.get("GH_CONFIG_DIR")
if gh_dir:
    os.environ["GH_CONFIG_HOME"] = gh_dir
```

This ensures `gh` reads credentials from the mounted volume.

### tests/test_daemon_main.py (modify)

Add tests:
- _setup_git_auth calls subprocess with "gh auth setup-git"
- _setup_git_auth does not crash on CalledProcessError
- _validate_auth returns correct booleans based on mocked subprocess results
- main() calls _setup_git_auth before creating runners
