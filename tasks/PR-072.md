# PR-072: BUG-3 - codex review dedup by PR number

Branch: pr-072-review-dedup-by-pr
- Type: bugfix
- Complexity: low
- Depends on: none
- Priority: 3
- Coder: any

## Problem

Current @codex review dedup uses `within_minutes=5`. FIX cycles taking > 5 min repost @codex review on the same PR even when no new push occurred. Wastes Codex Review quota.

## Scope

Track last @codex review post per PR number. Skip posting if same PR and no new push since last post.

## Changes

### src/daemon/repo_ops.py (or wherever _post_codex_review lives)

Add instance variable `_last_codex_review_pr: int | None = None` and `_last_codex_review_push_count: int | None = None` to runner __init__.

In _post_codex_review:
```python
pr_number = self.state.current_pr.number
push_count = self.state.current_pr.push_count
if (
    self._last_codex_review_pr == pr_number
    and self._last_codex_review_push_count == push_count
):
    self.log_event(f"Skipping duplicate @codex review for PR #{pr_number}")
    return
self._last_codex_review_pr = pr_number
self._last_codex_review_push_count = push_count
```

Remove the old `within_minutes` time-based dedup if it exists.

### tests/

- test_codex_review_not_reposted_same_pr_same_push: same PR, same push count, skip
- test_codex_review_reposted_after_new_push: same PR, push count incremented, post

## Success criteria
- scripts/ci.sh exits 0
- @codex review not reposted on same PR without a new push
- @codex review still posted after each new push
- All tests pass
