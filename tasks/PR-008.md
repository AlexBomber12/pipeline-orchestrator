# PR-008: Pipeline runner state machine

Branch: pr-008-runner

## Scope

Create the core PipelineRunner class that implements the state machine: PREFLIGHT -> IDLE -> CODING -> WATCH -> FIX -> MERGE / HUNG / ERROR. This is the heart of the daemon. One instance per connected repository.

## Files to create

### src/daemon/runner.py

Imports: asyncio, logging, subprocess, datetime, pathlib, redis.asyncio, and all relevant src modules (config, models, queue_parser, github_client, claude_cli).

Helper functions (module level):

repo_name_from_url(url: str) -> str
- Same logic as in src/web/app.py. Extract last URL segment, strip .git.

repo_owner_from_url(url: str) -> str
- Use github_client.get_repo_full_name(url) to get "owner/repo".

Class PipelineRunner:

__init__(self, repo_config: RepoConfig, app_config: AppConfig, redis_client: redis.Redis)
- Store repo_config, app_config, redis client
- Derive self.name from URL (repo_name_from_url)
- Derive self.owner_repo from URL (repo_owner_from_url)
- self.repo_path = f"/data/repos/{self.name}"
- Initialize self.state = RepoState(url=..., name=..., last_updated=datetime.now(timezone.utc))

async publish_state(self)
- Update self.state.last_updated to now
- Serialize self.state to JSON and redis.set("pipeline:{self.name}", ...)

log_event(self, event: str)
- Append {"time": iso_now, "state": current_state.value, "event": event} to self.state.history
- Keep only last 100 entries
- Log via logger.info with [self.name] prefix

async ensure_repo_cloned(self)
- If repo_path does not exist: git clone URL to repo_path (subprocess, timeout 120)
- If exists: git fetch origin {branch} (subprocess, timeout 60)
- Raise RuntimeError on failure

preflight(self) -> bool
- Run git status --porcelain in repo_path
- If stdout not empty: log_event, set state to ERROR with error_message, return False
- If clean: return True

async run_cycle(self)
- This is called once per poll interval by the daemon main loop.
- Call ensure_repo_cloned
- Call preflight, if fails -> publish_state and return
- Dispatch based on current self.state.state:
  - IDLE -> handle_idle()
  - WATCH -> handle_watch()
  - HUNG -> handle_hung()
  - ERROR -> if error_handler_use_ai: handle_error()
  - All others (CODING, FIX, MERGE) should not be persisted between cycles. If somehow in these states at cycle start, reset to IDLE.
- publish_state at end

async handle_idle(self)
- git pull origin {branch} in repo_path to get latest QUEUE.md
- Parse QUEUE.md via queue_parser.parse_queue
- Call get_next_task. If None: log "No tasks available", return.
- Set current_task, set state to CODING, publish_state
- Call handle_coding()

async handle_coding(self)
- Transient state: runs within a single cycle, does not persist.
- Call claude_cli.run_planned_pr(repo_path)
- If exit != 0: set ERROR with stderr, log, return
- After claude finishes: check github_client.get_open_prs for new PR
- If PR found: set current_pr, set state to WATCH, log
- If no PR: set ERROR "Claude CLI succeeded but no PR found", log

async handle_watch(self)
- If no current_pr: set IDLE, return
- Call github_client.get_open_prs, find PR matching current_pr.number
- If PR not found (merged externally or closed): set IDLE, clear current_pr and current_task, log, return
- Update current_pr with fresh data (ci_status, review_status, push_count)
- Decision:
  - ci_status SUCCESS + review_status APPROVED: call handle_merge()
  - review_status CHANGES_REQUESTED or ci_status FAILURE: call handle_fix()
  - review_status EYES or PENDING: check timeout. If exceeded -> set HUNG. Else log "waiting" and return.

async handle_fix(self)
- Transient state.
- Set state to FIX, publish_state, log
- Call claude_cli.fix_review(repo_path)
- If exit != 0: set ERROR with stderr, log, return
- Increment current_pr.push_count
- Update current_pr.last_activity to now
- Set state to WATCH, log "Fix pushed, iteration #{push_count}"

async handle_merge(self)
- Transient state.
- Log "Merging PR #{number}"
- Call github_client.merge_pr(owner_repo, pr_number)
- On success: set IDLE, clear current_pr and current_task, log
- On exception: set ERROR with message, log

async handle_hung(self)
- If hung_fallback_codex_review enabled:
  - Post "@codex review" comment on PR via github_client.post_comment
  - Reset last_activity to now
  - Set state back to WATCH, log
- Else: log "skipping", set IDLE, clear current_pr and current_task

async handle_error(self, error_context: str | None = None)
- context = error_context or self.state.error_message or "Unknown error"
- Call claude_cli.diagnose_error(repo_path, context)
- Parse response via claude_cli.parse_diagnosis
- "SKIP": set IDLE, clear task/PR/error, log
- "FIX": set IDLE, clear error, log with AI response
- "ESCALATE": log, keep ERROR state (dashboard will show it)

Timeout logic in handle_watch:
- last_activity comes from current_pr.last_activity
- If None, use self.state.last_updated as fallback
- Timeout = repo_config.review_timeout_min minutes

### tests/test_runner.py

Test the state machine transitions using mocks. Do NOT test actual git/claude/gh calls.

Create fixtures:
- mock_redis: async mock with set/get methods
- sample RepoConfig and AppConfig
- runner instance with mocked dependencies

Test cases:
- preflight returns False on dirty repo (mock git status returning output)
- preflight returns True on clean repo
- handle_idle with no tasks does not change state from IDLE
- handle_watch with approved + CI green calls handle_merge
- handle_watch with changes_requested calls handle_fix
- handle_watch with timeout sets HUNG
- handle_hung posts @codex review and returns to WATCH
- handle_merge success sets IDLE
- log_event appends to history and caps at 100

Use unittest.mock.patch or monkeypatch to mock subprocess.run, github_client functions, and claude_cli functions.
