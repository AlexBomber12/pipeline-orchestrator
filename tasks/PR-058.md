# PR-058: Upload staging cleanup guards

Branch: pr-058-upload-cleanup

- Type: bugfix
- Complexity: low

## Problem

The upload endpoint creates a staging directory, writes uploaded files to it, then stores a manifest in Redis pointing at the directory. If the Redis write fails after files land on disk, the staging directory leaks. Old staging directories can also persist across overwrites.

Over time, /data/uploads accumulates garbage.

## Root cause

`src/web/app.py` around lines 1282-1325 has no cleanup in the failure path after staging directory creation. The happy path cleans up after the daemon processes the upload, but failure modes (Redis unavailable, invalid manifest, disk error mid-write) leave orphan directories.

## Scope

Add try/finally cleanup guards around staging directory creation. Remove the directory if manifest write fails. Add a periodic sweep for abandoned staging directories older than a configurable threshold.

## Changes

### src/web/app.py

Wrap the upload handler flow in a cleanup guard. Staging directories are named with a unique upload key (e.g. `/data/uploads/<uuid>/`). Track the directory path and remove it on any exception between creation and successful Redis commit:

```python
staging_dir = None
try:
    staging_dir = _create_staging_dir(upload_key)
    _write_files_to_staging(staging_dir, uploaded_files)
    manifest = _build_manifest(staging_dir, uploaded_files)
    redis_result = await redis.set(manifest_key, json.dumps(manifest))
    if not redis_result:
        raise RuntimeError("redis set returned falsy value")
    staging_dir = None  # commit successful, don't clean up
except Exception as exc:
    logger.error("Upload failed, cleaning up: %s", exc)
    raise
finally:
    if staging_dir is not None:
        shutil.rmtree(staging_dir, ignore_errors=True)
```

Add a helper for abandoned staging sweep:

```python
def _sweep_abandoned_staging(
    uploads_root: str,
    redis_keys: set[str],
    max_age_hours: int = 24,
) -> int:
    """Remove staging directories older than max_age_hours with no Redis key.

    Returns count of directories removed.
    """
    now = time.time()
    cutoff = now - max_age_hours * 3600
    removed = 0
    for entry in Path(uploads_root).iterdir():
        if not entry.is_dir():
            continue
        if entry.name in redis_keys:
            continue
        try:
            mtime = entry.stat().st_mtime
        except OSError:
            continue
        if mtime < cutoff:
            shutil.rmtree(entry, ignore_errors=True)
            removed += 1
    return removed
```

Call the sweep periodically. Options:
- On each upload (cheap, runs frequently)
- On daemon startup (once per daemon lifetime)
- Scheduled task via asyncio loop (every N hours)

Recommended: on daemon startup + on each upload for defense in depth.

### src/config.py and config.yml

Add optional config:

```yaml
daemon:
  upload_staging_max_age_hours: 24  # default
```

### tests/test_upload.py or tests/test_web_app.py

Add tests:
- test_upload_cleans_staging_on_redis_failure: mock redis.set to raise, expect staging directory removed after exception
- test_upload_cleans_staging_on_write_failure: mock file write to raise, expect directory removed
- test_upload_preserves_staging_on_success: redis success, expect directory still exists
- test_sweep_removes_old_orphans: create directory older than threshold, no redis key, expect removed
- test_sweep_preserves_young_directories: directory newer than threshold, expect preserved
- test_sweep_preserves_keyed_directories: directory older than threshold BUT matches redis key, expect preserved
