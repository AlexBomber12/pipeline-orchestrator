# PR-070: Async run_claude in merge handler

Branch: pr-070-async-merge-resolve
- Type: bugfix
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

In merge.py line 76, sync `claude_cli.run_claude(...)` blocks the event loop during merge conflict resolution. All other repos stall until conflict resolution completes (up to 300s timeout).

## Scope

Replace `claude_cli.run_claude(...)` with `await claude_cli.run_claude_async(...)` in handle_merge.

## Changes

### src/daemon/handlers/merge.py

Line 76: change `code, _stdout, _stderr = claude_cli.run_claude(` to `code, _stdout, _stderr = await claude_cli.run_claude_async(`. Adjust parameters if needed (run_claude_async uses `system_prompt_file` param).

### tests/

Update mock target from `claude_cli.run_claude` to `claude_cli.run_claude_async` in merge-related tests.

## Success criteria
- scripts/ci.sh exits 0
- All tests pass
- Merge conflict resolution no longer blocks event loop
