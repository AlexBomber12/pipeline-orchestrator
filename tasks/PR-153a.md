# PR-153a: Python e2e test infrastructure (conftest skeleton + smoke test)

- Type: feature
- Complexity: low
- Depends on: none
- Priority: 1

## Problem

The repo has 33 unit test files at 100% coverage but no end-to-end test layer that exercises the running daemon. Sprint F1.0 introduces such a layer in three small infrastructure PRs (PR-153a Python skeleton, PR-153b Docker test stack, PR-153c test config and dependencies) followed by behavioral test PRs (PR-154 series) and the GHA gate (PR-155 series). This PR is the first of those: it lays down the Python directory structure, the conftest fixtures Python-side, a single smoke test, and gitignore patterns. It does NOT add any docker compose, dependency, or config changes. Those are the next two PRs.

The smoke test does not need a docker stack to be useful: pytest can collect it and verify it imports cleanly, even though running it requires the test stack from PR-153b/c. The split exists because previous attempts to combine all infrastructure in one PR resulted in the coder stuck on edge cases of one part while other parts went unwritten.

## Scope

Create tests/e2e/ subtree with __init__ files, conftest.py with the Python-side fixtures (no docker invocations yet), test_smoke.py with one trivial test, README.md explaining the layout, and gitignore patterns for runtime artifacts. Configure pytest to NOT collect tests/e2e/ when running scripts/ci.sh (the existing fast tier).

This PR creates files but does NOT make them runnable end-to-end: dependencies (pytest-playwright, requests) are not yet installed and the docker stack does not exist. Smoke test is collected by pytest tests/e2e/ but will fail to run because requests is not in requirements.txt; that is fine for this PR. PR-153c adds requirements-test.txt and PR-153b adds the docker stack; only after both does pytest tests/e2e/ pass.

## Files to create

`tests/e2e/__init__.py`: empty file marking the package.

`tests/e2e/lib/__init__.py`: empty file marking the lib subpackage.

`tests/e2e/conftest.py`: Python module with fixture skeletons. Imports: pytest, os, time, json, subprocess, urllib.request, urllib.error, pathlib.Path. Do NOT import requests yet (not in main requirements.txt; PR-153c adds it). Use urllib.request for the few HTTP calls in this PR; PR-154 series may switch to requests once it is declared.

Constants at top of file:
- TEST_DASHBOARD_URL = "http://localhost:18800"
- TESTBED_SLUG = "AlexBomber12__pipeline-orchestrator-testbed"
- TESTBED_REPO = "AlexBomber12/pipeline-orchestrator-testbed"
- REPO_DIR = Path(__file__).resolve().parents[2]
- TEST_DATA_DIR = REPO_DIR / "tests/e2e/data"
- EVIDENCE_DIR = REPO_DIR / "tests/e2e/evidence"

Session fixture `dashboard_url`: returns TEST_DASHBOARD_URL.

Session fixture `testbed_slug`: returns TESTBED_SLUG.

Session fixture `testbed_url`: returns f"{TEST_DASHBOARD_URL}/repo/{TESTBED_SLUG}".

Function fixture `get_state`: returns a callable. The callable takes optional slug parameter (defaults to TESTBED_SLUG). It performs a GET to f"{TEST_DASHBOARD_URL}/api/states" using urllib.request.urlopen with timeout 5 seconds. On HTTP error or connection error, returns None. On success, parses JSON, finds entry whose slug field matches the parameter, returns that dict or None if not found.

Function fixture `wait_for_state`: returns a callable. The callable takes states (list of strings), timeout_sec (int default 30), slug (default TESTBED_SLUG), poll_interval_sec (float default 1.0). It loops: call get_state, check if returned dict is not None and dict["state"] is in states, return the dict if matched. Sleep poll_interval_sec between iterations. After timeout, raise TimeoutError with a message including the last seen state and the expected states.

Function fixture `take_screenshot`: returns a callable. The callable takes a name string. It uses the Playwright `page` fixture (must be requested by the test that wants screenshots). Creates EVIDENCE_DIR if missing. Writes screenshot to EVIDENCE_DIR / f"{timestamp}_{name}.png" where timestamp is YYYYMMDD-HHMMSS UTC. Note: this fixture imports playwright lazily (inside the function body) to keep conftest.py importable even if playwright is not yet installed. PR-153c adds playwright to requirements-test.txt.

NO docker-related fixtures in this PR. NO upload_zip. NO make_task_zip. NO reset_testbed_clean. Those land in PR-154a (shim mechanism) along with the tests that use them.

`tests/e2e/test_smoke.py`: one test `test_smoke_imports_only`. The test function takes no fixtures, imports nothing extra. Body: assert 1 + 1 == 2. The test exists to verify pytest collection and discovery work for tests/e2e/. It does NOT verify the dashboard, the daemon, or anything functional. PR-153b adds a stack-aware smoke; PR-153a is just the Python plumbing.

`tests/e2e/README.md`: 30 to 50 lines explaining the layer.

Sections:
- What this is: integration test layer running against a docker compose test stack (introduced fully in PR-153b) on port 18800, isolated from production stack on 8800.
- Status of this PR (PR-153a): Python skeleton only. Not yet runnable end-to-end.
- Directory layout: __init__.py marker, lib/ for shared helpers, evidence/ for screenshots (gitignored), data/ for runtime artifacts (gitignored), test_*.py for behavioral tests added in PR-154 series.
- How to run (forward-looking, will be valid after PR-153c): pip install -r requirements-test.txt, playwright install chromium, then pytest tests/e2e/. For local debugging once PR-155c lands, scripts/test-e2e.sh.
- Why not in scripts/ci.sh: the coder runs ci.sh inside the daemon container which has no host docker socket; integration tests require docker compose against a separate stack and therefore must run from a host-level entry point. Mainstream CI pattern.

## Success criteria

1. `python -m pytest -q` (default invocation, equivalent to scripts/ci.sh second step) collects ONLY tests under tests/ excluding tests/e2e/. Coverage for src/ remains 100%. The pytest output shows the same number of tests collected as before this PR.

2. `python -m pytest tests/e2e/ -q` collects exactly one test (test_smoke_imports_only) and that test passes. No HTTP, no docker, no Playwright invoked at this stage.

3. `python -c "from tests.e2e import conftest"` succeeds (conftest is importable without docker stack running and without playwright installed). The take_screenshot fixture imports playwright lazily inside its body so that a missing playwright does not break import.

4. `git status` after the test run shows no tracked-file changes; `tests/e2e/data/` and `tests/e2e/evidence/` may exist locally but are gitignored.

5. `scripts/ci.sh` exits 0 unchanged.

6. ruff check src tests passes. Note that conftest.py and test_smoke.py are under tests/, so ruff will lint them; ensure no linter errors.

7. The pyproject.toml change is non-destructive: if the file already had `[tool.pytest.ini_options]` with other keys (e.g. asyncio_mode), those keys remain.

8. tests/e2e/README.md exists and is under 60 lines, with the five sections listed in the spec above.
