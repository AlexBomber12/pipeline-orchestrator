# PR-154c: e2e test 2 — stop and resume during CODING

## Problem

PR-154b validated the happy-path PLANNED PR lifecycle. This PR adds the second behavioral test: user clicks Stop while daemon is in CODING, then later clicks Resume to continue. The test verifies that Stop cleanly abandons the in-flight task and that Resume picks up subsequent uploads. This protects user-initiated interruption flow which is core dashboard functionality.

The test uses a "slow" shim scenario which sleeps 30 seconds between checkout and commit. This deliberately holds the daemon in CODING state long enough for the test to send Stop while CODING is still active. Without the slow scenario, the success path completes faster than the test can react.

## Scope

Add the "slow" scenario to tests/e2e/lib/coder_shim.sh (extending the case statement from PR-154a). Add tests/e2e/test_stop_and_resume.py with one test. No other changes.

## Files to touch

`tests/e2e/lib/coder_shim.sh` (created in PR-154a): extend the case statement to support a new scenario "slow".

The slow scenario does the same as success but with a 30-second sleep inserted between `git checkout` and the marker file creation. Pseudo-code structure for the case branch:

```
slow)
  REPO_DIR="/data/repos/AlexBomber12__pipeline-orchestrator-testbed"
  cd "$REPO_DIR"
  git config user.email "shim@test.invalid"
  git config user.name "Shim Coder"

  # parse PR_ID and BRANCH from QUEUE.md (same logic as success scenario,
  # extract into a shell function shared between scenarios to avoid duplication)
  parse_queue_doing  # sets PR_ID and BRANCH

  git fetch origin "$BRANCH" 2>/dev/null && git checkout "$BRANCH" || git checkout -b "$BRANCH"

  sleep 30

  MARKER="tests/e2e-shim-marker.txt"
  mkdir -p "$(dirname "$MARKER")"
  echo "shim slow marker for ${PR_ID} at $(date -u +%Y-%m-%dT%H:%M:%SZ)" >> "$MARKER"
  git add "$MARKER"
  git commit -m "${PR_ID}: shim slow implementation"
  git push -u origin "$BRANCH"

  if ! gh pr view "$BRANCH" >/dev/null 2>&1; then
    gh pr create --base main --head "$BRANCH" --title "${PR_ID}: shim" --body "Slow shim PR for testing"
  fi
  PR_URL=$(gh pr view "$BRANCH" --json url -q .url)
  gh pr comment "$PR_URL" --body "@codex review"
  exit 0
  ;;
```

Important: refactor the QUEUE.md parsing into a shared shell function `parse_queue_doing` to avoid duplicating the awk logic across scenarios. The function sets PR_ID and BRANCH variables. This refactor is part of this PR.

## Files to create

`tests/e2e/test_stop_and_resume.py`: one test `test_stop_during_coding_then_resume_picks_next_task`.

Fixtures used: dashboard_url, testbed_slug, wait_for_state, get_state, upload_zip, make_task_zip, reset_testbed.

Procedure:
1. Wait for daemon IDLE: `wait_for_state(["IDLE"], timeout_sec=30)`.
2. Open `coder_shim("slow")` context. Inside the context:
3. Build zip with pr_id=801, upload, assert HTTP status 200 or 201.
4. Wait CODING: `wait_for_state(["CODING"], timeout_sec=30)`.
5. Send Stop via `requests.post(f"{dashboard_url}/repos/{testbed_slug}/stop", timeout=10)`. Assert status 200, 204, or HTMX 200 (`text/html` body, follow redirect convention used by the app).
6. Wait for daemon to acknowledge stop: `wait_for_state(["IDLE", "PAUSED"], timeout_sec=30)`. Both PAUSED and IDLE are accepted; the implementation may use either.
7. Verify state shows no current task: `state = get_state(); assert state["state"] in ("IDLE", "PAUSED"); assert state.get("current_task") is None or state["current_task"].get("pr_id") != 801`.
8. Send Resume: `requests.post(f"{dashboard_url}/repos/{testbed_slug}/resume", timeout=10)`. Assert HTTP success.
9. Exit the slow shim context (the with block ends naturally; the next steps use a fresh context with success scenario).
10. Open `coder_shim("success")` context. Inside:
11. Build fresh zip with pr_id=802, upload, assert HTTP success.
12. Wait CODING: `wait_for_state(["CODING"], timeout_sec=30)`. This is the key assertion: daemon resumed work after Stop and picked the new task.
13. Wait IDLE: `wait_for_state(["IDLE"], timeout_sec=180)`. PR-802 completes naturally and is auto-merged.
14. Final: `state = get_state(); assert state["state"] == "IDLE"`.

The test does NOT verify the original PR-801 ever completes. Stop semantics are "abandon current task gracefully"; the test only verifies abandonment AND subsequent task acceptance succeed. PR-801's orphaned branch on testbed (if any) is cleaned up by reset_testbed teardown.
