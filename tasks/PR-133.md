# PR-133: Light/dark theme toggle

Branch: pr-133-theme-toggle
- Type: feature
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

The dashboard is dark-mode only. Some users prefer light mode (daylight visibility, accessibility, preference). There is no toggle.

Dark should remain the default (it is what exists and most users expect for a developer tool). The toggle should be a small icon in the top navigation, persisting the choice in localStorage so it survives reloads.

## Scope

Add a sun/moon icon button in the top navigation bar. Clicking toggles between `data-theme="dark"` (default) and `data-theme="light"` on the `<html>` or `<body>` element. Persist the choice in localStorage under key `theme`.

On page load:
1. Read `localStorage.theme`.
2. If set, apply that theme.
3. If not set, check `prefers-color-scheme` media query and honor the user's OS preference.
4. If neither, default to dark.

Light theme color palette: invert the current palette sensibly.
- `--bg-0` (currently near-black) becomes near-white
- `--bg-1`, `--bg-2`, `--bg-3` become progressively light grays
- Text colors: near-white text becomes near-black
- Accent colors (cyan-ish in dark) can stay similar or shift slightly for contrast
- Keep semantic colors (green/yellow/red/orange) but tune saturation for light-mode readability

Do NOT do this as a CSS-variable-only swap without testing - check every component renders readably in both themes. Common trap: dark borders that vanish on light backgrounds.

## Files to modify

- `src/web/templates/base.html`: add the toggle button in the nav bar. Add `data-theme` attribute to html/body. Add JS for toggle and localStorage. Add `<meta name="color-scheme" content="dark light">`.
- CSS: define light-mode variable overrides under `html[data-theme="light"]`. Ensure all components use variables, not hardcoded colors.
- Tests: render test in both themes if feasible; otherwise document manual QA checklist in PR description covering the main pages (index, repo, settings).

## Success criteria

- Sun/moon icon visible in top nav.
- Click toggles theme; choice persists across reloads via localStorage.
- Fresh user with OS set to "light" sees light mode by default; OS set to "dark" sees dark mode by default.
- Explicit user choice overrides OS preference.
- All main pages (index, repo detail, settings) render readably in light mode - no invisible text, no vanishing borders.
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
