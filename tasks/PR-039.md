# PR-039: Honest dashboard states

Branch: pr-039-honest-states

## Scope

Dashboard sometimes shows IDLE when real state is unknown (Redis missing, decode failed, runner not initialized). This masks problems and erodes trust in the dashboard. Add explicit visual indicators for degraded states.

## Changes

### src/web/app.py

In the fallback code that creates a default RepoState when Redis read fails or key is missing:

1. When Redis key does not exist (runner never published state): add a visual marker. Set `error_message` to "Waiting for daemon to initialize" instead of showing clean IDLE.

2. When Redis value fails to decode (JSON parse error): set state to ERROR with message "State decode failed" instead of silently falling back to IDLE.

3. When Redis connection fails entirely: show a banner on the dashboard "Redis connection lost" instead of showing stale or default states.

Add a helper:
```python
def _get_repo_state_safe(redis_client, name: str, url: str) -> tuple[RepoState, str | None]:
    """Return (state, warning). Warning is non-None when state is synthetic."""
    try:
        raw = await redis_client.get(f"pipeline:{name}")
    except Exception:
        return _default_state(name, url), "Redis unavailable"
    if raw is None:
        return _default_state(name, url), "Awaiting daemon initialization"
    try:
        return RepoState.model_validate_json(raw), None
    except Exception:
        return _default_state(name, url, error="State decode failed"), "State decode error"
```
