# PR-045: FIX idle timeout based on last push

Branch: pr-045-idle-timeout

## Scope

Replace absolute fix_review_timeout_sec with idle-based fix_idle_timeout_sec that measures time since the last push, not since start. A productive Claude fixing 11 review comments over 90 minutes should never be killed as long as it keeps pushing. If no push for fix_idle_timeout_sec (default 1800), kill the process.

## Changes

### src/daemon/runner.py

Add idle monitor coroutine:
```python
async def _monitor_fix_idle(self, pr_number: int, idle_limit: int):
    last_known_push = time.monotonic()
    while True:
        await asyncio.sleep(60)
        latest_push_at = await self.github.get_branch_last_push_time(
            self.repo_id, pr_number
        )
        if latest_push_at and latest_push_at > last_known_push:
            last_known_push = latest_push_at
            self.log_event("FIX: Claude pushed, resetting idle timer")
        elapsed = time.monotonic() - last_known_push
        if elapsed > idle_limit:
            self.log_event(
                f"FIX: idle timeout ({idle_limit}s since last push), killing"
            )
            raise asyncio.CancelledError
```

In handle_fix, run Claude and idle monitor concurrently:
```python
idle_limit = self.config.fix_idle_timeout_sec
claude_task = asyncio.create_task(
    claude_cli.fix_review_async(self.repo_path, model=self.model)
)
idle_monitor = asyncio.create_task(
    self._monitor_fix_idle(pr_number, idle_limit)
)
try:
    code, stdout, stderr = await claude_task
finally:
    idle_monitor.cancel()
```

Remove all references to fix_review_timeout_sec.
