# PR-014: Config writer and hot reload

Branch: pr-014-config-writer

## Scope

Add ability to write config.yml from the web app (for settings page) and make the daemon re-read config.yml on each cycle so changes take effect without restart.

## Files to modify

### src/config.py

Add function:

save_config(config: AppConfig, path: str = "config.yml") -> None
- Serialize AppConfig to dict via model_dump()
- Convert repositories list to list of dicts
- Write to YAML file with yaml.dump, default_flow_style=False, sort_keys=False
- Use atomic write: write to tmp file, then os.replace to target path (prevents corruption on crash)

Add function:

add_repository(url: str, path: str = "config.yml", **overrides) -> AppConfig
- Load current config
- Check if URL already exists (by comparing normalized URLs - strip .git, trailing slash)
- If exists, raise ValueError("Repository already configured: {url}")
- Create RepoConfig with url and any overrides (branch, auto_merge, etc.)
- Append to repositories list
- Save and return updated config

remove_repository(url: str, path: str = "config.yml") -> AppConfig
- Load current config
- Find repo by URL (normalized comparison)
- If not found, raise ValueError("Repository not found: {url}")
- Remove from list
- Save and return updated config

update_repository(url: str, path: str = "config.yml", **updates) -> AppConfig
- Load current config
- Find repo by URL (normalized comparison)
- If not found, raise ValueError
- Update fields from **updates (only known fields: branch, auto_merge, review_timeout_min, poll_interval_sec)
- Save and return updated config

update_daemon_config(path: str = "config.yml", **updates) -> AppConfig
- Load current config
- Update daemon section fields (poll_interval_sec, review_timeout_min, hung_fallback_codex_review, error_handler_use_ai)
- Save and return updated config

### src/daemon/main.py

Modify the main loop: re-read config.yml at the start of each cycle (not every poll, but every N polls - configurable, default every 5 cycles = every 5 minutes at 60s interval).

When config changes detected:
- New repos: create new PipelineRunner instances
- Removed repos: stop tracking (do not run_cycle on them anymore)
- Changed settings: update app_config on existing runners

Compare by serializing to JSON and checking equality - simple and reliable.

### tests/test_config.py (modify)

Add tests:
- save_config writes valid YAML that load_config can read back
- save_config atomic write (target file exists with old content, verify new content after save)
- add_repository adds new repo
- add_repository raises on duplicate URL
- add_repository normalizes URL (.git, trailing slash)
- remove_repository removes repo
- remove_repository raises on missing URL
- update_repository updates specific fields
- update_daemon_config updates daemon fields

### tests/test_daemon_main.py (modify)

Add test:
- Config reload detects new repository and creates runner
