# PR-046: Per-repo poll interval

Branch: pr-046-per-repo-poll

## Scope

Add per-repo poll intervals so repos can have different check frequencies. Instead of sleeping daemon.poll_interval_sec for all repos, track last_run per runner and skip repos whose interval has not elapsed.

## Changes

### src/daemon/main.py

Track last_run per runner:
```python
last_run: dict[str, float] = {}

for key, runner in list(runners.items()):
    interval = runner.repo_config.poll_interval_sec
    now = time.monotonic()
    if now - last_run.get(key, 0) < interval:
        continue
    last_run[key] = now
```

### src/config.py

Re-add per-repo poll interval to RepoConfig:
```python
poll_interval_sec: int = 60
```

Re-add to _REPO_FIELDS.

### tests/test_daemon_main.py

- test_per_repo_poll_interval: verify repos with different intervals are polled at different rates

### tests/test_config.py

- test_repo_poll_interval_default: verify default 60

## Success criteria
- scripts/ci.sh exits 0
- Per-repo poll_interval_sec controls polling frequency
- Repos with longer intervals are skipped when not yet due
