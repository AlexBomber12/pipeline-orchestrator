# PR-009: Daemon main loop

Branch: pr-009-daemon-main

## Scope

Create the daemon entry point that initializes PipelineRunner instances for each configured repo and runs the poll loop. This makes `docker compose up daemon` actually work.

## Files to create

### src/daemon/main.py

Import: asyncio, logging, os, redis.asyncio, src.config.load_config, src.daemon.runner.PipelineRunner

Configure logging at module level: level INFO, format "%(asctime)s [%(name)s] %(levelname)s: %(message)s", datefmt "%Y-%m-%d %H:%M:%S".

async def main():
- Load config via load_config()
- Connect to Redis via REDIS_URL env var (default redis://localhost:6379/0), with decode_responses=True
- Log startup message with number of repositories
- If no repositories configured: log warning, but do not exit (daemon stays alive waiting for config change)
- Create PipelineRunner instance for each repo in config
- Enter infinite loop:
  - For each runner: call run_cycle() in try/except. On exception, log error with exc_info=True but continue to next runner.
  - asyncio.sleep(config.daemon.poll_interval_sec)

if __name__ == "__main__": asyncio.run(main())

### Verify docker-compose.yml

The daemon service command is `python -m src.daemon.main`. Verify this matches the module path. If it does not, fix docker-compose.yml in this PR.

### tests/test_daemon_main.py

- Test that main() logs warning when no repos configured (mock everything, check log output)
- Test that exception in one runner does not crash the loop (mock runner.run_cycle to raise on first runner, verify second runner still runs)
