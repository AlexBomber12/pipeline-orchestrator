# PR-051: Check for existing open PR before CODING

Branch: pr-051-coding-pr-guard

- Type: bugfix
- Complexity: low

## Problem

When a task reaches HUNG and `hung_fallback_codex_review=False`, handle_hung clears current_pr and current_task, then transitions to IDLE. On the next cycle, handle_idle re-picks the same task (still TODO in queue) and proceeds to CODING. But the open PR from the previous attempt still exists on GitHub.

Consequences:
- Claude creates a duplicate PR attempt (or branch name collision breaks CODING)
- Two open PRs reference the same task
- Operator must manually close the orphan

## Root cause

Two code paths combine to produce the bug:

1. `src/daemon/runner.py:2109-2112` in handle_hung clears task context on fallback-disabled:
```python
self.log_event("hung fallback disabled, skipping")
self.state.current_pr = None
self.state.current_task = None
self.state.state = PipelineState.IDLE
```

2. `src/daemon/runner.py:1234-1265` in handle_idle picks the next task without checking for an existing open PR on that task's branch.

## Scope

Add a guard in handle_idle: before entering CODING, check if any open PR already exists with the selected task's branch. If found, attach to it and go to WATCH instead of running PLANNED PR.

Also reconsider handle_hung: instead of silently abandoning the task, leave it in HUNG or in a dedicated MANUAL state so the operator sees there's something to address.

## Changes

### src/daemon/runner.py

In `handle_idle` around line 1261-1265, before transitioning to CODING, check for existing PR:

```python
self.state.current_task = task
task_branch = task.branch

if task_branch:
    try:
        prs = github_client.get_open_prs(self.owner_repo)
        existing = next(
            (p for p in prs if p.branch == task_branch), None
        )
    except Exception as exc:
        self.log_event(
            f"IDLE: open PR check failed: {exc}; deferring task dispatch"
        )
        return

    if existing is not None:
        self.state.current_pr = existing
        self.state.state = PipelineState.WATCH
        self._rehydrate_last_push_at(existing)
        self.log_event(
            f"Task {task.pr_id} has existing open PR #{existing.number} "
            f"on {task_branch!r} -> WATCH (no duplicate CODING)"
        )
        await self.publish_state()
        return

self.state.state = PipelineState.CODING
self.log_event(f"Picked task {task.pr_id}: {task.title}")
await self.publish_state()
await self.handle_coding()
```

In `handle_hung` around lines 2109-2112, when fallback is disabled, do not clear task context and do not transition to IDLE. Leave the runner in HUNG so the dashboard surfaces it for operator action:

```python
self.log_event(
    "hung fallback disabled; leaving runner in HUNG for operator action. "
    "Resolve the PR manually or re-enable hung_fallback_codex_review."
)
# state remains HUNG, current_pr and current_task preserved
```

This prevents the silent transition back to IDLE and the subsequent duplicate PR risk. The dashboard already shows HUNG distinctly; operators can close or merge the PR manually.
