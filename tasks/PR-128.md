# PR-128: Play/Pause/Stop icon controls in UI

Branch: pr-128-control-icons-ui
- Type: feature
- Complexity: medium
- Depends on: PR-127
- Priority: 2
- Coder: any

## Problem

PR-127 adds backend endpoints for pause/resume/stop but no UI to trigger them. The operator still cannot control the daemon from the dashboard.

This PR adds three icon buttons (Play / Pause / Stop) on the repo card and in the repo detail page header, wired to the PR-127 endpoints via HTMX.

## Scope

Use inline SVG icons from Heroicons (MIT license, already-embedded pattern). Three icons:
- Play (triangle pointing right) - shown when `repo.user_paused` is true OR the daemon is idle with pending tasks
- Pause (two vertical bars) - shown when the daemon is active and not user-paused
- Stop (filled square, red/danger variant) - always visible when the daemon has an active coder process (any non-IDLE state)

Wire the icons to HTMX POST calls:
- Play: `hx-post="/repos/{name}/resume"`
- Pause: `hx-post="/repos/{name}/pause"`
- Stop: `hx-post="/repos/{name}/stop"` with `hx-confirm="Stop the running coder? The working tree may be left dirty."`

Icons are buttons, not text links. Use `type="button"` explicitly. Add clear hover states (opacity change + subtle background on hover). Minimum touch target size 44x44 pixels for mobile (Apple HIG). Use `aria-label` for screen-reader accessibility.

Mutual exclusivity: show exactly one of Play or Pause at a time based on state. Stop is independent.

Event propagation: the buttons are inside the repo card which is itself clickable (navigates to `/repo/{name}`). Stop click bubbling from the icon buttons so clicking a button does not also navigate the page. Use `event.stopPropagation()` or equivalent.

Do not change the "Start" badge logic yet - leave it as-is. The Play button is the real control; the badge is informational.

## Files to modify

- `src/web/templates/components/repo_cards.html`: add the three icon buttons in the header row next to "Upload tasks". Conditional rendering based on `repo.user_paused` and `repo.state.value`.
- `src/web/templates/repo.html`: add the same three buttons at the top of the detail page header, next to the state badge.
- `src/web/templates/base.html` or a new `_icons.html` partial: define reusable SVG icons.
- Tests: render tests covering the conditional visibility matrix (paused vs active, IDLE vs CODING vs ERROR).

## Success criteria

- All three icons visible and interactive on the repo card.
- Play visible when `user_paused=True`; Pause visible when `user_paused=False` and daemon is active.
- Stop visible during CODING/WATCH/FIX/MERGE states, hidden during IDLE/PAUSED.
- Click on any icon does not navigate to `/repo/{name}` (propagation stopped).
- Confirm dialog appears on Stop before the POST fires.
- Mobile-visible: buttons are at least 44x44 pixels on touch devices.
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
- Coverage stays at 100%.
