# PR-127: Pause/Stop control endpoints and persistent pause flag

Branch: pr-127-control-endpoints-backend
- Type: feature
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

The dashboard currently has no Pause or Stop controls. There is a conditional "Start" badge shown on the repo card when the queue has pending tasks, but it is a display-only element, not a button. The operator cannot pause the daemon between PRs (e.g. to inspect intermediate state) and cannot stop a running coder process.

Two distinct controls are needed with different semantics:
- **Pause**: graceful, idempotent, daemon finishes current PR to a terminal state (MERGE or ERROR) but does not pick up the next task from the queue. Persistent across daemon restarts.
- **Stop**: aggressive, kills the current coder subprocess and transitions to a paused state. Leaves the working tree as-is; operator must clean up manually if needed.

This PR introduces the backend: state fields, Redis persistence, and three FastAPI endpoints. The UI frontend is a separate PR.

## Scope

Add three HTTP POST endpoints in `src/web/app.py`:
- `POST /repos/{name}/pause` - sets `user_paused=True` in RepoState, returns 200. Idempotent.
- `POST /repos/{name}/resume` - clears `user_paused`, returns 200. Idempotent.
- `POST /repos/{name}/stop` - sets `user_paused=True` AND signals the coder subprocess to terminate (SIGTERM then SIGKILL after grace period). Returns 200.

Add a new field `user_paused: bool = False` to RepoState. Persist it to Redis via the existing state-save path so it survives daemon restart.

In the main runner loop, before `handle_idle` picks up a new task, check `self.state.user_paused` and short-circuit to a no-op poll if true. Log one "Paused by user, not picking up new tasks" event the first time paused is observed (debounced to avoid log spam).

For `stop`: add a `_terminate_current_coder()` helper on the runner that sends SIGTERM to the coder process (tracked via asyncio subprocess reference), waits up to 5 seconds, then sends SIGKILL. Set `user_paused=True` after termination.

Do not add UI yet - that comes in PR-128. This PR is backend + state + endpoints + tests only.

## Files to modify

- `src/models.py`: add `user_paused: bool = False` to RepoState.
- `src/web/app.py`: add three new endpoints.
- `src/daemon/runner.py`: add `user_paused` check in the main loop before task pickup. Add `_terminate_current_coder()` helper.
- `src/daemon/handlers/idle.py`: respect `user_paused` in `handle_idle` (do not pick up new task if paused).
- `src/state.py` or equivalent: ensure `user_paused` is part of the persisted/loaded state shape.
- Tests: endpoint tests covering idempotency, loop short-circuit test, stop test with mocked subprocess.

## Success criteria

- `curl -X POST http://localhost:8000/repos/<name>/pause` returns 200; subsequent IDLE cycles log the paused message once and do not pick up new tasks.
- `curl -X POST http://localhost:8000/repos/<name>/resume` returns 200; next IDLE cycle picks up a task normally.
- `curl -X POST http://localhost:8000/repos/<name>/stop` during an active CODING run terminates the coder process within 5 seconds and puts the repo into paused state.
- Daemon restart preserves `user_paused=True` state (verified via Redis inspection).
- `scripts/ci.sh` exits 0.
- Coverage stays at 100% for all modified files.
