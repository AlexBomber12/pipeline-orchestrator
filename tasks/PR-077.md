# PR-077: Runner uses plugin registry

Branch: pr-077-runner-plugin-integration
- Type: architecture
- Complexity: medium
- Depends on: PR-074, PR-075, PR-076
- Priority: 2
- Coder: any

## Problem

Runner still uses hardcoded `_get_coder()` with if/else for claude/codex. Adding a new coder requires editing runner code.

## Scope

Replace `_get_coder()` with registry lookup. Handler handle_coding/handle_fix call `plugin.run_planned_pr()` / `plugin.fix_review()`.

## Changes

### src/daemon/runner.py

Accept `CoderRegistry` in constructor. Replace `_get_coder`:
```python
def _get_coder(self) -> tuple[str, CoderPlugin]:
    coder_name = (self.repo_config.coder or self.app_config.daemon.coder).value
    return coder_name, self._registry.get(coder_name)
```

### src/daemon/handlers/coding.py, fix.py

Replace `coder_module.run_planned_pr_async(...)` with `await plugin.run_planned_pr(...)`.

### src/daemon/main.py

Build registry at startup, register Claude + Codex plugins, pass to runners.

### tests/

Update runner tests to use registry with mock plugins.

## Success criteria
- scripts/ci.sh exits 0
- Runner uses registry for all coder operations
- No direct claude_cli/codex_cli imports in runner or handlers (except error.py which always uses Claude for diagnosis)
- Adding a new coder requires only a new plugin file + registration
- All tests pass
