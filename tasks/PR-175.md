# PR-175: Light-theme dropdown fix (color-scheme dark)

Branch: pr-175-light-theme-dropdown-fix
- Type: ux
- Complexity: low
- Depends on: none
- Priority: 3
- Coder: any

## Problem

When user's OS or browser is in light theme, native HTML select dropdowns (the `<select>` elements like coder selector) inherit light-mode browser styling: white background, dark text. This clashes with the dashboard's dark UI theme. Dropdowns appear as bright white rectangles against the dark dashboard, visually jarring.

The fix is the standard CSS `color-scheme` property. Setting `color-scheme: dark` on the dashboard root tells browsers to render native form controls (selects, dropdowns, scrollbars) in dark theme regardless of OS preference.

## Scope

Add `color-scheme: dark` to dashboard root CSS. One-line CSS change. Optionally extend to specific select elements if root-level approach causes side effects elsewhere.

## Files to touch

`src/web/templates/base.html` (or wherever the root `<html>` or `<body>` styling lives): add `color-scheme: dark` to the appropriate scope.

Sketch (in the `<style>` block in base.html or in a linked CSS file):

```css
:root {
    color-scheme: dark;
}
```

This propagates to all native form controls within the dashboard. Browser will render select dropdowns, scrollbars, file inputs, etc. in dark theme.

If side effects appear (e.g., checkboxes look wrong), narrow the scope:

```css
select, input[type="text"], input[type="file"] {
    color-scheme: dark;
}
```

`tests/test_templates.py` (or `tests/test_dashboard.py`): assert `color-scheme: dark` is present in the rendered base.html CSS.

## Files NOT to touch

`src/daemon/`, `src/models.py`, `config.yml`: no backend changes.

`docker-compose.yml`, `Dockerfile`: untouched.

`src/web/templates/components/`: only base.html or main CSS; component templates inherit from root.
