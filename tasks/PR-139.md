# PR-139: Mobile responsive dashboard layout

Branch: pr-139-mobile-responsive
- Type: feature
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

On mobile devices the dashboard is hard to use:
- Repo cards squish because the 2-column grid (`md:grid-cols-2`) is fine but the card body has a 2-column grid of its own that overflows on narrow screens.
- Top navigation overlaps on small widths.
- Current Task / Current PR boxes on the detail page are side-by-side forced, making each unreadable.
- Tap targets for Play / Pause / Stop (from PR-128) may be too small for thumb use.
- Font sizes are tuned for desktop.

This PR adapts the layout for mobile (≤640px wide) while preserving desktop rendering.

## Scope

### Breakpoint strategy

Tailwind default `sm` breakpoint is 640px. Use `sm:` and `md:` prefixes to differentiate mobile from tablet/desktop.

### Repo cards index page

Current: `grid gap-4 md:grid-cols-2`. Keep this - single column on mobile is correct.

Inside each card, the inner `grid grid-cols-2 gap-4 text-xs mb-2` for Sprint progress / Current task is too cramped on mobile. Change to `grid-cols-1 sm:grid-cols-2` so mobile gets stacked rows.

### Repo detail page

Current: Current Task and Current PR boxes are side-by-side (one grid). On mobile they should stack.

Adjust to `grid-cols-1 lg:grid-cols-2` (or similar breakpoint appropriate for the layout).

Event log should remain full-width.

### Navigation

If the top nav has multiple items, collapse overflow into a hamburger menu on mobile. If the nav is already minimal, just make sure it wraps cleanly.

### Control buttons

Ensure Play / Pause / Stop icons from PR-128 have minimum 44x44px tap targets on mobile. On desktop they can be smaller. Use `p-2 sm:p-1` pattern or explicit `min-w-[44px] min-h-[44px] sm:min-w-0 sm:min-h-0`.

### Typography

Body text already readable. Verify headings are not oversized on narrow viewports (some use text-xl that may be fine, some may need sm:text-xl tuning).

### What NOT to change

- Core color palette, layout direction, template structure.
- Functionality - this is visual/responsive only.

## Files to modify

- `src/web/templates/components/repo_cards.html`
- `src/web/templates/repo.html`
- `src/web/templates/base.html` (nav)
- Possibly `src/web/templates/settings.html`

## Success criteria

- Manual test on 375px wide viewport (iPhone SE) and 768px (iPad): no horizontal scroll, no text overflow, all controls tappable.
- Desktop rendering at 1280px+ unchanged visually.
- All Play/Pause/Stop buttons at least 44x44px on mobile.
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
- Include screenshots in the PR description: mobile (375px), tablet (768px), desktop (1280px+) for the index page and repo detail page.
