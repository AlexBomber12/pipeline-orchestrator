# PR-042: Async Claude CLI, idle timeout, heartbeat, per-repo scheduling

Branch: pr-042-async-cli

## Scope

Replace blocking subprocess.run in Claude CLI calls with asyncio.create_subprocess_exec. Currently one 30-minute FIX REVIEW blocks the entire daemon: no state updates, no dashboard refresh, no other repos processed. Replace absolute fix_review_timeout_sec with idle-based fix_idle_timeout_sec that measures time since the last push, not since start. A productive Claude fixing 11 review comments over 90 minutes should never be killed as long as it keeps pushing. Also implement per-repo poll intervals so repos can have different check frequencies.

## Changes

### src/claude_cli.py

Add async version of run_claude:
```python
async def run_claude_async(
    prompt: str,
    cwd: str,
    timeout: int = 600,
    model: str | None = None,
) -> tuple[int, str, str]:
    cmd = ["claude", "--print", "--dangerously-skip-permissions"]
    if model:
        cmd.extend(["--model", model])
    cmd.append(prompt)
    logger.info("running claude CLI with prompt: %s", prompt[:80])
    try:
        proc = await asyncio.create_subprocess_exec(
            *cmd,
            stdout=asyncio.subprocess.PIPE,
            stderr=asyncio.subprocess.PIPE,
            cwd=cwd,
            stdin=asyncio.subprocess.DEVNULL,
        )
        stdout_bytes, stderr_bytes = await asyncio.wait_for(
            proc.communicate(), timeout=timeout
        )
        stdout = stdout_bytes.decode("utf-8", errors="replace")
        stderr = stderr_bytes.decode("utf-8", errors="replace")
        code = proc.returncode or 0
    except asyncio.TimeoutError:
        proc.kill()
        await proc.wait()
        logger.error("claude CLI timed out after %ss", timeout)
        return (-1, "", f"Timeout after {timeout}s")
    except FileNotFoundError as exc:
        missing = getattr(exc, "filename", "")
        if missing and missing != cmd[0]:
            return (-1, "", f"cwd not found: {missing}")
        return (-1, "", "claude CLI not found")
    logger.info("claude CLI exited with code %s", code)
    return (code, stdout, stderr)
```

Add async wrappers:

4. Idle timeout for FIX state. Replace the current absolute timeout with push-activity monitoring:

```python
async def _monitor_fix_idle(self, pr_number: int, idle_limit: int):
    """Cancel FIX if no new push within idle_limit seconds.

    Polls GitHub for the latest push timestamp on the PR branch.
    Resets the idle timer every time a new push is detected.
    If idle_limit seconds pass with no new push, kills the Claude process.
    """
    last_known_push = time.monotonic()
    while True:
        await asyncio.sleep(60)
        latest_push_at = await self.github.get_branch_last_push_time(
            self.repo_id, pr_number
        )
        if latest_push_at and latest_push_at > last_known_push:
            last_known_push = latest_push_at
            self.log_event("FIX: Claude pushed, resetting idle timer")
        elapsed = time.monotonic() - last_known_push
        if elapsed > idle_limit:
            self.log_event(
                f"FIX: idle timeout ({idle_limit}s since last push), killing"
            )
            raise asyncio.CancelledError
```

In handle_fix, run Claude and idle monitor concurrently:

```python
idle_limit = self.config.fix_idle_timeout_sec
```

### src/github_client.py

Add method to get the latest push time for a PR branch:

```python
async def get_branch_last_push_time(
    self, repo_id: str, pr_number: int
) -> float | None:
    """Return monotonic-equivalent timestamp of the latest push event on PR branch.

    Uses the PR's head SHA updated_at or commits endpoint to determine
    when the last push happened. Returns None if unable to determine.
    """
```

Implementation: compare the head SHA from `GET /repos/{owner}/{repo}/pulls/{pr_number}` against the previously known SHA. If changed, return current time.monotonic(). This avoids needing to parse GitHub event timestamps and dealing with clock skew between GitHub and local server.

### src/daemon/main.py

Add per-repo poll interval support. Instead of sleeping daemon.poll_interval_sec for all repos, track last_run per runner and skip repos whose interval hasn't elapsed:

```python
last_run: dict[str, float] = {}

for key, runner in list(runners.items()):
    interval = runner.repo_config.poll_interval_sec
    now = time.monotonic()
    if now - last_run.get(key, 0) < interval:
        continue
    last_run[key] = now
```

## Success criteria
- scripts/ci.sh exits 0
- Dashboard updates every 30s during Claude CLI runs (heartbeat)
- FIX state timeout is measured from the last push, not from process start
- Claude working productively (pushing every N minutes) is never killed regardless of total duration
- Claude idle for fix_idle_timeout_sec (default 1800) is killed and state transitions to ERROR
- Settings UI shows fix_idle_timeout_sec with correct label
- fix_review_timeout_sec fully removed from config, settings, and runner
- Other repos continue to be processed while one repo's Claude run is in progress
- Per-repo poll_interval_sec actually controls polling frequency
