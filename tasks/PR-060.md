# PR-060: Settings UX - inline hints and logical grouping

Branch: pr-060-settings-ux

- Type: ux
- Complexity: low

## Problem

The Settings page shows daemon fields as a flat grid with minimal context. Two issues:

1. Most fields have no hint at all, or cryptic ones. Users (including the operator) don't know what sane values look like or what effect each field has. Examples:
   - "Poll interval (seconds)" with value 60 - is that a lot? A little? What's the tradeoff?
   - "Default review timeout (minutes)" - timeout of what? What happens when it fires?
   - "FIX idle timeout" has a useful qualifier in the label ("seconds since last push") but no hint about when it triggers

2. Related fields are visually separated. HUNG fallback checkbox lives in a different section from Default review timeout, but the checkbox is exactly the action that fires when the timeout hits. Users reading top-to-bottom don't see the connection.

Result: operator changes a field, isn't sure what it actually does, experiments with restarts to verify.

## Scope

Add a one-line hint under each daemon field. Keep hints short (user requirement: "nobody reads long docs"). Regroup fields by logical function so related settings are adjacent.

No behavioral changes. Pure UX improvement.

## Changes

### src/web/templates/components/settings_daemon.html

Restructure into four logical groups with headers:

**Timing**
- Poll interval (seconds) [60]
  *How often the daemon checks each repo. Lower = faster response, more API calls.*
- PLANNED PR timeout (seconds) [900]
  *Max time Claude gets to create a PR before being killed.*

**Review handling**
- Default review timeout (minutes) [20]
  *If Codex doesn't review within this time, PR flips to HUNG.*
- [✓] HUNG fallback: post @codex review
  *When a PR exceeds its review timeout, re-ping Codex on the anchor comment.*
- FIX idle timeout (seconds since last push) [1800]
  *Max time without a push during FIX before Claude is killed. Resets on each push.*

**Rate limits**
- Rate limit pause threshold (%) [90]
  *Daemon pauses work when subscription usage hits this percentage.*

**Error recovery**
- [✓] Error handler: use AI diagnosis
  *Let Claude analyze failures before falling back to the default recovery flow.*

### Per-repo settings

In the repositories table, the "Review timeout (min)" column per repo should show placeholder like `"20 (default)"` instead of a hardcoded `"60"` when the override is not set. This communicates inheritance from daemon settings.

Update src/web/templates/components/settings_repo_list.html to render the placeholder dynamically from `daemon.review_timeout_min`.

### Styling

- Section headers: same style as existing "Daemon Settings" h2 but smaller (h3 or styled span)
- Hints: `text-xs text-gray-500` (matches existing hint style on Rate limit threshold)
- Hints immediately below the input, inside the label
- Grid layout preserved within groups; groups stack vertically
