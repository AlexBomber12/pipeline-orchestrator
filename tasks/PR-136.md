# PR-136: Sprint progress bar real-time updates via SSE

Branch: pr-136-progress-bar-sse
- Type: feature
- Complexity: low
- Depends on: PR-135
- Priority: 2
- Coder: any

## Problem

The sprint progress bar on the repo card (`43 / 61 done`) only updates on full-page reload. If the daemon merges a PR the user does not see the progress until they refresh manually. On a running automat this feels stale.

PR-135 introduced the SSE infrastructure. This PR wires the first consumer: sprint progress updates.

## Scope

### Publisher: daemon publishes on queue change

In the places where `queue_done` or `queue_total` is updated (currently `handle_idle` and `recovery`), after the state save, publish:

```
await publish_repo_event(
    self.repo_name,
    "progress_updated",
    {"queue_done": self.state.queue_done, "queue_total": self.state.queue_total},
)
```

Debounce: if queue_done and queue_total are unchanged from last publish, skip. Track last-published values in-memory on the runner to avoid spamming identical events.

### Consumer: client-side SSE handler

Add a small script on the index page (where repo cards live) that:
1. Opens an `EventSource('/api/repos/{name}/events')` for each rendered repo card (up to a reasonable connection limit).
2. On `progress_updated` event, finds the progress bar DOM element for that repo and updates:
   - The text `{done} / {total} done`
   - The width of the inner bar (`width: {percent}%`)
3. Gracefully handles disconnect and auto-reconnects with exponential backoff.

Keep the initial render server-side (current HTMX / Jinja output) - SSE only updates after the page is loaded. This way reload still works without SSE.

Scope this to the index page only in this PR. Detail page can get SSE in a later PR if we find it useful.

## Files to modify

- `src/daemon/handlers/idle.py`: add publish call after `queue_done`/`queue_total` update.
- `src/daemon/recovery.py`: same.
- `src/web/templates/index.html` or included partial: add JS that opens EventSource and handles `progress_updated` events.
- `src/web/templates/components/repo_cards.html`: add `data-repo="{name}"` attribute on elements that need to be updated, so the JS can find them.
- Tests: publisher-side test that merging a PR triggers `progress_updated` publish; dedup test that unchanged values do not publish.

## Success criteria

- When a PR is merged, the progress bar on any open index page updates within 2 seconds without a page reload.
- Publish is debounced - unchanged queue_done/queue_total do not trigger new SSE messages.
- Full page reload still renders correct progress server-side (SSE is additive, not required).
- Reconnect works if server restarts (client re-establishes the event stream within 10 seconds with exponential backoff).
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
- Coverage stays at 100%.
