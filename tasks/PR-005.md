# PR-005: Dashboard base layout and repo list

Branch: pr-005-dashboard-base

## Scope

Create the FastAPI web application with base HTML template (Tailwind CDN + HTMX), index page showing connected repositories and their states. Dashboard reads state from Redis (published by daemon) with fallback to config.yml for repo list.

## Files to create

### src/web/app.py

FastAPI application:
- Jinja2 templates from src/web/templates/
- On startup: connect to Redis (REDIS_URL env var, default redis://localhost:6379/0)
- On shutdown: close Redis

Routes:
- GET / -> index.html (list of repos with states)
- GET /api/states -> JSON array of all repo states
- GET /partials/repo-list -> repo_list.html component (for HTMX swap)

Helper:
get_all_repo_states() -> list[RepoState]
- Load config.yml to get repo list
- For each repo, try redis.get("pipeline:{name}") and parse as RepoState
- If not in Redis, return RepoState with defaults (IDLE, no task, no PR)
- name = last segment of URL without .git

### src/web/templates/base.html
- DOCTYPE html, lang en
- Tailwind CSS via CDN (script src https://cdn.tailwindcss.com)
- HTMX via CDN (script src https://unpkg.com/htmx.org@2.0.4)
- Tailwind config: dark mode class, custom colors (surface-1: #0a0a0f, surface-2: #12121a, surface-3: #1a1a25, accent: #22d3ee, ok: #34d399, warn: #fbbf24, fail: #f87171, hung: #a78bfa)
- Google Fonts: DM Sans (400-700) for body, JetBrains Mono (400-500) for code
- Nav bar with "PO" logo badge and "Pipeline Orchestrator" title, clock on the right (JS updated every second)
- Main content area max-w-7xl centered
- Body: bg-surface-1, text-gray-200
- CSS keyframe pulse-dot animation (opacity 1 -> 0.3 -> 1, 2s ease-in-out infinite)
- Block: title, content

### src/web/templates/index.html
- Extends base.html
- Content block: div with hx-get="/partials/repo-list" hx-trigger="every 10s" hx-swap="innerHTML"
- Include components/repo_list.html inside the div

### src/web/templates/components/repo_list.html
- Header row: "Repositories" title + count
- If no repos: empty state card pointing to config.yml
- Grid of repo cards, each card is a link to /repo/{name}
- Each card shows:
  - State indicator dot (color-coded by state, pulse animation for active states CODING/WATCH/FIX)
  - Repo name (font-mono)
  - State badge (colored background + text matching state)
  - Current task PR ID (if any)
  - Current PR number (if any)
  - Error message truncated (if any, red background)
- State color mapping: IDLE=gray, CODING=accent(cyan), WATCH=warn(yellow), FIX=warn, MERGE=ok(green), HUNG=hung(purple), ERROR=fail(red)

## Design notes
- Dark theme, industrial/utilitarian feel
- No generic AI aesthetics
- Monospace for data, sans-serif for labels
- Minimal borders (border-white/5), subtle hover effects

## Success criteria
- scripts/ci.sh exits 0
- `uvicorn src.web.app:app` starts without errors
- GET / returns HTML with repo list (even if empty)
- GET /api/states returns JSON array
- GET /partials/repo-list returns HTML fragment
- HTMX auto-refreshes repo list every 10 seconds
