# PR-093: Capture merged-PR metrics with enriched RunRecord

Branch: pr-093-watch-merge-capture
- Type: bugfix+feature
- Complexity: medium
- Depends on: PR-080
- Priority: 1
- Coder: any

## Problem

Three issues handled together because all touch RunRecord schema and save paths; splitting would force sequential schema migrations.

**(1) Missing metrics for merged PRs.** Recent PRs panel shows "0 rows" despite 22+ merged PRs in sprint 8. Root cause: `_TERMINAL_METRICS_EXIT_REASONS` in `src/web/app.py` filters records to `{"error", "rate_limit", "success_merged"}`, but records from successfully-merged PRs only reach exit_reason `"coding_complete"`. `PipelineState.MERGE` is never assigned in the normal flow. All merges go through GitHub auto-merge, bypassing `handlers/merge.py`, which is the only site that writes `"success_merged"`.

**(2) Sparse RunRecord fields limit Phase 4 routing quality.** Current schema has 13 fields. Enough to distinguish slow-vs-fast and pass-vs-fail runs, but not enough to distinguish which KIND of task each coder handles well. Without files_touched, languages, diff size, or test-file ratio, routing has a 13-dim input where most signal is collapsed into task_type (which comes from manual header labeling). A 20-dim vector with diff-derived fields multiplies the information content of each record.

**(3) No stage attribution in RunRecord.** The orchestrator's long-term pipeline is Planner -> Coder -> Reviewer -> merge, with each stage a separate agent consuming tokens and contributing to cost-per-merged-PR. Today only the Coder stage is implemented, but RunRecord has no `stage` field to distinguish records from different stages. Adding the field later would require data migration on accumulated records. Adding it now with default `"coder"` is cheap and future-proof: future Planner runs write `stage="planner"`, future Reviewer runs write `stage="reviewer"`, PR-level cost aggregation (future PR-094) sums across stages for one task_id.

## Scope

Hook into WATCH when current PR disappears from open list. Determine merged-vs-closed via new `github_client.is_pr_merged` helper. Finalize run record with appropriate exit_reason. Expand `_TERMINAL_METRICS_EXIT_REASONS`.

Expand RunRecord schema with diff-derived instrumentation fields plus a `stage` field defaulting to `"coder"`. Populate diff fields in `_save_current_run_record` via a new `_compute_diff_stats` helper. All new fields are optional with sensible defaults so existing stored records parse without migration.

```
def is_pr_merged(repo: str, pr_number: int) -> bool | None:
    """Return True if PR is merged, False if closed without merge, None on lookup failure."""
    try:
        raw = run_gh([
            "api",
            f"repos/{repo}/pulls/{pr_number}",
            "--jq",
            "{state: .state, merged: .merged}",
        ])
    except RuntimeError:
        return None
    if isinstance(raw, dict):
        parsed = raw
    elif isinstance(raw, str):
        try:
            parsed = json.loads(raw)
        except json.JSONDecodeError:
            return None
    else:
        return None
    if parsed.get("merged") is True:
        return True
    if parsed.get("state") == "closed":
        return False
    return None
```

Add helper `_compute_diff_stats`:
```
def _compute_diff_stats(self, base_branch: str) -> dict:
    """Compute file/language/line stats for the current branch vs base."""
    try:
        numstat = subprocess.run(
            ["git", "diff", "--numstat", f"origin/{base_branch}...HEAD"],
            cwd=self.repo_path, capture_output=True, text=True, timeout=30,
        )
    except (subprocess.TimeoutExpired, OSError):
        return {}
    if numstat.returncode != 0:
        return {}
    added = 0
    deleted = 0
    files: list[str] = []
    for line in numstat.stdout.splitlines():
        parts = line.split("\t")
        if len(parts) != 3:
            continue
        try:
            added += int(parts[0]) if parts[0] != "-" else 0
            deleted += int(parts[1]) if parts[1] != "-" else 0
        except ValueError:
            continue
        files.append(parts[2])
    languages = sorted({_ext_to_language(f) for f in files if _ext_to_language(f)})
    test_files = sum(1 for f in files if "test" in f.lower() or f.startswith("tests/"))
    ratio = (test_files / len(files)) if files else 0.0
    return {
        "files_touched_count": len(files),
        "languages_touched": languages,
        "diff_lines_added": added,
        "diff_lines_deleted": deleted,
        "test_file_ratio": round(ratio, 3),
    }
```

Add `_ext_to_language` mapping (py -> python, ts -> typescript, js -> javascript, go -> go, rs -> rust, etc.; return None for unknown extensions).

Update `_save_current_run_record` to populate new fields before `await self._metrics_store.save(record)`:
```
if exit_reason in ("success_merged", "coding_complete", "closed_unmerged"):
    base_branch = self.app_config.daemon.default_base_branch or "main"
    stats = self._compute_diff_stats(base_branch)
    record.files_touched_count = stats.get("files_touched_count", 0)
    record.languages_touched = stats.get("languages_touched", [])
    record.diff_lines_added = stats.get("diff_lines_added", 0)
    record.diff_lines_deleted = stats.get("diff_lines_deleted", 0)
    record.test_file_ratio = stats.get("test_file_ratio", 0.0)
    record.base_branch = base_branch
```
