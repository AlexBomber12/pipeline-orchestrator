# PR-093: Capture merged-PR metrics via WATCH handler hook

Branch: pr-093-watch-merge-capture
- Type: bugfix
- Complexity: low
- Depends on: PR-080
- Priority: 1
- Coder: any

## Problem

Recent PRs metrics panel shows "0 rows / No completed PR runs recorded yet" despite 22+ merged PRs in sprint 8. Root cause: `_TERMINAL_METRICS_EXIT_REASONS` in `src/web/app.py` filters records to `{"error", "rate_limit", "success_merged"}`. Records from successfully-merged PRs only reach exit_reason `"coding_complete"` because `PipelineState.MERGE` is never set anywhere in the normal flow (grep confirms: no target-state assignment to MERGE outside of recovery). All merges happen externally through GitHub auto-merge, bypassing `handlers/merge.py`, which is the only site that writes `"success_merged"`.

Effect: every merged PR is indistinguishable from a mid-flight one at the record level; the UI filter drops them all; panel stays empty; no metrics accumulate. This blocks Phase 4 quality routing which requires 100+ correctly-attributed RunRecord samples.

## Scope

Hook into WATCH at the moment the current PR disappears from the open PRs list. Query GitHub for the actual state (merged vs closed-unmerged) via a new `github_client.is_pr_merged` helper. Finalize the run record with the appropriate exit_reason. Add `"closed_unmerged"` to `_TERMINAL_METRICS_EXIT_REASONS` so both outcomes surface in the metrics panel.

Does not modify `handlers/merge.py` or change the existing daemon-managed merge path — that path remains correct when it fires. This PR just covers the externally-merged case, which is the common one.

## Changes

### src/github_client.py

Add new helper below `get_pr_metadata`:
```
def is_pr_merged(repo: str, pr_number: int) -> bool | None:
    """Return True if PR is merged, False if closed without merge, None on lookup failure."""
    try:
        raw = run_gh([
            "api",
            f"repos/{repo}/pulls/{pr_number}",
            "--jq",
            "{state: .state, merged: .merged}",
        ])
    except RuntimeError:
        return None
    if isinstance(raw, dict):
        parsed = raw
    elif isinstance(raw, str):
        try:
            parsed = json.loads(raw)
        except json.JSONDecodeError:
            return None
    else:
        return None
    if parsed.get("merged") is True:
        return True
    if parsed.get("state") == "closed":
        return False
    return None
```

### src/daemon/handlers/watch.py

Replace the current "PR no longer open" block (the `if found is None:` branch at approximately lines 42-47) with:
```
if found is None:
    merged = github_client.is_pr_merged(self.owner_repo, current_number)
    if merged is True:
        await self._save_current_run_record("success_merged")
        self.log_event(f"PR #{current_number} merged externally -> IDLE")
    elif merged is False:
        await self._save_current_run_record("closed_unmerged")
        self.log_event(f"PR #{current_number} closed without merge -> IDLE")
    else:
        self.log_event(f"PR #{current_number} no longer open (state unknown) -> IDLE")
    self._current_run_record = None
    self.state.current_pr = None
    self.state.current_task = None
    self.state.state = PipelineState.IDLE
    return
```
