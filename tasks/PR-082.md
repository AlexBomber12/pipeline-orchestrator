# PR-082: Capture metrics in fix and merge handlers

Branch: pr-082-fix-merge-metrics
- Type: feature
- Complexity: low
- Depends on: PR-081
- Priority: 2
- Coder: any

## Problem

FIX iterations and merge timing not captured in RunRecord.

## Scope

Update RunRecord during FIX and MERGE handlers.

## Changes

### src/daemon/handlers/fix.py

On each FIX entry: `self._current_run_record.fix_iterations += 1` (if record exists).

### src/daemon/handlers/merge.py

On successful merge: `self._current_run_record.exit_reason = "success_merged"`, set `ended_at`, calculate `duration_ms`, save to MetricsStore. Reset `self._current_run_record = None`.

On merge failure: `exit_reason = "error"`, save.

### tests/

- test_fix_increments_iterations
- test_merge_finalizes_record
- test_merge_calculates_duration

## Success criteria
- scripts/ci.sh exits 0
- fix_iterations accurately reflects number of FIX cycles
- Merged PRs have complete RunRecord with duration
- All tests pass
