# PR-160: Per-test reset_testbed wipes tasks/ on testbed

Branch: pr-160-per-test-tasks-wipe
- Type: bugfix
- Complexity: low
- Depends on: none
- Priority: 1
- Coder: any

## Problem

The `_reset_testbed_session` session-scoped autouse fixture in `tests/e2e/conftest.py` correctly wipes `tasks/` on testbed main at the start of every pytest session. However the per-test `reset_testbed` fixture only closes open PRs:

```python
@pytest.fixture
def reset_testbed():
    _close_open_testbed_prs()
    yield
    _close_open_testbed_prs()
```

It does NOT wipe `tasks/` between individual tests. As a result, when test A pushes a task file (e.g. `PR-1777222119.md` for sigkill_recovery) and the test ends, the task file persists on testbed main. Test B then begins, uploads its own task file (e.g. `PR-1777220290.md` for stop_and_resume), and now testbed main has TWO task files in `tasks/`. Daemon's `_select_next_task_from_dag` sorts task files alphabetically and picks the older one (PR-1777222119), not the freshly-uploaded one for the current test. Test B fails because it expects the daemon to pick its own task.

This was masked by B-1 race (shim exited 0 before reaching push) until PR-159 fixed B-1. Now the race is gone, exposing the cleanup gap.

Diagnosis confirmed by Claude in PR-159 self-review (PR #217 commit 6c5fdfc), reproduced in CI run 24961823433 logs.

## Scope

Modify the per-test `reset_testbed` fixture in `tests/e2e/conftest.py` to also wipe `tasks/` on testbed main between tests. Reuse the existing `wipe_tasks_dir_on_main()` helper from `tests/e2e/lib/testbed_reset.py` (introduced in PR-157). Add unit test for the updated fixture behavior.

This PR does NOT change session-start cleanup. It does NOT change daemon-side behavior. It does NOT change shim or task selection logic. It is a pure test-infrastructure fix.

## Files to touch

`tests/e2e/conftest.py`: extend `reset_testbed` fixture. New behavior — call `wipe_tasks_dir_on_main()` in addition to `_close_open_testbed_prs()` both before yield and after yield.

Sketch:

```python
from tests.e2e.lib.testbed_reset import wipe_tasks_dir_on_main


@pytest.fixture
def reset_testbed():
    _close_open_testbed_prs()
    wipe_tasks_dir_on_main()
    yield
    _close_open_testbed_prs()
    wipe_tasks_dir_on_main()
```

Order of operations matters slightly: close PRs first (so branch deletion happens before task file removal), then wipe tasks/. After yield same order for symmetry.

The helper `wipe_tasks_dir_on_main()` returns `False` cleanly when `tasks/` does not exist, so calling it twice (session-start already wiped, then per-test re-wipe) is harmless. The function is documented as idempotent.

`tests/test_conftest_e2e.py` (or wherever conftest fixtures are unit-tested; create if missing): add test verifying that `reset_testbed` calls `wipe_tasks_dir_on_main` exactly twice (before yield, after yield). Mock both `_close_open_testbed_prs` and `wipe_tasks_dir_on_main` to track call counts.

If unit tests for conftest fixtures do not exist as a pattern in this repo, create a single test file `tests/test_e2e_conftest_helpers.py` at repo root tests/ level (NOT under tests/e2e/ which is excluded from default pytest collection per `pyproject.toml` norecursedirs).
