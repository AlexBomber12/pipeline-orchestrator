# PR-085: Status derivation from git

## Problem

Task status (TODO/DOING/DONE) stored in QUEUE.md and updated by daemon. Race condition when operator edits QUEUE.md while daemon modifies it. Status should be derived from git/GitHub state instead.

## Scope

New function that determines task status from branch state in git and open PRs on GitHub.

## Changes

### src/task_status.py (new)

```python
def derive_task_status(
    task_header: TaskHeader,
    merged_branches: set[str],
    open_pr_branches: set[str],
) -> TaskStatus:
    """Derive task status from git state.

    - Branch merged into main = DONE
    - Open PR for branch = DOING
    - Neither = TODO
    """
    if task_header.branch in merged_branches:
        return TaskStatus.DONE
    if task_header.branch in open_pr_branches:
        return TaskStatus.DOING
    return TaskStatus.TODO

def get_merged_branches(repo_path: str, base_branch: str) -> set[str]:
    """Return set of branches that have been merged into base_branch."""
    result = subprocess.run(
        ["git", "-C", repo_path, "branch", "--merged", f"origin/{base_branch}", "-r"],
        capture_output=True, text=True, timeout=10,
    )
    branches = set()
    for line in result.stdout.strip().splitlines():
        ref = line.strip().removeprefix("origin/")
        if ref and ref != base_branch:
            branches.add(ref)
    return branches
```
