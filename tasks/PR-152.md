# PR-152: Remove nonfunctional Start badge from repo cards

## Problem

`src/web/templates/components/repo_cards.html` lines 63-65 render a non-interactive `<span>` labelled "Start" when a repo is IDLE with pending queue tasks:

```
{% if repo.state.value == 'IDLE' and repo.queue_total > repo.queue_done %}
<span class="text-xs font-medium px-2 py-0.5 rounded border border-accent/30 text-accent">Start</span>
{% endif %}
```

This badge is purely decorative - no click handler, no HTMX binding. The functional element for starting a run is the Play icon button immediately to its left (rendered by `repo_controls`), which IS clickable and correctly wired to `/repos/{name}/resume`. The badge duplicates the semantic signal and risks confusing users who click it expecting a response.

## Scope

Delete the three lines (conditional + span + endif) from `repo_cards.html`. No other change. The Play button covers the semantic already.

## Files to touch

- `src/web/templates/components/repo_cards.html`: delete lines 63-65 (the `{% if %}` / `<span>Start</span>` / `{% endif %}` block)

## Files NOT to touch

- `repo_controls` macro in `_controls.html`
- Other templates that may have their own Start indicators
- Any Python code
- Tests that may snapshot-match the current HTML: update as needed to reflect the removal (allowed in the touched files list implicitly for tests).

If the implementer finds the fix requires touching any file outside the "Files to touch" list, stop and ask in the PR description. Do not proceed on your own.
