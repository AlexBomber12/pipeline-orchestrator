# PR-003: Queue parser

Branch: pr-003-queue-parser

## Scope

Create a parser that reads QUEUE.md files and extracts task list with statuses, dependencies, and metadata. Also create the task selection logic.

## Files to create

### src/queue_parser.py

Two public functions:

parse_queue(queue_path: str) -> list[QueueTask]
- Read QUEUE.md from given file path
- Parse each task block. A task block starts with `## PR-XXX: title`
- Extract fields from lines below the header:
  - `- Status: TODO|DOING|DONE` -> TaskStatus
  - `- Tasks file: path` -> task_file
  - `- Depends on: PR-XXX, PR-YYY` -> depends_on list
  - `- Branch: name` -> branch
- Return list of QueueTask objects in order they appear
- If file does not exist, return empty list

get_next_task(tasks: list[QueueTask]) -> QueueTask | None
- If any task has status DOING, return the earliest one
- Otherwise find the earliest TODO task whose all depends_on are DONE
- If no eligible task, return None

### tests/test_queue_parser.py
Create a fixture QUEUE.md content string and write to tmp_path.

Test cases:
- parse_queue on nonexistent file returns []
- parse_queue extracts correct number of tasks
- parse_queue extracts pr_id, title, status, task_file, depends_on, branch correctly
- get_next_task returns first TODO when no DOING exists and no dependencies
- get_next_task returns DOING task over TODO
- get_next_task skips TODO with unmet dependencies
- get_next_task returns None when all tasks are DONE
- get_next_task returns TODO when dependency is DONE

Use this sample QUEUE.md for tests:

```

- Status: DONE
- Tasks file: tasks/PR-001.md
- Branch: pr-001-bootstrap

## PR-002: Models
- Status: DONE
- Tasks file: tasks/PR-002.md
- Branch: pr-002-models
- Depends on: PR-001

## PR-003: Parser

- Depends on: PR-002

## PR-004: Client

```
