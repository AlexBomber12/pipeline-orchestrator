# PR-080: RunRecord schema and storage

Branch: pr-080-run-record-schema
- Type: feature
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

No per-run metrics collected. Cannot answer: how long do PRs take? Which coder is more reliable? How many FIX iterations per PR? What is the cost?

## Scope

Define RunRecord dataclass and MetricsStore for Redis-based storage. No integration with handlers yet.

## Changes

### src/metrics.py (new)

```python
@dataclass
class RunRecord:
    run_id: str              # uuid
    task_id: str             # "PR-067"
    profile_id: str          # "claude:opus:container"
    task_type: str           # from task file header
    complexity: str          # from task file header
    started_at: str          # ISO timestamp
    ended_at: str | None     # ISO timestamp
    duration_ms: int | None
    fix_iterations: int
    tokens_in: int
    tokens_out: int
    exit_reason: str         # success_merged, error, timeout, oom, rate_limit, hung
    operator_intervention: bool

class MetricsStore:
    def __init__(self, redis_client):
        self._redis = redis_client

    async def save(self, record: RunRecord) -> None:
        key = f"metrics:run:{record.run_id}"
        await self._redis.set(key, json.dumps(asdict(record)), ex=90*86400)
        await self._redis.lpush(f"metrics:repo:{record.task_id.split('-')[0]}", record.run_id)

    async def get(self, run_id: str) -> RunRecord | None: ...

    async def recent(self, limit: int = 20) -> list[RunRecord]: ...
```
