# PR-132: Grey state label when daemon has no updates for 30 seconds

Branch: pr-132-stalled-state-grey
- Type: feature
- Complexity: low
- Depends on: PR-131
- Priority: 2
- Coder: any

## Problem

If the daemon process crashes or hangs, the dashboard keeps showing the last-known state (e.g. CODING) with its pulse animation. The operator cannot tell the daemon is dead until they notice the timestamp "updated 5 minutes ago" which is easy to miss.

A visual grey-out on the state badge when the last state update is older than 30 seconds makes stall immediately obvious.

## Scope

Add JavaScript logic in `base.html` (or a dedicated script file) that:
1. Reads the `updated_at` timestamp of each repo (or the single repo on the detail page).
2. Every second, checks if `now - updated_at > 30 seconds`.
3. If yes, adds a `stalled` CSS class to the state badge and dot. The class applies grayscale filter + reduced opacity (e.g. `filter: grayscale(100%); opacity: 0.5;`). Also stops the pulse animation.
4. If no, removes the `stalled` class and restores normal rendering.

The threshold is 30 seconds as defined. Make it a constant in the JS so it is easy to tune later.

This applies to both the repo card grid and the detail page. Use the same JS helper for both.

Render a small "stalled" text hint next to the badge when in stalled mode, for clarity (e.g. small gray "stalled" label).

## Files to modify

- `src/web/templates/base.html`: add JS that polls the DOM and applies `stalled` class based on `updated_at`.
- CSS: add `.stalled { filter: grayscale(100%); opacity: 0.5; }` and ensure pulse-dot animation is suppressed when `.stalled` is applied.
- `src/web/templates/components/repo_cards.html`: ensure each card exposes its `updated_at` timestamp as a `data-updated-at` attribute on the state-badge container.
- `src/web/templates/repo.html`: same for the detail page.
- Tests: given backend behavior is unchanged, primary testing is visual. Optionally unit-test the JS helper with a small JS test setup (or add a Selenium/Playwright snapshot test if that infra exists). If no JS test infra, rely on manual verification and document in PR description.

## Success criteria

- When the page has been loaded and the backend stops updating a repo (simulate by stopping the daemon), within 30 seconds the state badge on that repo turns grayscale and stops pulsing, with a small "stalled" text hint.
- When the daemon resumes and the `updated_at` refreshes, the badge returns to normal appearance immediately on next tick.
- Works on both the repo card and the detail page.
- No flicker or infinite-update loop (check browser console for errors).
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
