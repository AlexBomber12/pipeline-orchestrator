Branch: pr-035-dashboard-ux

## Scope

## Changes

### src/web/templates/base.html

Add JS function to convert UTC timestamps to local time:
```javascript
function formatTimestamps() {
    document.querySelectorAll('[data-ts]').forEach(el => {
        const ts = el.dataset.ts;
        if (!ts) return;
        const dt = new Date(ts);
        if (isNaN(dt)) return;
        const diffMin = (Date.now() - dt.getTime()) / 60000;
        if (diffMin < 1) el.textContent = 'just now';
        else if (diffMin < 60) el.textContent = Math.round(diffMin) + ' min ago';
        else if (diffMin < 1440) el.textContent = Math.round(diffMin / 60) + 'h ago';
        else el.textContent = dt.toLocaleDateString([], {month: 'short', day: 'numeric'});
    });
}
document.addEventListener('DOMContentLoaded', formatTimestamps);
document.body.addEventListener('htmx:afterSwap', formatTimestamps);
```

### src/web/templates/components/event_log.html

1. Replace raw ISO timestamp with data-ts attribute for JS conversion:
```html
<span class="text-gray-500 shrink-0" data-ts="{{ entry.get('time', '') }}">
    {{ entry.get('time', '--:--:--') }}
</span>
```
The JS will overwrite the text content with local formatted time. The raw ISO is fallback before JS runs.

2. Fix scroll preservation. Change the HTMX swap strategy. Instead of hx-swap="outerHTML" on the whole section, split into:
- The header (title, count, filter buttons) stays outside the swap target
- The event list (ul) is the swap target with id="event-list-{{ repo.name }}"
- Use hx-swap="innerHTML" on the list container
- Add htmx:beforeSwap event handler to save scrollTop:
```html
<div id="event-list-wrapper-{{ repo.name }}"
     class="max-h-96 overflow-y-auto"
     hx-get="/partials/repo/{{ repo.name }}/events?filter={{ event_filter }}"
     hx-trigger="every 5s"
     hx-swap="innerHTML"
     hx-on::before-swap="this._scrollPos = this.scrollTop"
     hx-on::after-swap="this.scrollTop = this._scrollPos">
```

3. Remove the filter buttons section entirely. They add complexity without value at this stage. Can be re-added in a future PR if needed.
