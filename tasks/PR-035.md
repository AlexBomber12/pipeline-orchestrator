# PR-035: Repo card click and dashboard polish

Branch: pr-035-card-click

## Scope

Make entire repo card clickable (not just the repo name link). Remove non-functional event log filter buttons or make them work. Small visual polish.

## Changes

### src/web/templates/components/repo_cards.html

Wrap each card div in an anchor tag or use JS onclick to navigate to /repo/{name}. The card div itself becomes the click target. Exclude the "Upload tasks" button area from the click handler so file upload still works.

Option: use a wrapper div with cursor-pointer and onclick="window.location='/repo/{{ repo.name }}'" on the card div. Use event.stopPropagation() on interactive children (upload label, start button).

### src/web/templates/components/event_log.html

Either remove filter buttons entirely OR fix them:
- "Errors" filter: show only entries where state == "ERROR"
- "State changes" filter: show only entries where the state differs from the previous entry

The filter logic should be in the web app route that renders the partial, using the filter query param that is already passed.

### src/web/app.py

If keeping filters: verify the /partials/repo/{name}/events route correctly filters events based on the filter parameter. Currently it should be filtering but may have a bug.

### tests/test_web_app.py

- test_repo_card_has_click_handler: verify onclick or anchor in card HTML
- test_event_filter_errors: verify only ERROR events returned when filter=errors
- test_event_filter_state_changes: verify state change events only

## Success criteria
- scripts/ci.sh exits 0
- Clicking anywhere on repo card navigates to detail page
- Upload button still works without navigating
- Event log filters work or are removed
