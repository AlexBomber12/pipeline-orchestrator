## Problem

The first GHA integration run after PR-155a/b merged failed because the testbed repository (https://github.com/AlexBomber12/pipeline-orchestrator-testbed) had stale state: 22 task files in tasks/, a populated QUEUE.md from manual day5 testing-week, and no automated cleanup mechanism. Daemon-test on session start saw stale DOING tasks (PR-301 specifically) and started a CODING flow on them instead of starting clean. The shim could not progress these tasks because its expected workflow assumes tests upload their own task files; subsequent tests blocked on daemon stuck in this stale work.

The state was cleaned manually on 2026-04-25 (open PRs closed, non-main branches deleted, tasks/ removed from main). But without automation, the same problem recurs every time a future test or coder leaves debris on testbed.

This PR adds a session-scoped pytest fixture that runs at the start of every e2e test session and resets the testbed to a known-clean state: no open PRs, no non-main branches, empty tasks/ on main. The fixture is autouse=True so all tests benefit without explicit opt-in.

This addresses OBS-15 from roadmap.md (testbed cleanup) which was deliberately deferred from PR-153a to avoid the original PR-153 over-scoping mistake. Now that we have empirical evidence the cleanup is required, we add the focused PR.

## Scope

Add a session-scoped autouse fixture `_reset_testbed_session` in tests/e2e/conftest.py. Add unit tests for the helper functions in a new tests/e2e/lib/test_testbed_reset.py file. No changes to docker-compose.test.yml, config files, or src/.

The fixture runs ONCE per pytest session, BEFORE any other tests. It does NOT run between individual tests in the same session (the existing `reset_testbed` function fixture handles per-test PR cleanup). The session-scoped reset is heavier: it includes branch deletion and tasks/ wipe.

## Files to create

`tests/e2e/lib/testbed_reset.py`: new module with three helper functions and one orchestrator function.

```python
"""Helpers to reset the testbed repository to a known-clean state.

Used by the session-scoped fixture at pytest session start. NOT used between
individual tests; the per-test reset_testbed fixture in conftest handles the
lighter per-test cleanup.
"""
from __future__ import annotations

import subprocess
from pathlib import Path
import tempfile
import shutil

TESTBED_REPO = "AlexBomber12/pipeline-orchestrator-testbed"
TESTBED_URL = f"https://github.com/{TESTBED_REPO}.git"


def close_all_open_prs() -> int:
    """Close every open PR on testbed with --delete-branch. Returns count closed."""
    listing = subprocess.run(
        ["gh", "pr", "list", "-R", TESTBED_REPO, "--state", "open",
         "--json", "number", "--jq", ".[].number"],
        capture_output=True, text=True, check=False, timeout=30,
    )
    closed = 0
    for line in listing.stdout.splitlines():
        n = line.strip()
        if not n:
            continue
        result = subprocess.run(
            ["gh", "pr", "close", n, "-R", TESTBED_REPO, "--delete-branch"],
            capture_output=True, text=True, check=False, timeout=30,
        )
        if result.returncode == 0:
            closed += 1
    return closed


def delete_non_main_branches() -> int:
    """Delete every branch on testbed that is NOT main. Returns count deleted."""
    listing = subprocess.run(
        ["gh", "api", f"repos/{TESTBED_REPO}/branches", "--jq", ".[].name"],
        capture_output=True, text=True, check=False, timeout=30,
    )
    deleted = 0
    for line in listing.stdout.splitlines():
        name = line.strip()
        if not name or name == "main":
            continue
        result = subprocess.run(
            ["gh", "api", "-X", "DELETE",
             f"repos/{TESTBED_REPO}/git/refs/heads/{name}"],
            capture_output=True, text=True, check=False, timeout=30,
        )
        if result.returncode == 0:
            deleted += 1
    return deleted


def wipe_tasks_dir_on_main() -> bool:
    """Clone testbed, remove tasks/ if present, push back. Returns True if a push happened."""
    workdir = Path(tempfile.mkdtemp(prefix="testbed-reset-"))
    try:
        clone = subprocess.run(
            ["git", "clone", "--depth", "1", TESTBED_URL, str(workdir / "repo")],
            capture_output=True, text=True, check=False, timeout=60,
        )
        if clone.returncode != 0:
            return False
        repo = workdir / "repo"
        tasks = repo / "tasks"
        if not tasks.exists():
            return False
        # Configure committer locally for this clone only
        subprocess.run(["git", "-C", str(repo), "config", "user.email",
                       "testbed-reset@test.invalid"], check=False, timeout=10)
        subprocess.run(["git", "-C", str(repo), "config", "user.name",
                       "Testbed Reset Fixture"], check=False, timeout=10)
        subprocess.run(["git", "-C", str(repo), "rm", "-rf", "tasks/"],
                      capture_output=True, check=False, timeout=10)
        commit = subprocess.run(
            ["git", "-C", str(repo), "commit", "-m", "test: reset tasks/ at session start"],
            capture_output=True, text=True, check=False, timeout=10,
        )
        if commit.returncode != 0:
            return False
        push = subprocess.run(
            ["git", "-C", str(repo), "push", "origin", "main"],
            capture_output=True, text=True, check=False, timeout=30,
        )
        return push.returncode == 0
    finally:
        shutil.rmtree(workdir, ignore_errors=True)


def reset_testbed_full() -> dict:
    """Full reset: close PRs, delete branches, wipe tasks/. Returns counts dict."""
    return {
        "prs_closed": close_all_open_prs(),
        "branches_deleted": delete_non_main_branches(),
        "tasks_wiped": wipe_tasks_dir_on_main(),
    }
```

`tests/e2e/lib/test_testbed_reset.py`: pytest module with unit tests for the helpers using subprocess mocking. The tests verify:
- `close_all_open_prs` parses the listing output and calls `gh pr close` for each PR number.
- `delete_non_main_branches` filters out 'main' and calls `gh api -X DELETE` for others.
- `wipe_tasks_dir_on_main` returns False when clone fails, returns False when tasks/ does not exist, returns True after successful push.
- `reset_testbed_full` returns a dict with three expected keys.

Use unittest.mock.patch for subprocess.run to avoid actual GitHub API calls. The tests are pure-Python and run as part of the unit tier (NOT under tests/e2e/), so they live at tests/e2e/lib/test_testbed_reset.py BUT pyproject.toml's norecursedirs excludes tests/e2e/.

Wait, that creates a conflict. The unit tests need to be collected, but tests/e2e/ is excluded.

Resolution: place the unit tests in `tests/test_testbed_reset.py` (under tests/ root), NOT under tests/e2e/. The module under test is at tests/e2e/lib/testbed_reset.py and is importable by Python module path `tests.e2e.lib.testbed_reset` since both `tests/` and `tests/e2e/` and `tests/e2e/lib/` have `__init__.py` from PR-153a.

`tests/test_testbed_reset.py`: unit tests at tests/ root level. Imports the helpers from tests.e2e.lib.testbed_reset, mocks subprocess.run, asserts call sequences and return values. Coverage target: 100% of testbed_reset.py module.

`tests/e2e/conftest.py`: add an autouse session-scoped fixture immediately after the existing fixtures. Also import the reset_testbed_full helper from the new module.

```python
from tests.e2e.lib.testbed_reset import reset_testbed_full


@pytest.fixture(scope="session", autouse=True)
def _reset_testbed_session():
    """Reset testbed to clean state at session start.
    
    Closes open PRs, deletes non-main branches, wipes tasks/ on main.
    Runs ONCE per pytest session before any test. The per-test reset_testbed
    fixture handles lighter cleanup between individual tests.
    """
    counts = reset_testbed_full()
    yield counts
```

The fixture yields the counts dict in case tests want to inspect what was cleaned (though they typically do not).

## Success criteria

1. `tests/e2e/lib/testbed_reset.py` exists with the four functions specified.

2. `tests/test_testbed_reset.py` exists at tests/ root and contains pytest tests that mock subprocess.run.

3. `python -m pytest tests/test_testbed_reset.py -v` passes with 100% coverage of tests/e2e/lib/testbed_reset.py module. Coverage check is enforced via the existing `--cov-fail-under=100` flag in scripts/ci.sh.

4. `python -m pytest -q` (default invocation, no arguments) collects the new test file under tests/ root and excludes everything under tests/e2e/. Total test count increases by however many test functions are in test_testbed_reset.py (typically 5 to 10).

5. The autouse fixture `_reset_testbed_session` is defined in tests/e2e/conftest.py with `scope="session"` and `autouse=True`.

6. With the GHA test stack running and a polluted testbed (manual setup: push 5 stub task files to testbed main, open one stub PR), running `pytest tests/e2e/test_smoke.py -v` causes the fixture to clean: after the test exits, testbed has no open PRs, no non-main branches, and no tasks/ directory. Verifiable manually before merge.

7. With a CLEAN testbed (post-cleanup state from 2026-04-25), running the fixture is a no-op: it does not error, but counts dict shows zero closed/deleted/wiped. The `wipe_tasks_dir_on_main` returns False because tasks/ does not exist, which is the documented "no work needed" return.

8. The fixture handles failures gracefully: if gh CLI is not authenticated, the helper functions return their counts as-is (zero) without raising. The fixture does NOT fail the entire pytest session; tests that subsequently fail because cleanup was incomplete will fail with their own clear errors.

9. scripts/ci.sh exits 0 with the new test file collected and passing. ruff check src tests passes.

10. The `wipe_tasks_dir_on_main` function uses `tempfile.mkdtemp` for the work directory and shutil.rmtree in a finally block; the work directory is removed regardless of success or failure.

11. Every subprocess.run call in the helpers has an explicit `timeout=` argument. No bare invocations.

12. After this PR merges, the next GHA integration run starts with a guaranteed-clean testbed regardless of what the previous run left behind. Verifiable by inspecting daemon-test logs in the e2e-evidence artifact: the daemon should NOT log "Picked task PR-XXX" for any PR not uploaded by the current test session.
