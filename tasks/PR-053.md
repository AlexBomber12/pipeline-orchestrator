# PR-053: Separate timeout from rate-limit in error handler

Branch: pr-053-timeout-classification

- Type: bugfix
- Complexity: low

## Problem

`handle_error` treats any error message containing "rate limit", "timeout", or "429" as a reason to skip AI diagnosis and stay in ERROR state. This lumps together very different situations:
- real Anthropic rate limits (429) - skipping diagnosis is correct
- generic network timeouts - transient, should retry
- slow `git fetch` hitting 60s timeout - transient, should retry
- slow `gh api` calls - transient, should retry
- Claude CLI subprocess timeout - task-specific, should escalate differently

A transient network timeout pushes the runner into ERROR without any recovery path. The runner stays stuck until manual intervention.

## Root cause

`src/daemon/runner.py:2217-2221`:
```python
lowered = context.lower()
if "rate limit" in lowered or "timeout" in lowered or re.search(r"\b429\b", lowered):
    self.log_event("Skipping AI diagnosis for rate-limit/timeout error")
    return
```

The substring match treats all three patterns identically.

## Scope

Split error classification into distinct categories. Rate limit stays as-is (skip diagnosis, wait for _try_recover_rate_limit). Generic timeout becomes a separate path that the caller can treat as transient.

This PR only changes classification and logging. Retry logic for transient timeouts is PR-054.

## Changes

### src/daemon/runner.py

Define an error category enum near the other enums:

```python
class ErrorCategory(Enum):
    RATE_LIMIT = "rate_limit"       # 429 or explicit "rate limit" phrase
    TIMEOUT = "timeout"             # generic transient timeout
    OTHER = "other"                 # everything else, route to AI diagnosis
```

Add a classifier function:

```python
def _classify_error(context: str) -> ErrorCategory:
    lowered = context.lower()
    if "rate limit" in lowered or re.search(r"\b429\b", lowered):
        return ErrorCategory.RATE_LIMIT
    if "timeout" in lowered:
        return ErrorCategory.TIMEOUT
    return ErrorCategory.OTHER
```

Update `handle_error`:

```python
category = _classify_error(context)
if category == ErrorCategory.RATE_LIMIT:
    self.log_event("Skipping AI diagnosis for rate-limit error")
    return
if category == ErrorCategory.TIMEOUT:
    self.log_event(
        "Skipping AI diagnosis for timeout error; "
        "will retry on next cycle"
    )
    # Future: PR-054 adds bounded retry with backoff here
    return
# OTHER: fall through to existing AI diagnosis flow
self._error_diagnose_count += 1
...
```
