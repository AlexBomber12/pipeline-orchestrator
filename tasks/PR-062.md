# PR-062: PAUSED state for rate limit

Branch: pr-062-paused-state

- Type: feature
- Complexity: medium

## Problem

Rate limit is currently signaled through ERROR state with `rate_limited_until` set. This is semantically wrong:
- ERROR implies something broke and needs operator attention
- Rate limit is expected, temporary, and auto-recovers
- Dashboard shows red ERROR badge for a situation that is normal operation
- Event log mixes rate-limit pauses with real errors, making debugging harder

Current flow:
1. CLI returns non-zero exit with "rate limit" in stderr
2. `_detect_rate_limit` sets `rate_limited_until`
3. Caller sets state = ERROR (because CLI exited non-zero)
4. Next cycle `_try_recover_rate_limit` checks if window expired, recovers to WATCH or IDLE

The semantic mismatch also complicates HUNG detection, error classification (PR-053), and future multi-coder fallback logic (sprint 8): distinguishing "rate limit, just wait" from "something broke" should not require string matching on error_message.

## Scope

Add a new PipelineState called PAUSED. Rate-limit detection sets state to PAUSED instead of ERROR. Dashboard shows PAUSED distinctly from ERROR with reset countdown. Recovery logic flows through the same state machine cleanly.

Add `handle_paused` method:

```python
async def handle_paused(self) -> None:
    """Wait for rate limit window to expire, then resume previous flow."""
    if self.state.rate_limited_until is None:
        self.log_event("PAUSED without rate_limited_until -> IDLE")
        self.state.state = PipelineState.IDLE
        return
    if datetime.now(timezone.utc) < self.state.rate_limited_until:
        remaining = (self.state.rate_limited_until - datetime.now(timezone.utc)).total_seconds()
        self.log_event(f"Paused, resuming in {int(remaining)}s")
        return
    # Window expired: resume to appropriate state
    self.state.rate_limited_until = None
    if (
        self.state.current_pr is not None
        and self.state.current_task is not None
        and self.state.current_pr.branch == self.state.current_task.branch
    ):
        self.state.state = PipelineState.WATCH
        self.log_event("Rate limit expired, resuming -> WATCH")
    else:
        self.state.state = PipelineState.IDLE
        self.log_event("Rate limit expired, resuming -> IDLE")
```

Remove `_try_recover_rate_limit` from the ERROR-handling path in run_cycle. Rate limit recovery now happens through the PAUSED handler.

Update `_check_rate_limit` to optionally transition to PAUSED when called while not already paused (covers edge cases where rate_limited_until is set from a previous cycle but state was somehow reset):

```python
def _check_rate_limit(self) -> bool:
    if self.state.rate_limited_until is not None:
        if datetime.now(timezone.utc) < self.state.rate_limited_until:
            if self.state.state != PipelineState.PAUSED:
                self.state.state = PipelineState.PAUSED
            return False
        self.state.rate_limited_until = None
    return True
```

### _TRANSIENT_STATES

Decide: should PAUSED be in _TRANSIENT_STATES (reset to IDLE on next cycle)? No. Unlike CODING/FIX/MERGE which reset on crash recovery, PAUSED is the correct state if rate_limited_until is still in the future. run_cycle's TRANSIENT_STATES reset should exclude PAUSED explicitly.
