# PR-138: Coder selector on repo detail page with hot-reload

Branch: pr-138-repo-coder-selector
- Type: feature
- Complexity: medium
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Changing the coder for a repo (Claude / Codex / any) currently requires:
1. Navigate to Settings
2. Find the repos list
3. Change the dropdown
4. Submit
5. Restart the daemon container (because `repo_config.coder` is not hot-reloaded by the runner)

This is too many steps for a routine operation, and the restart requirement kills any in-progress work.

This PR adds a coder dropdown directly on the repo detail page header, next to the existing `Coder: X CLI` badge. Changing it hot-reloads the config without a daemon restart.

## Scope

### UI

On the repo detail page, replace the read-only `Coder: Codex CLI` badge with an inline dropdown showing:
- Any (bandit picks per-PR)
- Claude CLI
- Codex CLI

Current selection reflects `repo_config.coder`. On change, `hx-post="/repos/{name}/coder"` with the new value.

### Backend

Add `POST /repos/{name}/coder` endpoint in `src/web/app.py`:
1. Validate the new value (`any` / `claude` / `codex`).
2. Update `repo_config.coder` in Redis state.
3. Publish an SSE event `config_reloaded` so other open dashboards see the change.
4. Signal the runner to reload `repo_config` for this repo without restart.

### Hot-reload

The runner currently caches `self.repo_config` at startup. Add a mechanism so config changes via API are picked up:
- Option A: runner checks Redis for config version hash each loop tick and re-reads if changed.
- Option B: web endpoint writes a "config_dirty" flag, runner checks and reloads at next safe boundary (between PR pickups, not mid-CODING).

Option B is safer - no mid-flight config swap. Implement B.

The reload applies only to `coder` field for now. Other fields (timeouts, rate limits) can follow in later PRs.

### Edge case

If the daemon is currently in CODING/WATCH/FIX/MERGE and the user changes coder, the in-flight PR continues with the old coder (do not interrupt). The new coder applies only to the next task pickup.

Show a brief toast or inline message confirming the change and noting when it takes effect ("Switching to Claude - applies after current PR completes").

## Files to modify

- `src/web/app.py`: add POST endpoint and Redis flag writer.
- `src/daemon/runner.py` or `src/config.py`: add `reload_repo_config_if_dirty()` method called at idle boundary.
- Tests: endpoint validation test, hot-reload test, mid-flight-change-does-not-interrupt test.

## Success criteria

- Detail page shows inline coder dropdown with three options.
- Changing the dropdown posts to the new endpoint, returns 200, shows confirmation message.
- Next task picked up uses the new coder without daemon restart.
- Current in-flight task finishes with the old coder (not interrupted).
- Other open dashboard instances see the change via SSE within a few seconds.
- `scripts/ci.sh` exits 0.
- `ruff check src tests` exits 0.
- Coverage stays at 100%.
