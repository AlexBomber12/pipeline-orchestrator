# PR-004: GitHub client

Branch: pr-004-github-client

## Scope

Create a wrapper around `gh` CLI that reads PR status, CI checks, Codex review reactions, and can merge PRs and post comments. This module is used by both the daemon (future PR) and the dashboard.

## Files to create

### src/github_client.py

Helper function:
run_gh(args: list[str], repo: str | None = None, timeout: int = 30) -> dict | list | str
- Run `gh` with given args via subprocess
- If repo provided, append `-R repo`
- Parse stdout as JSON, fallback to raw string
- Raise RuntimeError on non-zero exit

Public functions:

get_repo_full_name(url: str) -> str
- Extract "owner/repo" from URL like https://github.com/AlexBomber12/lan-transcriber.git -> AlexBomber12/lan-transcriber

get_open_prs(repo: str) -> list[PRInfo]
- repo is "owner/repo" format
- Use `gh pr list --state open --json number,headRefName,statusCheckRollup,url,updatedAt,commits`
- For each PR, parse CI status from statusCheckRollup and get review status via get_pr_review_status
- Return list of PRInfo

get_pr_review_status(repo: str, pr_number: int) -> ReviewStatus
- Use `gh api repos/{repo}/issues/{pr_number}/comments` to get PR comments
- Walk comments in reverse order
- Look for comments from users whose login contains "codex" (case-insensitive)
- Check reactions on those comments: +1 reaction = APPROVED, eyes reaction = EYES
- If codex comment body contains P1 or P2 = CHANGES_REQUESTED
- Default: PENDING

merge_pr(repo: str, pr_number: int) -> None
- Use `gh pr merge {number} --merge --delete-branch -R {repo}`

post_comment(repo: str, pr_number: int, body: str) -> None
- Use `gh pr comment {number} --body {body} -R {repo}`
