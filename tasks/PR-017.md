# PR-017: Dashboard observability - event log and stats

Branch: pr-017-observability

## Scope

Add a cross-repo event log and basic stats to the dashboard index page. Shows recent activity across all repos and aggregated metrics.

## Files to modify

### src/web/app.py

Add routes:

GET /partials/activity-feed -> activity feed component
- Collect history entries from all repo states in Redis
- Merge, sort by time descending, take last 50
- Return rendered component

GET /api/stats -> JSON
- For each repo: count DONE tasks (from QUEUE.md in Redis state history or parsed from config), count events today, current state
- Aggregate: total tasks done today, total tasks done this week (from history timestamps), average iterations per merge (from push_count in history)
- Return stats JSON

### src/web/templates/index.html

Restructure layout:
- Top row: summary stats cards (repos connected, tasks done today, active pipelines, errors/hung)
- Below: two columns (stack on mobile)
  - Left: repo cards (existing repo_list component)
  - Right: activity feed (cross-repo event log, newest first)

### src/web/templates/components/stats_cards.html
- 4 compact cards in a row:
  - "Repos" - count of connected repos
  - "Done today" - tasks merged today (count from history events containing "Merged")
  - "Active" - repos in CODING/WATCH/FIX state
  - "Alerts" - repos in HUNG/ERROR state (red if > 0)
- Each card: large number, small label below
- HTMX refresh: hx-get="/partials/stats" hx-trigger="every 10s"

### src/web/templates/components/activity_feed.html
- Scrollable list (max-h-96)
- Each entry: timestamp, repo name badge (colored by repo), state badge, event text
- Repo name links to /repo/{name}
- HTMX refresh: hx-get="/partials/activity-feed" hx-trigger="every 10s"

GET /partials/stats -> stats cards component
