# PR-174: Event log dedup display (Lucide rotate-ccw + count badge)

Branch: pr-174-event-log-dedup-display
- Type: ux
- Complexity: low
- Depends on: PR-173
- Priority: 2
- Coder: any

## Problem

PR-173 introduces backend dedup with `repeat_count` field. Dashboard now needs to render this count meaningfully so users see "this event repeated N times" rather than just one stale-looking row.

## Scope

When an event log entry has `repeat_count > 1`, render a small badge with the count next to the message, plus a Lucide `rotate-ccw` icon hinting "this is a repeat" semantics. Use existing icon system if Lucide is available; otherwise inline SVG.

## Files to touch

`src/web/templates/components/event_log_row.html` (or wherever a single event log row is rendered): conditionally render the dedup badge.

Sketch:

```html
<div class="event-log-row flex items-center gap-2">
    <span class="event-state-badge">{{ event.state }}</span>
    <span class="event-time text-xs text-gray-500">{{ event.timestamp_human }}</span>
    <span class="event-message flex-1">{{ event.message }}</span>
    {% if event.repeat_count and event.repeat_count > 1 %}
        <span class="event-dedup-badge inline-flex items-center gap-1 px-2 py-0.5 rounded text-xs bg-gray-700 text-gray-300" title="repeated {{ event.repeat_count }} times">
            {# Lucide rotate-ccw icon, inline SVG since CDN is uncertain #}
            <svg class="h-3 w-3" xmlns="http://www.w3.org/2000/svg" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2" stroke-linecap="round" stroke-linejoin="round">
                <path d="M3 12a9 9 0 1 0 9-9 9.75 9.75 0 0 0-6.74 2.74L3 8"/>
                <path d="M3 3v5h5"/>
            </svg>
            <span>×{{ event.repeat_count }}</span>
        </span>
    {% endif %}
</div>
```

If a different template structure is used in the codebase, adapt the sketch but keep the conditional logic and icon+count badge.

`src/web/static/css/` (if separate) or inline in the row template: `.event-dedup-badge` styling. Should be subtle (gray, small) so it's informative but doesn't dominate.

`tests/test_dashboard.py` (or template tests): assert the dedup badge:
- Is NOT rendered when `repeat_count` is None or 1
- IS rendered when `repeat_count >= 2`
- Includes the count value in the badge text and title attribute
- Tooltip says "repeated N times"

`src/daemon/handlers/`: untouched.
