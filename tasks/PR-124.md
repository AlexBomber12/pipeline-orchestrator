# PR-124: Lock coverage gate at 100%

Branch: pr-124-coverage-gate-100
- Type: config
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

After sprint 9 (PR-094 through PR-122) every module in `src/` reached 100% line coverage. The CI gate in `scripts/ci.sh` is still set to `--cov-fail-under=86`, which was the baseline at the start of sprint 9. With the current floor at 86, any regression that drops coverage below 100 but stays above 86 would silently land on main.

The floor must be raised to 100 to prevent unnoticed regressions from future PRs.

## Scope

Change exactly one number in `scripts/ci.sh`: `--cov-fail-under=86` becomes `--cov-fail-under=100`. Nothing else.

Do not change pytest arguments beyond that number. Do not touch `requirements.txt`. Do not add or remove any other flags.

## Files to modify

- `scripts/ci.sh`: change `--cov-fail-under=86` to `--cov-fail-under=100` on the existing pytest line.

## Success criteria

- `grep --cov-fail-under=100 scripts/ci.sh` returns a match.
- `grep --cov-fail-under=86 scripts/ci.sh` returns no match.
- `scripts/ci.sh` exits 0 on current main (coverage is at 100% after sprint 9).
- A future PR that drops any `src/` module below 100% line coverage will cause `scripts/ci.sh` to exit non-zero.
- `ruff check src tests` exits 0.
- No other files modified in this PR.
