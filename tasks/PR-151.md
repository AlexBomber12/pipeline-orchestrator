# PR-151: Play/Pause/Stop buttons size responsive and subtle

Branch: pr-151-button-sizing
- Type: ux
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

The Play/Pause/Stop control buttons in `src/web/templates/components/_controls.html` use a single fixed size `h-11 w-11` (44px square) regardless of viewport. On desktop this makes the buttons visually dominant and disproportionate to the rest of the UI. On mobile the 44px hit area is appropriate (Apple HIG minimum), but the chunky border styling looks like a primary CTA when it should be a secondary control.

Target styling:

- Desktop (md breakpoint and up): 24-28px visible size, flat (no border), subtle hover
- Mobile (default): 32-36px visible size, with padding-based hit area expansion to 44px for touch targets
- All sizes: gray icon color, hover reveals white icon on subtle background tint

## Scope

Rewrite the three button tags in the `repo_controls` macro (lines 27-61 of `_controls.html`). Replace the current class string:

`secondary-interactive inline-flex h-11 w-11 items-center justify-center rounded-lg border border-white/10 bg-surface-3/60 text-gray-300`

With a responsive, borderless variant:

`inline-flex items-center justify-center h-8 w-8 md:h-7 md:w-7 p-1.5 -m-1.5 md:p-0 md:m-0 rounded-md text-gray-400 hover:text-white hover:bg-white/5 transition-colors`

Breakdown:
- `h-8 w-8` base (mobile): 32px visible element
- `p-1.5 -m-1.5` base (mobile): 6px invisible padding around, so hit area is 32+12=44px with no layout shift
- `md:h-7 md:w-7 md:p-0 md:m-0`: 28px flat on desktop
- `rounded-md` instead of `rounded-lg` - tighter corner for smaller size
- `text-gray-400 hover:text-white hover:bg-white/5` - subtle default, reveal on hover
- No `border` class - flat
- `transition-colors` for smooth hover

Apply to all three buttons: Play, Pause, Stop. For Stop, keep the danger semantics via `hover:text-fail hover:bg-fail/10` instead of `hover:text-white hover:bg-white/5`. Default color stays `text-gray-400` so the red only appears on hover (less alarming in default view).

SVG icons inside: the `w-5 h-5` class on the SVG can stay (icon content size independent of button container). But review whether 20px icon in 28px desktop button looks proportional - if too big, reduce to `w-4 h-4` (16px) for desktop with `md:w-4 md:h-4` and keep `w-5 h-5` base.
