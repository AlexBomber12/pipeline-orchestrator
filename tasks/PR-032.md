# PR-032: Claude CLI stdout logging

Branch: pr-032-cli-stdout-log

## Scope

Claude CLI stdout is currently discarded (_stdout) in handle_coding and handle_fix. Without it, diagnosing failures requires guessing what Claude did. Save stdout to Redis for post-mortem and dashboard display.

## Changes

### src/daemon/runner.py

In handle_coding and handle_fix, after claude_cli call:
1. Store stdout in Redis with key "cli_log:{repo_name}:latest" and TTL 1 hour
2. Also store in "cli_log:{repo_name}:{timestamp}" with TTL 24 hours for history
3. Log first 200 chars of stdout at INFO level

Add helper method:
```python
async def _save_cli_log(self, stdout: str) -> None:
    key_latest = f"cli_log:{self.name}:latest"
    key_ts = f"cli_log:{self.name}:{datetime.now(timezone.utc).isoformat()}"
    await self.redis.set(key_latest, stdout, ex=3600)
    await self.redis.set(key_ts, stdout, ex=86400)
```

### src/web/app.py

Add route GET /partials/repo/{name}/cli-log that reads "cli_log:{name}:latest" from Redis and returns it in a pre-formatted HTML block.

### src/web/templates/repo.html

Add a collapsible "Claude CLI log" section below the event log that loads via hx-get="/partials/repo/{name}/cli-log".

### tests/test_runner.py

- test_handle_coding_saves_cli_stdout: verify Redis set called with stdout content
- test_handle_fix_saves_cli_stdout: same for fix path

## Success criteria
- scripts/ci.sh exits 0
- Claude CLI stdout visible on repo detail page
- Logs auto-expire after 24h
