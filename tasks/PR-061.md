# PR-061: Split rate-limit thresholds into session and weekly

Branch: pr-061-rate-limit-thresholds

- Type: feature
- Complexity: low

## Problem

Claude Max has two separate rate limits: a 5-hour session limit and a weekly limit. Current code uses a single `rate_limit_pause_percent` threshold (default 90%) that triggers on either limit. Two problems:

1. Weekly limit pausing at 90-95% means the daemon stops working for days until weekly reset. User wants to run weekly to depletion and let the Anthropic API itself return 429 when exhausted.

2. Session limit pausing at 90-95% is useful - the limit resets in 1-5 hours so a short pause is cheap, and running past 95% risks a mid-task 429.

The stderr regex already captures the percentage, but doesn't distinguish which limit reported it. Both hit the same threshold.

## Scope

Split the single threshold into two:
- `rate_limit_session_pause_percent` (default 95) - triggers on 5-hour session limit
- `rate_limit_weekly_pause_percent` (default 100) - triggers on weekly limit (100 = never pause proactively, only pause on actual 429)

Detect which limit the stderr message refers to. Apply the appropriate threshold.

## Changes

### src/config.py

Replace single field:
```python
rate_limit_pause_percent: int = 90
```

With:
```python
rate_limit_session_pause_percent: int = 95
rate_limit_weekly_pause_percent: int = 100
```

Keep `rate_limit_pause_percent` as a deprecated alias for backwards compatibility: if present in config.yml, read it, log a deprecation warning, use the value as session threshold.

Update ALLOWED_DAEMON_KEYS to include both new field names.

### src/daemon/runner.py

Rewrite `_detect_rate_limit` to identify which limit hit:

- If stderr contains "429", pause 30 min regardless (both limits share this signal)
- If stderr matches weekly pattern (contains "weekly" or "week" near "limit"), compare against weekly threshold
- Else if stderr matches session pattern (contains "session", "5-hour", or no qualifier), compare against session threshold
- Generic "rate limit" mention without percentage falls back to session threshold

Factor the pause trigger into a helper `_trigger_rate_limit_pause(limit_type, pause_min)` that logs the limit type clearly.

### src/web/templates/components/settings_daemon.html

Replace single rate limit field with two, with one-line hint under weekly: "Set to 100 to run until API returns 429."

### Tests

- test_detect_rate_limit_weekly_respects_weekly_threshold: "95% of weekly limit" with weekly=100 -> no pause
- test_detect_rate_limit_weekly_triggers_at_threshold: "95% of weekly limit" with weekly=90 -> pause
- test_detect_rate_limit_session_respects_session_threshold: "95% of session limit" with session=95 -> pause
- test_detect_rate_limit_429_always_pauses: "HTTP 429" -> pause regardless of thresholds
- test_detect_rate_limit_deprecated_field_still_works: old config uses rate_limit_pause_percent, expect it mapped to session threshold with deprecation log
- test_log_event_identifies_limit_type: verify "Rate limit detected (weekly)" vs "Rate limit detected (session)"
