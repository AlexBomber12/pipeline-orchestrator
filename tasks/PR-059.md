# PR-059: AGENTS.md update - artifacts and stale review

Branch: pr-059-agents-update

- Type: config
- Complexity: low

## Problem

Current AGENTS.md contains two contradictions that cause Claude Code to behave inconsistently:

1. Quick rules require generating `artifacts/pr.patch`, but Daemon Mode forbids committing files in artifacts/. Claude resolves the contradiction unpredictably (sometimes commits, sometimes refuses).

2. Codex Review gate only mentions +1 on the anchor comment, but Codex sometimes reacts on the PR body instead. When Claude follows AGENTS.md strictly, it misses approvals on the PR body.

Additionally, the repo path in the Repo invariants section still references the pre-PR-038 naming. Canonical repo identity changed to `/data/repos/<owner>__<repo>/`.

## Scope

Apply the AGENTS.md fixes prepared in the 2026-04-15 session. Single file change, no code.

Also update `templates/AGENTS.md` so newly scaffolded repos start with the correct AGENTS.md.

## Changes

### AGENTS.md

Replace content with the version from /mnt/user-data/outputs/AGENTS.md (prepared earlier).

Key changes in that version:

1. Quick rules clarifies artifacts handling:
   - "Always generate review artifacts... These are for review only and must not appear in commits (.gitignore handles this)."

2. Daemon Mode section gets two explicit rules:
   - "NEVER use `git add -f` or explicitly stage files matched by .gitignore."
   - "Artifacts (ci.log, pr.patch, structure.txt) are generated for Codex review but must not appear in commits."
   - "NEVER commit .patch files to the repository under any circumstances."

3. Codex Review gate pass signal covers both locations:
   - "thumbs up (`+1`) reaction from the Codex bot on either the PR body (description) OR the review anchor comment"
   - "The daemon compares the reaction timestamp against the latest push timestamp. A +1 that predates the most recent push is stale and does not count as a pass."

4. Fix loop step 5 added:
   - "If no new feedback exists after the latest push, stop. Do not fix already-resolved issues."

5. Fix loop step 3 updated:
   - "If a non-stale thumbs up exists (reaction created after the latest push), stop."

6. Repo invariants path updated:
   - "Cloned repos: `/data/repos/<owner>__<repo>/`" (was `<repo-name>`)

7. Checklist items note that artifacts are not committed:
   - "[ ] Generated `artifacts/ci.log`, `artifacts/pr.patch`, `artifacts/structure.txt` (not committed, excluded by .gitignore)"
