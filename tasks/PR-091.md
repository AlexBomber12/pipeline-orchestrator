# PR-091: Coder fallback policy with eligibility and priority

Branch: pr-091-fallback-policy
- Type: feature
- Complexity: medium
- Depends on: PR-077
- Priority: 2
- Coder: any

## Problem

Registry (PR-073) and plugin protocol (PR-077) are designed for N coders; Gemini/Qwen/future plugins are planned within 1-3 months. A hard-coded if-chain "on Claude rate limit try Codex" works for 2 coders but does not extend: each new coder would require rewriting the fallback cascade. We need a selector that takes the current system state and returns a ranked list of eligible coders, regardless of how many are registered.

## Scope

Introduce `src/daemon/selector.py` with two pure functions:
- `eligible_coders(ctx) -> list[str]` — filter out coders that cannot currently run (rate-limited, auth failed, pinned out by repo config, explicitly disabled for repo)
- `rank_coders(eligible, ctx) -> list[str]` — rank by priority (per-coder static priority from config, extensible later to metric-based score in Phase 4)

Compose them in `select_coder(ctx) -> (name, plugin) | None` and call it from `_get_coder` in runner. Keep `auto_fallback` config flag behaviour but implement through selector, not hard-coded branch. Static priority comes from `config.yml`; metric-based ranking is a future extension (Phase 4), not scoped here.

## Changes

### src/daemon/selector.py (new)

```
from __future__ import annotations
from dataclasses import dataclass
from src.coder_registry import CoderRegistry, CoderPlugin
from src.config import AppConfig, RepoConfig
from src.models import RepoState


@dataclass
class SelectionContext:
    registry: CoderRegistry
    repo_config: RepoConfig
    app_config: AppConfig
    state: RepoState


def eligible_coders(ctx: SelectionContext) -> list[str]:
    pinned = ctx.repo_config.coder
    if pinned is not None and not ctx.app_config.daemon.auto_fallback:
        return [pinned.value]
    result: list[str] = []
    for name in ctx.registry.coder_names():
        if _is_rate_limited(name, ctx.state):
            continue
        if _auth_failed(name, ctx.registry):
            continue
        if _is_disabled_for_repo(name, ctx.repo_config):
            continue
        result.append(name)
    if pinned is not None and pinned.value in result:
        return [pinned.value] + [n for n in result if n != pinned.value]
    return result


def rank_coders(eligible: list[str], ctx: SelectionContext) -> list[str]:
    def priority(name: str) -> int:
        return ctx.app_config.daemon.coder_priority.get(name, 0)
    return sorted(eligible, key=priority, reverse=True)


def select_coder(ctx: SelectionContext) -> tuple[str, CoderPlugin] | None:
    eligible = eligible_coders(ctx)
    if not eligible:
        return None
    name = rank_coders(eligible, ctx)[0]
    return name, ctx.registry.get(name)


def _is_rate_limited(name: str, state: RepoState) -> bool:
    if state.rate_limit_reactive_coder == name:
        return True
    limited = getattr(state, "rate_limited_coders", None) or set()
    return name in limited


def _auth_failed(name: str, registry: CoderRegistry) -> bool:
    try:
        status = registry.get(name).check_auth()
    except Exception:
        return True
    return status.get("status") == "failed"


def _is_disabled_for_repo(name: str, repo_config: RepoConfig) -> bool:
    return name in (repo_config.disabled_coders or [])
```

### src/daemon/runner.py

Replace `_get_coder` body to use selector:
```
def _get_coder(self) -> tuple[str, CoderPlugin]:
    ctx = SelectionContext(
        registry=self._registry,
        repo_config=self.repo_config,
        app_config=self.app_config,
        state=self.state,
    )
    result = select_coder(ctx)
    if result is not None:
        return result
    coder = self.repo_config.coder or self.app_config.daemon.coder
    name = coder.value if hasattr(coder, "value") else str(coder)
    return name, self._registry.get(name)
```

Import `select_coder` and `SelectionContext` at top.

### src/daemon/rate_limit.py

Remove any draft chain logic that existed in the prior PR-091 sketch (e.g. the `_fallback_coder` attribute).

## Success criteria

- scripts/ci.sh exits 0
- Selector is a pure function module: no imports from runner, handlers, or rate_limit
- `_get_coder` contains no coder-specific branching (no `if coder_name == "codex"`)
- Adding a new coder requires zero edits to selector.py (only a priority entry in config and plugin registration)
- Existing rate-limit tests pass unchanged
- With auto_fallback on and active coder rate-limited, next eligible coder is selected on next `_get_coder` call
- Repo-pinned coder with auto_fallback off never switches
- When no coder is eligible, selector returns None and runner falls through to configured default (pre-existing behaviour preserved)
