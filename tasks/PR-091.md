# PR-091: Auto-fallback chain on rate limit

Branch: pr-091-auto-fallback
- Type: feature
- Complexity: medium
- Depends on: PR-077
- Priority: 2
- Coder: any

## Problem

When Claude hits rate limit, operator must manually switch to Codex in Settings UI. Daemon pauses until operator acts or limit resets. With two coders available, daemon should auto-switch.

## Scope

When coder hits rate limit and auto_fallback enabled: check if other registered coder is available and not rate-limited. If yes, switch for current task. Log the fallback. Respect per-repo coder override (if explicitly set to "claude", no fallback to codex).

## Changes

### src/config.py

Add to DaemonConfig:
```python
auto_fallback: bool = True
```

Add to _DAEMON_FIELDS.

### src/daemon/rate_limit.py

In `_check_rate_limit`, after determining rate limit is active for current coder:

```python
if self.app_config.daemon.auto_fallback:
    repo_coder = self.repo_config.coder
    if repo_coder is None:  # not explicitly pinned
        other_coders = [c for c in self._registry.coder_names() if c != coder_name]
        for other in other_coders:
            other_provider = self._registry.get(other).create_usage_provider(...)
            # check other coder not also limited
            if self.state.rate_limit_reactive_coder != other:
                self.log_event(
                    f"[{coder_name}] Rate limited, falling back to [{other}]"
                )
                self._fallback_coder = other
                return True  # allow CLI call with fallback coder
```

In `_get_coder`: check `_fallback_coder` before registry lookup. Clear fallback after successful run.

### src/web/templates/components/settings_daemon.html

Add checkbox: "Auto-fallback: switch coder on rate limit". Hint: "When active coder hits rate limit, try other available coders before pausing."
