# PR-141: Rate limit display only for the active coder

Branch: pr-141-rate-limit-active-only
- Type: refactor
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

The repo detail page header currently shows `Session X% | Weekly Y%` rate limit stats. These values apply to ONE specific coder (historically Claude, since Claude's usage API is what the daemon polls). With multi-coder support now real, the display is ambiguous: the user cannot tell which coder these numbers refer to, and if Codex is the active coder the Claude numbers are irrelevant.

Two possible solutions:
1. Show rate limits for BOTH coders (Session/Weekly per coder).
2. Show rate limits only for the currently ACTIVE coder, with a clear label.

The decision: option 2. Keep the display compact. If the user wants cross-coder rate limit detail, they can find it in Settings (the auth status section already shows per-coder status).

## Scope

Change the header rate-limit badge from `Session X% | Weekly Y%` (ambiguous) to `{CoderName} Session X% | Weekly Y%` where CoderName comes from the current active coder selection.

If the active coder has no rate-limit data (e.g. Codex, which the daemon does not poll for usage), hide the whole badge rather than showing stale Claude numbers.

Edge case: if `repo_config.coder == "any"` and the bandit just picked a coder for the current PR, show the rate limit of whichever coder is currently running (from RepoState.current_coder or similar). If no active task, hide.

## Files to modify

- `src/web/templates/repo.html`: conditional rendering of the rate-limit badge based on active coder.
- `src/models.py` or state derivation: ensure "current coder for rate-limit display" is exposed to the template. If no such field, derive from last run or active task.
- `src/web/app.py`: if the repo context passed to the template needs a new field, add it.
- Tests: render test covering (a) Claude active -> Claude Session/Weekly shown, (b) Codex active -> no rate limit shown (or zero-state variant), (c) no active task -> hidden.

## Success criteria

- Displayed number is clearly labeled with the coder name (`Claude Session 16%` or similar).
- No stale Claude numbers shown when Codex is active.
