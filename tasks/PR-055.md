# PR-055: Strict queue validation

Branch: pr-055-queue-validation

- Type: feature
- Complexity: medium

## Problem

The queue parser silently normalizes ambiguous inputs. Consequences:
- Unknown Status value (e.g. "INPROGRESS", "done") silently degrades to TODO
- Duplicate pr_id accepted - two entries for PR-042 both get parsed
- Duplicate branch accepted - branch-based recovery maps to the wrong task
- Missing dependency accepted - "Depends on: PR-999" where PR-999 doesn't exist
- No cycle detection - PR-A depends on PR-B depends on PR-A is accepted

The queue may look fine but behave unpredictably: tasks silently re-enter execution, impossible dependency graphs block progress, recovery maps the wrong PR to the wrong task.

## Scope

Add strict validation on both upload (via dashboard) and startup/reload (daemon parse). Reject ambiguous queues with specific error messages (line number, conflicting ids, cycle path). Allow normal queues to proceed unchanged.

## Changes

### src/queue_parser.py

Add validation errors as a dedicated exception type:

```python
class QueueValidationError(ValueError):
    """Raised when queue contents cannot be unambiguously parsed."""

    def __init__(self, issues: list[str]):
        self.issues = issues
        super().__init__("Queue validation failed:\n" + "\n".join(f"  - {i}" for i in issues))
```

Add a validation function that runs after parsing:

```python
def validate_queue(tasks: list[QueueTask]) -> None:
    """Raise QueueValidationError if the parsed queue is ambiguous.

    Checks performed:
    - No duplicate pr_id
    - No duplicate branch (where branch is set)
    - All dependencies reference known pr_id
    - No dependency cycles
    - No unknown Status values (caller is responsible for rejecting these
      before calling validate_queue; see parse_queue strict mode)
    """
    issues: list[str] = []

    seen_ids: dict[str, int] = {}
    for idx, task in enumerate(tasks, start=1):
        if task.pr_id in seen_ids:
            issues.append(
                f"duplicate pr_id {task.pr_id!r} at task #{idx} "
                f"(first seen at task #{seen_ids[task.pr_id]})"
            )
        else:
            seen_ids[task.pr_id] = idx

    seen_branches: dict[str, str] = {}
    for task in tasks:
        if not task.branch:
            continue
        if task.branch in seen_branches:
            issues.append(
                f"duplicate branch {task.branch!r}: used by "
                f"{seen_branches[task.branch]} and {task.pr_id}"
            )
        else:
            seen_branches[task.branch] = task.pr_id

    known_ids = set(seen_ids.keys())
    for task in tasks:
        for dep in task.depends_on:
            if dep not in known_ids:
                issues.append(
                    f"{task.pr_id} depends on unknown task {dep!r}"
                )

    cycle = _find_dependency_cycle(tasks)
    if cycle:
        issues.append("dependency cycle: " + " -> ".join(cycle))

    if issues:
        raise QueueValidationError(issues)


def _find_dependency_cycle(tasks: list[QueueTask]) -> list[str] | None:
    """Return cycle path if a dependency cycle exists, else None."""
    graph = {t.pr_id: list(t.depends_on) for t in tasks}
    visiting: set[str] = set()
    visited: set[str] = set()
    path: list[str] = []

    def dfs(node: str) -> list[str] | None:
        if node in visited:
            return None
        if node in visiting:
            idx = path.index(node) if node in path else 0
            return path[idx:] + [node]
        visiting.add(node)
        path.append(node)
        for neighbor in graph.get(node, []):
            result = dfs(neighbor)
            if result:
                return result
        path.pop()
        visiting.remove(node)
        visited.add(node)
        return None

    for pr_id in graph:
        result = dfs(pr_id)
        if result:
            return result
    return None
```

Existing callers pass strict=False (current behavior). Upload handler and daemon startup pass strict=True.

### src/web/app.py

In the upload handler, call `parse_queue(path, strict=True)`. On QueueValidationError, return HTTP 400 with the issues list formatted for display.

### src/daemon/main.py or runner.py

At daemon startup (before first cycle), parse QUEUE.md from main with strict=True. If validation fails, the daemon logs the issues and enters a dedicated QUEUE_INVALID error state (or equivalent) rather than silently proceeding with a mis-parsed queue.

Optionally, log a warning and continue in non-strict mode if operator prefers to debug via dashboard. Add config flag:

```yaml
daemon:
  strict_queue_validation: true  # default
```
