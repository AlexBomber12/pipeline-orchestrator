# PR-030: Detect and push unpushed commits

Branch: pr-030-unpushed-detect

## Scope

_commit_and_push_dirty only catches uncommitted changes (dirty tree). If Claude commits but does not push, git status --porcelain is clean, the method returns False, and daemon posts @codex review on stale remote code. Codex reviews old code, finds same issues, fix loop repeats.

Add unpushed commit detection alongside dirty tree detection.

## Changes

### src/daemon/runner.py

In _commit_and_push_dirty, after the git status check (whether clean or dirty):

1. Run git rev-list HEAD..origin/{expected_branch} --count (or origin/{expected_branch}..HEAD --count for unpushed)
2. If count > 0 and tree is clean: skip CI (code was already committed, presumably CI-tested by Claude), just git push origin {expected_branch}:{expected_branch}
3. If count > 0 and tree is dirty: run CI, git add, git commit, git push (existing flow)
4. If count == 0 and tree is clean: return False (nothing to do)

Handle the case where origin/{expected_branch} does not exist yet (first push) - git rev-list will fail, treat as "has unpushed".

### tests/test_runner.py

- test_commit_and_push_dirty_pushes_unpushed_commits: tree clean, 1 unpushed commit, verify push happens
- test_commit_and_push_dirty_noop_when_clean_and_synced: tree clean, 0 unpushed, verify returns False
- test_commit_and_push_dirty_handles_missing_remote_branch: origin/branch missing, tree dirty, verify full flow runs

## Success criteria
- scripts/ci.sh exits 0
- Unpushed commits get pushed even when tree is clean
- Missing remote branch does not crash
