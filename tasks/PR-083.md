# PR-083: Dashboard per-PR metrics panel

Branch: pr-083-metrics-dashboard
- Type: feature
- Complexity: medium
- Depends on: PR-082
- Priority: 3
- Coder: any

## Problem

No visibility into historical PR performance. Operator cannot see which PRs took long, which had many FIX iterations, or which coder was used.

## Scope

Add "Recent PRs" panel on repo detail page showing last 20 completed runs.

## Changes

### src/web/app.py

New endpoint `GET /repo/{name}/metrics` returning recent RunRecords as JSON for HTMX partial.

### src/web/templates/components/pr_metrics.html (new)

Table with columns: Task, Coder, Model, Duration, FIX Iterations, Exit Reason. Sorted by started_at desc. Color-coded exit_reason (green=merged, red=error, yellow=rate_limit).

### src/web/templates/repo.html

Add `{% include "components/pr_metrics.html" %}` below event log. HTMX poll every 60s.

### tests/test_web_app.py

- test_metrics_endpoint_returns_records
- test_metrics_panel_renders

## Success criteria
- scripts/ci.sh exits 0
- Repo detail page shows recent PR metrics
- Table sorted by most recent first
- Color-coded exit reasons
- All tests pass
