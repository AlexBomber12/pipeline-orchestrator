# PR-154a: Mock coder shim wrapper and Python context manager

## Problem

PR-153 series delivered the test stack (Python skeleton, docker compose, config and deps). To run behavioral tests against this stack, the test daemon must invoke the coder CLIs (claude, codex) but those calls must NOT hit Anthropic or OpenAI APIs (cost, quota, non-determinism). The solution is a wrapper script that replaces the claude and codex binaries inside the test containers with a deterministic shim. The shim controls the daemon's perceived coder behavior via a SCENARIO file: each test sets a scenario before uploading a task, the shim reads that scenario on every invocation and acts accordingly.

This PR delivers ONLY the shim infrastructure: the bash wrapper script, the Python context manager that writes scenario files, the docker compose mounts that override /usr/local/bin/claude and /usr/local/bin/codex inside the test containers, and a verification helper. It does NOT deliver any tests yet — those are in PR-154b/c/d.

The shim performs real git operations (checkout, commit, push, gh pr create, gh pr comment) but no LLM API calls. Token cost per test run is zero. Tests still exercise the daemon's full GitHub interaction code paths.

## Scope

Add tests/e2e/lib/coder_shim.sh (bash wrapper). Add tests/e2e/lib/coder_shim.py (Python context manager and verifier). Update docker-compose.test.yml to mount the shim script as /usr/local/bin/claude and /usr/local/bin/codex on web-test and daemon-test, and add the PO_SHIM_SCENARIO_FILE environment variable. Add a single shim-verification test in tests/e2e/test_shim.py that confirms the shim is mounted and responds.

## Files to create

`tests/e2e/lib/coder_shim.sh`: bash script. Begins with shebang and `set -euo pipefail`.

The script reads the scenario from these sources in priority order:
1. PO_SHIM_SCENARIO_FILE env var pointing at a path; if file exists, read its first line
2. SHIM_SCENARIO env var
3. Default "success"

Whitespace from the scenario string is trimmed.

The script accepts arbitrary arguments (the daemon may pass any flags through; both claude and codex CLIs accept --print, --append-system-prompt, etc.). The shim ignores all arguments except for parsing the slug from the working directory.

Behavior per scenario, all running from `cd /data/repos/AlexBomber12__pipeline-orchestrator-testbed` (the only repo in config.test.yml):

For "success":
- Read the active DOING task from tasks/QUEUE.md. Look for the first "## PR-NUMBER:" heading followed (within 4 lines) by "Status: DOING" and "Branch: BRANCH_NAME". Extract PR-NUMBER and BRANCH_NAME using grep+sed or awk.
- If no DOING task is found, print "shim: no DOING task in QUEUE.md, exiting 0" to stderr and exit 0 (this is normal during IDLE polls).
- Configure git user.email "shim@test.invalid" and git user.name "Shim Coder" via `git config` (local to repo).
- Run `git fetch origin` then `git checkout -B "$BRANCH_NAME" origin/main` (force-create or reset the branch from main).
- Append a single line to a file named `tests/e2e-shim-marker.txt` inside the repo: "shim marker for PR-NUMBER at YYYY-MM-DDTHH:MM:SSZ". Create the file and parent directories if needed.
- `git add tests/e2e-shim-marker.txt`. `git commit -m "PR-NUMBER: shim implementation"`.
- `git push -u origin "$BRANCH_NAME" --force-with-lease`.
- `gh pr create --base main --head "$BRANCH_NAME" --title "PR-NUMBER: shim" --body "Shim PR for testing"`. Capture stdout (the PR URL).
- `gh pr comment <pr_url> --body "@codex review"`.
- Exit 0.

For "no_pr": same as success up to and including git push, but skip `gh pr create` and the comment. Exit 0.

For "exit_nonzero": print "shim: simulating coder failure" to stderr, exit 1.

For "hang": sleep 120 (longer than test config's planned_pr_timeout_sec=60 and fix_idle_timeout_sec=60). Exit 0 if not killed.

For "malformed_pr": same as success but force the branch name to "wrong-prefix-PR-NUMBER" (replace "pr-NUMBER-..." with "wrong-prefix-PR-NUMBER") so the daemon's branch validity check fails.

For "slow": same as success but `sleep 30` between checkout and commit, giving tests time to send Stop while daemon is in CODING.

The script is idempotent for re-runs: if the branch already exists locally or on origin, the `git checkout -B` and `git push --force-with-lease` handle reset cleanly.

`tests/e2e/lib/coder_shim.py`: Python module.

Imports: contextlib, pathlib.Path, subprocess.

Constants:
- SHIM_SCENARIO_PATH: Path to "tests/e2e/data/shim-scenario" relative to repo root (REPO_DIR from conftest)
- VALID_SCENARIOS: set of {"success", "no_pr", "exit_nonzero", "hang", "malformed_pr", "slow"}

`@contextlib.contextmanager` function `coder_shim(scenario: str)`:
- Assert scenario in VALID_SCENARIOS
- Ensure SHIM_SCENARIO_PATH parent directory exists
- Save the previous content of the file (if any) to restore on exit
- Write the scenario string + newline to the file
- yield
- On exit (finally block), restore previous content or write "success" if there was no previous content

Function `verify_shim_installed(compose_file: str = "docker-compose.test.yml") -> None`:
- Run `docker compose -f compose_file exec -T daemon-test which claude`
- Capture stdout, strip whitespace
- Assert equals "/usr/local/bin/claude"
- Run the same for codex
- On any assertion failure, raise RuntimeError with a clear message indicating the shim is not mounted. Suggest checking docker-compose.test.yml volume mounts.

`tests/e2e/test_shim.py`: one test `test_shim_is_mounted_and_callable`.

Imports verify_shim_installed from coder_shim.py.

Body:
- Call verify_shim_installed(). If it raises, the test fails with the exception message.
- Run `docker compose -f docker-compose.test.yml exec -T daemon-test /usr/local/bin/claude` with timeout 5 seconds. Since no SCENARIO is set in the env, default "success" applies, but no DOING task exists in QUEUE.md (testbed is freshly cloned), so the shim prints "no DOING task" to stderr and exits 0.
- Assert exit code is 0 OR the stderr contains "no DOING task".

This test is NOT a behavioral test of the shim itself (those come in PR-154b/c/d via real flows). It exists to catch misconfigured docker-compose.test.yml mounts before debugging confusing test failures elsewhere.

## Success criteria

1. `bash -n tests/e2e/lib/coder_shim.sh` exits 0 (bash syntax check). If shellcheck is available, `shellcheck tests/e2e/lib/coder_shim.sh` exits 0 (the implementer may need to add shellcheck disable comments for unavoidable issues, but should not have substantive warnings).

2. `python -c "from tests.e2e.lib.coder_shim import coder_shim, verify_shim_installed, VALID_SCENARIOS; assert 'success' in VALID_SCENARIOS"` exits 0 (module is importable, exports the expected names).

3. With the test stack rebuilt and up (`docker compose -f docker-compose.test.yml down -v && docker compose -f docker-compose.test.yml up -d`), `docker compose -f docker-compose.test.yml exec daemon-test which claude` returns `/usr/local/bin/claude` and `which codex` returns `/usr/local/bin/codex`.

4. `pytest tests/e2e/test_shim.py -v` collects exactly one test and that test passes.

5. The full e2e suite `pytest tests/e2e/ -v` collects 2 tests (smoke from PR-153a/c plus shim verification from this PR) and both pass.

6. Manual verification: with the test stack up, write "success" to tests/e2e/data/shim-scenario, observe that the daemon's polling cycle invokes the shim. Since QUEUE.md is empty initially, the shim prints "no DOING task" to stderr (visible in `docker compose -f docker-compose.test.yml logs daemon-test`).

7. The Python context manager `coder_shim("hang")` correctly writes "hang" to tests/e2e/data/shim-scenario and restores the previous value (or "success" if no previous value) on exit. Verifiable via a one-liner: `python -c "from tests.e2e.lib.coder_shim import coder_shim; from pathlib import Path; p = Path('tests/e2e/data/shim-scenario'); p.parent.mkdir(parents=True, exist_ok=True); p.write_text('success'); 
with coder_shim('hang'): assert p.read_text().strip() == 'hang'
assert p.read_text().strip() == 'success'"`.

8. The Python context manager rejects unknown scenarios: `coder_shim("invalid_name")` raises AssertionError before writing anything to the file.

9. `git ls-tree HEAD tests/e2e/lib/coder_shim.sh` shows mode "100755" (executable bit committed).

10. scripts/ci.sh exits 0 unchanged. ruff check src tests passes. Coverage stays at 100% (no src/ changes in this PR).
