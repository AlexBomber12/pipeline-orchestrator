# PR-024: Upload tasks and git push from dashboard

Branch: pr-024-upload-tasks

## Scope

Add file upload endpoint that accepts QUEUE.md and task files (PR-*.md), writes them to the repo clone, commits, and pushes to origin. This allows loading sprint tasks directly from the dashboard without opening VS Code or GitHub.

## How it works

1. User drags & drops (or selects) files onto the "Upload tasks" area on a repo card
2. Dashboard sends files to POST /repos/{name}/upload-tasks
3. Server writes files to /data/repos/{name}/tasks/
4. Server commits and pushes to origin/{branch}
5. Dashboard refreshes the repo card to show new queue status

## Changes

### src/web/app.py

Add route:

POST /repos/{name}/upload-tasks
- Accept multipart file upload (multiple files)
- Validate:
  - Repo exists in config
  - Repo is cloned at /data/repos/{name}
  - Files are .md files only
  - At least one file named QUEUE.md
  - File names match pattern: QUEUE.md or PR-*.md
  - Total upload size < 1MB
- For each file:
  - Write to /data/repos/{name}/tasks/{filename}
  - Overwrite if exists (QUEUE.md always overwrites, PR-*.md files too)
- Git operations (subprocess in /data/repos/{name}):
  - `git checkout {branch}` (ensure on base branch)
  - `git pull origin {branch}` (get latest)
  - Write files to tasks/
  - `git add tasks/QUEUE.md tasks/PR-*.md`
  - `git commit -m "chore: upload sprint tasks via dashboard"`
  - `git push origin {branch}`
- On success: return updated repo card partial (HTMX swap)
- On error: return error message partial with 422 status

Helper function:

_git_run(repo_path: str, *args: str, timeout: int = 30) -> subprocess.CompletedProcess
- Wrapper for subprocess.run with cwd, check, capture_output, text, timeout
- Raises RuntimeError with stderr on failure

### Environment for git push

The web container needs git write access. Currently docker-compose.yml mounts `./data:/data` and has GH_CONFIG_DIR set. The web container should also have `gh auth setup-git` run at startup, or the git push will fail.

Add to docker-compose.yml web service command or entrypoint:
- Create scripts/web-entrypoint.sh:
```bash
#!/usr/bin/env bash
gh auth setup-git 2>/dev/null || true
exec "$@"
```
- Set entrypoint in docker-compose.yml: `entrypoint: ["bash", "scripts/web-entrypoint.sh"]`
