# PR-043: GitHub API optimization

Branch: pr-043-gh-api-optimize

## Scope

Reduce GitHub API calls per poll cycle. Currently get_pr_review_status makes 4-5 paginated API calls per PR per cycle (issue reactions, issue comments, review comments, anchor reactions, codex reviews). With 60s polling, that's 300+ API calls per hour for a single repo. Does not scale.

## Changes

### src/github_client.py

1. Combine data fetching. Instead of separate paginated calls for reactions, comments, and review comments, fetch what we can in fewer calls:

```python
def get_pr_review_status_cached(
    repo: str,
    pr_number: int,
    pr_author: str = "",
    head_sha: str = "",
    _cache: dict | None = None,
) -> ReviewStatus:
```

Pass a shared cache dict from get_open_prs so multiple PRs in the same repo share comment data.

2. Use `gh api` with `--jq` to extract only needed fields instead of fetching full payloads:

For issue reactions:
```python
run_gh(["api", f"repos/{repo}/issues/{pr_number}/reactions", "--jq",
        '[.[] | select(.user.login | test("codex";"i")) | {content, created_at}]'])
```

This returns only Codex reactions, not all reactions.

3. Skip deep inspection when not needed. Add a fast-path:
- If PR is in CODING state, don't check review status (waste of API calls)
- If PR just entered WATCH (push_count == current), skip review check on first cycle (Codex hasn't had time to review)
- Cache review status for 30 seconds - if last check was < 30s ago, return cached result

4. Consolidate has_recent_codex_review_request, get_pr_author, get_pr_head_commit_iso into fewer API calls:

```python
def get_pr_metadata(repo: str, pr_number: int) -> dict:
    """Fetch PR author, head SHA, and head commit date in 1 API call."""
    raw = run_gh([
        "api", f"repos/{repo}/pulls/{pr_number}",
        "--jq", '{author: .user.login, head_sha: .head.sha}'
    ])
    # ... then get commit date from head_sha in 1 more call
```

This replaces 3 separate API calls with 1-2.

5. Add optional short-circuit: if PR body has fresh +1 from Codex AND no P1/P2 comments after last push, return APPROVED without checking comment reactions (saves 2 API calls).

### src/daemon/runner.py

1. In handle_watch, pass a hint about current state to avoid unnecessary API calls:
```python
prs = github_client.get_open_prs(self.owner_repo, skip_review_for=[])
```

2. In _post_codex_review, batch the metadata fetch:
```python
metadata = github_client.get_pr_metadata(self.owner_repo, pr_number)
pr_author = metadata.get("author", "")
head_commit_iso = metadata.get("head_commit_date", "")
```

### tests/test_github_client.py

- test_review_status_fast_path_approved: PR body +1, no P1/P2, verify returns APPROVED with fewer API calls
- test_review_status_cached: verify repeated calls within 30s return cached result
- test_get_pr_metadata_single_call: verify 1 API call returns author + head_sha

## Success criteria
- scripts/ci.sh exits 0
- API calls per cycle reduced by at least 40%
- Review status caching prevents redundant checks
- Existing review detection behavior unchanged
- No regressions in stale review detection
