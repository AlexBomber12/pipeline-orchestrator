# PR-131: Active state label pulses while daemon is live

Branch: pr-131-active-state-pulse
- Type: feature
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Current state badge (IDLE/CODING/WATCH/FIX/MERGE) in the repo detail page header is static. On the repo card there is already a pulsing dot for active states (CODING, WATCH, FIX) via `pulse-dot` animation, but the detail-page header does not follow suit.

Users need a clear visual cue that the daemon is alive and working. A pulse on the state badge during active states makes this immediate.

## Scope

Apply the existing `pulse-dot` CSS animation to the state badge dot in the repo detail page for states that are considered "active": CODING, WATCH, FIX, MERGE.

Do NOT pulse for IDLE, PREFLIGHT, PAUSED, HUNG, ERROR - those are "resting" or "error" states and a pulse would be misleading.

Reuse the `state_styles` dict pattern from `repo_cards.html` - it already has a `pulse: true/false` field per state. In `repo.html`, use the same dict to drive the pulse conditional on the detail page state badge.

This PR does NOT include the "stalled/grey" overlay - that is PR-132. This is purely about adding pulse to the detail page.

## Files to modify

- `src/web/templates/repo.html`: import or duplicate the `state_styles` dict, apply `pulse-dot` class conditionally to the state badge dot.
- If `state_styles` dict is currently only in `repo_cards.html`, extract it to a shared macro or include-partial so both templates reference the same source of truth.
- Tests: render test verifying `pulse-dot` class is present for CODING/WATCH/FIX/MERGE on the detail page and absent for IDLE/ERROR.

## Success criteria

- Detail page state badge pulses during CODING/WATCH/FIX/MERGE.
- Detail page state badge is static during IDLE/PREFLIGHT/PAUSED/HUNG/ERROR.
- Repo card behavior unchanged (it already had pulse).
- No duplicate `state_styles` dict - either extracted to a shared partial or defined once and used in both.
