# PR-033: Model config in config.yml

Branch: pr-033-model-config

## Scope

Claude CLI currently uses the container default model. Add explicit model configuration to config.yml so it is version-controlled and not dependent on container settings.

## Changes

### src/config.py

Add to DaemonConfig:
```python
claude_model: str = "opus"
```

### src/claude_cli.py

Add model parameter to run_claude:
```python
def run_claude(
    prompt: str,
    cwd: str,
    timeout: int = 600,
    model: str | None = None,
) -> tuple[int, str, str]:
```

If model is not None, add ["--model", model] to cmd.

Update run_planned_pr, fix_review, diagnose_error to accept and pass model parameter.

### src/daemon/runner.py

Pass self.app_config.daemon.claude_model to all claude_cli calls.

### config.yml

Add under daemon section:
```yaml
daemon:
  claude_model: opus
```

### tests/test_claude_cli.py

- test_run_claude_passes_model_flag: verify --model added to cmd
- test_run_claude_no_model_flag_when_none: verify no --model when None

### tests/test_config.py

- test_daemon_config_claude_model_default: verify default is "opus"

## Success criteria
- scripts/ci.sh exits 0
- Model is explicitly configured, not implicit
- Changing config.yml changes the model without rebuilding the Docker image
