# PR-150: Event log state badges have consistent width

Branch: pr-150-state-badge-alignment
- Type: ux
- Complexity: low
- Depends on: none
- Priority: 2
- Coder: any

## Problem

Event log state badges (IDLE, CODING, WATCH, FIX, MERGE, HUNG, ERROR, PAUSED, PREFLIGHT) render with intrinsic text width. The longest state name "PREFLIGHT" (9 chars) is noticeably wider than "FIX" (3 chars). Rows with shorter states have their message text starting further left, rows with longer states push the message right. The event log looks ragged and the eye cannot scan the message column.

Current badge class in two templates:

`text-[10px] font-medium px-1.5 py-0.5 rounded border {{ entry_style.badge }} shrink-0`

No min-width, no text-align. Width is `content + px-1.5 * 2 = content + 12px`.

## Scope

Add `min-w-[4.5rem] text-center` to the badge span class. 4.5rem = 72px, sufficient for "PREFLIGHT" at 10px font + px-1.5 padding (measured: ~65px natural width). 72px gives a small buffer while staying compact. If visual review shows 4.5rem is tight, bump to 5rem.

Apply identically in both files:

- `src/web/templates/components/event_list.html` line 41 (full event log on detail page)
- `src/web/templates/components/activity_feed.html` line 27 (compact activity feed on dashboard)

No other changes. Badge colors per-state (from `entry_style.badge`) stay as-is.
