# PR-170: Remove STALLED indicator

- Type: ux
- Complexity: low

## Problem

The dashboard shows a `STALLED` badge next to the IDLE state when daemon hasn't updated for 30+ seconds. Implementation in `src/web/templates/base.html`: `.stalled` CSS class (greyscale + 50% opacity) plus `syncStalledBadges()` JS that toggles it via 5-second polling. The threshold constant is `STALLED_THRESHOLD_MS = 30 * 1000`.

This produces visual false positives. Daemon in IDLE between cycles is normal behavior, not a problem. Users see a faded grey card and assume something is broken when in fact the system is healthy. Also reported per memory entry from earlier sessions: "STALLED indicator появляется рядом с IDLE даже сразу после успешного merge'а" — the badge persists right after a successful merge, when daemon is healthily idle waiting for next task.

The architectural decision in roadmap line 152 was to remove STALLED entirely. This PR executes that decision.

## Scope

Delete the STALLED badge JS, CSS, hint spans, and supporting attributes. Pure deletion. No replacement. Daemon health is communicated through other signals (state badge, event log timestamps).
