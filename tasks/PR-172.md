## Problem

Continuation of PR-171. The Start/Stop controls (and Play/Pause when added in PR-179/180) and the coder selector dropdown also lack visual feedback during HTMX requests. Same root issue, different templates.

## Scope

Apply the spinner pattern from PR-171 to `_controls.html` and the coder selector. Reuse `components/_spinner.html`.

## Files to touch

`src/web/templates/components/_controls.html`: add `hx-indicator` to Start/Stop buttons. Include spinner partial inline. Each button gets its own spinner instance with unique id (e.g., `controls-spinner-{repo_slug}`) so spinners don't conflict across multiple repos on dashboard.

`src/web/templates/components/coder_selector.html` (or wherever the per-repo coder dropdown lives): the dropdown's Save action gets `hx-indicator` and a spinner. Spinner appears next to the dropdown when the change is being persisted.

`tests/test_templates.py` (or `tests/test_dashboard.py`): assert spinners present on the controls component and coder selector. Multi-repo case: two repo cards have two distinct spinner ids (no collision).

## Files NOT to touch

`components/_spinner.html`: created in PR-171, reuse as is.

`src/daemon/`, `src/models.py`: no backend changes.

`src/web/templates/index.html`, `settings.html`: handled in PR-171.

`docker-compose.yml`, `Dockerfile`, `config.yml`: untouched.
