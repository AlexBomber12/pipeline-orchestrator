# PR-089: Upload validation for task file headers

Branch: pr-089-upload-header-validation
- Type: feature
- Complexity: low
- Depends on: PR-084
- Priority: 3
- Coder: any

## Problem

Dashboard upload accepts any task file without validating structured headers. Missing "Depends on" field silently makes task eligible immediately, which may be a mistake.

## Scope

When a task file (PR-*.md) is uploaded via dashboard, validate that it has all required header fields. Reject with HTTP 400 and clear error if "Depends on" is missing.

## Changes

### src/web/app.py

In upload_tasks handler, after reading file content: if filename matches `PR-*.md`, call `parse_task_header()` on content. If QueueValidationError raised, return HTTP 400 with error message: "Task file missing required field: Depends on. Use 'Depends on: none' for tasks with no dependencies."

### tests/test_upload.py

- test_upload_rejects_task_without_depends_on
- test_upload_accepts_task_with_depends_on_none
- test_upload_accepts_task_with_dependencies
- test_upload_allows_non_task_files_without_validation (QUEUE.md, AGENTS.md, etc.)

## Success criteria
- scripts/ci.sh exits 0
- Upload of PR-*.md without "Depends on" line returns 400 with clear message
- Upload of PR-*.md with "Depends on: none" succeeds
- Non-task files (QUEUE.md, AGENTS.md) not validated by task header parser
- All tests pass
