# PR-076: Shared process-level usage provider

Branch: pr-076-shared-usage-provider
- Type: architecture
- Complexity: medium
- Depends on: PR-074, PR-075
- Priority: 2
- Coder: any

## Problem

Each PipelineRunner creates its own OAuthUsageProvider and OpenAIUsageProvider. With N repos, same endpoint is called N times per cycle. Waste of API calls and latency.

## Scope

Move usage provider creation to daemon main.py. One provider per coder, shared across all runners.

## Changes

### src/daemon/main.py

Create providers once at startup:
```python
claude_provider = ClaudePlugin().create_usage_provider(config=app_config)
codex_provider = CodexPlugin().create_usage_provider(config=app_config)
```

Pass to each runner constructor.

### src/daemon/runner.py

Remove provider creation from `__init__`. Accept `claude_usage_provider` and `codex_usage_provider` as constructor params. Store as `self._claude_usage_provider` and `self._codex_usage_provider` (same attribute names, different source).

Update `app_config.setter` to NOT recreate providers (they live in main.py now).

### tests/

Update runner construction in tests to pass mock providers.

## Success criteria
- scripts/ci.sh exits 0
- Only 1 HTTP call per coder per cache TTL cycle regardless of repo count
- All tests pass
- No behavior change for single-repo setup
