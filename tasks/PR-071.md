# PR-071: Pydantic validation for config fields

Branch: pr-071-config-validation
- Type: feature
- Complexity: low
- Depends on: none
- Priority: 4
- Coder: any

## Problem

Most numeric config fields lack model-level validation. Manual config.yml edits with invalid values (negative timeouts, port 99999) pass silently and cause runtime errors.

## Scope

Add Pydantic Field constraints to DaemonConfig and WebConfig.

## Changes

### src/config.py

Add constraints:
- `review_timeout_min: int = Field(default=60, ge=1)`
- `planned_pr_timeout_sec: int = Field(default=900, ge=60)`
- `upload_staging_max_age_hours: int = Field(default=24, ge=1)`
- `rate_limit_session_pause_percent: int = Field(default=95, ge=0, le=100)`
- `rate_limit_weekly_pause_percent: int = Field(default=100, ge=0, le=100)`
- WebConfig: `port: int = Field(default=8000, ge=1, le=65535)`

### tests/test_config.py

Add tests:
- test_config_rejects_negative_review_timeout
- test_config_rejects_zero_planned_pr_timeout
- test_config_rejects_rate_limit_over_100
- test_config_rejects_invalid_port

## Success criteria
- scripts/ci.sh exits 0
- Invalid config values raise ValidationError on load
- All existing tests pass (may need minor updates for new constraints)
